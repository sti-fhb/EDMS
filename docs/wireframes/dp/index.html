💡 內部設計註記（UCDP005 使用者管理）
  • 帳號為 DP_USER，ET / DM 共用；此為共用項，ET 管理者與 DM 管理者皆可存取與操作（DP 後台由兩模組管理者共用，不另設獨立管理角色）。

  • 連續 90 日未登入之帳號由系統自動停用 / 鎖定。
  • 使用者可自助註冊即用；管理者代建之帳號亦直接可用（不寄開通確認信）。
  • 此頁不設角色指派——ET / DM 之業務角色（教師 / 學員、DM 各角色）於 DP 後台「權限管理」畫面按模組過濾指派，寫入模組角色表（ET_USER_ROLE / DM_USER_ROLE）；權限判定與 enforce 在模組，平台不做全域 RBAC。

💡 內部設計註記（UCDP006 系統參數與清單維護）
  • 全平台參數值與清單定義統一存 DP_PARAM_M / DP_PARAM_D，以 PARAM_ID 前綴分模組（無前綴 DP_ 平台級 / ET_ / DM_）；清單型（分類 / 標籤名稱 / func_name / 可見對象）以一個 PARAM_ID 下多筆 PARAM_KEY 表達。異動寫入稽核。
  • 三頁籤按模組區分：平台（共用）/ ET / DM 三頁籤；登入為 ET 管理者只顯示「平台 + ET」頁籤、DM 管理者只顯示「平台 + DM」（對方模組頁籤不顯示、互不可見）；平台級 DP_ 參數兩者共用可改。
  • 鎖定碼：需鎖定之碼（如文件分類碼建立後不可改）以 DP_PARAM_M.DETAIL_LOCK 標記，UI 呈唯讀鎖 。
  • 啟用 / 停用：清單項目（分類 / func_name / 標籤 / 可見對象）以動作欄「停用 / 啟用」按鈕切換 DP_PARAM_D.IS_ACTIVE——停用不刪除、既有引用保留，僅不出現在後續新增 / 搜尋下拉；可見對象停用採 soft-retire（不收回既有可見性）；ET / DM「全體」（IS_ALL / 通用值）不可停用（按鈕鎖定）。
  • 定義 vs 關聯分層：清單「定義」（有哪些）在 DP_PARAM；「誰配誰」關聯（文件屬哪分類、使用者 × 標籤、課程 × 標籤、閱覽者 × 可見對象）與套用規則（DOC_ID 產碼、IS_ALL 展開與自動邀請、func_name 唯一手冊檢核）留各模組。
  • 密碼長度分級（一般 8 / 特權 12）保留以對齊國防部資安法規；密碼重設 TTL、發信調校（重試 / 速率 / 告警）改列平台級 DP_（LOGIN / MAIL）。
  • 帳號類信件（密碼重設 / 帳號驗證）不列為參數，已改為通知範本（DP_NOTIFY_TEMPLATE MODULE=DP），於「通知範本」畫面維護。
