畫面操作說明（僅供需求確認，正式系統不顯示）
本畫面用途：教師編輯課程內容，採單頁 scroll 設計（不用 tab，方便拖拉編排）
編輯 vs 檢視模式：
  • 編輯模式（owner 進入）：全功能可用，所有編輯控制項可操作
  • 檢視模式（非 owner 進入）：頂部顯示黃色 banner、所有編輯控制項禁用；新增 / 刪除 / 儲存 / 發布 / 邀請 / 停課 等所有按鈕皆隱藏，僅可閱覽內容
右上「邀請學員」按鈕：僅已發布的課程顯示；草稿狀態下隱藏。點擊開啟邀請 modal，含 Email 邀請 + 邀請碼兩分頁
基本資料（摺疊式）：名稱、關聯模組、狀態、課程描述；開課日期由系統自動以首次發布時間寫入（唯讀顯示）；預設展開、可摺疊讓編排區佔比較大空間
章節編排（主區塊）：RQET002 拖拉式介面
• 章節層級：拖動 grip 調整章節順序；強制完成（須依序完成每章後才能進下一章）為系統內建規則，不需逐章設定（RQET004）
• 章節內項目層級：拖動 grip 可在章節內或跨章節移動；每個項目 icon 顯示類型（影片/文件/圖文/測驗）
• 新增項目：點下拉選單選類型（教材 / 測驗）→ 開對應 modal；教材 modal 可同時包含說明文字、本地上傳影片、與多份 DM 訓練教材文件（RQET003）
教材文件來源：教材所引用的文件（PDF / PPT 等）統一由 DM 模組提供（同一 DB）；ET 端下拉清單僅顯示 DM「訓練教材」分類之已發布文件，點選即建立關聯；DM 端發布新版時，教材自動帶最新版
Sticky 動作列：常用動作（儲存草稿、儲存並發布）固定在頁底，scroll 時不消失；學員管理請從左側 sidebar 進入
對應需求：RQET001（影片本地上傳）、RQET002（拖拉）、RQET003（混合媒材）、RQET004（強制完成，系統內建）

停課
停課後此課程將自動下架、無法再編輯。此操作不可逆。
  • ET01 課程列表此課程將顯示「已停課」狀態 pill，僅可進入檢視模式
  • 學員端 ET04 我的課程之「進行中 / 未開始」分區不再顯示本課程；「已完成課程」分區仍保留歷史紀錄
  • 學員嘗試開啟此課程時顯示「此課程已停止開放」
  • ET03 學員教師端可繼續查看已加入學員之完課狀態 / 平均成績，但無法再邀請新學員
  • 已加入學員之歷史學習紀錄與測驗成績完整保留

重置重考次數
即將為 學員 重置該課程內所有測驗的可用重考次數。

  • 系統將記錄修改者與時間於稽核 log
移除學員

  • 學習歷程與測驗紀錄保留於稽核 log，不會刪除
