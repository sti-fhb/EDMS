> 本文件以 `_refs/09-平台模組.md` 之章節結構為主體。

## 模組概述

平台模組（DP）為 EDMS 全系統共用基礎 + **統一管理後台**，非業務模組。ET（教育訓練）與 DM（文件管理）共用其**帳號、認證、稽核**，以及系統參數 / 清單定義 / 通知範本 / 排程之**儲存與引擎**；各模組的業務角色**能力與判定**、以及關聯 / 指派資料（誰配哪個標籤/角色/分類）仍留各模組（權限分開做）。表名沿用 `DP_` 前綴。涵蓋：使用者登入（簡單 JWT）、忘記密碼、密碼變更、使用者/帳號管理、帳號鎖定、**系統參數與清單定義（全平台集中、前綴分模組）、通知範本與發信引擎、排程引擎、權限管理（角色指派）**、操作記錄（稽核）。所有管理畫面集中於 DP 後台、按模組過濾；**「有哪些」的定義（分類/標籤/func_name/參數值）存 DP、「誰配誰」的關聯與角色判定留模組**（2026-07-08 集中化決策，見 `_refs/09-平台模組.md`）。

### 操作角色

平台**不另設**「平台管理員」；DP 後台由 ET / DM 管理者進入，靠模組過濾決定可見與可改內容。

| 角色 | 職責範圍 |
|------|---------|
| ET 管理者 | ET 業務角色（`ET_USER_ROLE`）。於 DP 後台維護 ET 相關（前綴 `ET_` / `MODULE=ET`）之參數 / 清單 / 範本 / ET 角色指派；並可存取共用項 |
| DM 管理者 | DM 業務角色（`DM_USER_ROLE`）。於 DP 後台維護 DM 相關（前綴 `DM_` / `MODULE=DM`）之參數 / 清單 / 範本 / DM 角色指派；並可存取共用項 |
| 一般使用者 | 登入 / 登出、忘記密碼、變更密碼、維護自己的基本資料 |
| eMail Server | 外部郵件系統，平台發信引擎透過它寄發所有信件（密碼重設 / 帳號驗證 / 各模組業務通知）|

> **可見與存取規則**：模組項（前綴 `ET_`/`DM_`、`MODULE=ET`/`DM`、各模組角色）按模組隔離、互不可見；**共用項**（帳號管理、平台級無前綴 `DP_` 參數、操作記錄稽核）不分模組，**ET / DM 管理者皆可存取**。
> **取捨**：帳號與平台級參數為共用，故 ET 管理者可停用帳號（影響 DM）、可改密碼策略（影響全平台）、可查全部稽核——為換取「單一後台、無獨立 IT 角色」之簡化，接受此共管範圍。

### 作業流程

```
使用者登入
─────────
1. 輸入帳號（Email）+ 密碼 → 系統驗證帳密
2. 失敗累計次數，達上限自動鎖定；成功則核發 JWT、寫登入稽核
3. 登入後由各模組依角色導向預設首頁

忘記密碼
─────────
1. 登入頁點「忘記密碼」，輸入帳號
2. 系統產生具時效之一次性重設 token，寄重設信（防帳號列舉）
3. 使用者點連結、驗證 token 與期限、輸入新密碼（檢核複雜度/重複性）→ 更新並記錄稽核

使用者/帳號管理
─────────
1. 使用者自助註冊即用（自動取得所屬模組預設角色）
2. 管理者可建立 / 停用 / 啟用帳號、維護基本資料
3. 帳號（Email）變更採新信箱驗證後切換（延遲生效）

系統維運（DP 統一後台，ET / DM 管理者，按模組過濾）
─────────
1. 維護系統參數與清單定義（平台級共用；ET_/DM_ 按模組）、通知範本、排程時間
2. 指派各模組角色（畫面在後台，寫入模組角色表）
3. 帳號管理（共用）、操作記錄查詢（共用）
```

## 跨模組介接

| 介接對象 | 介接方式 | 說明 |
|---------|---------|------|
| ET / DM | 共用帳號 | 兩模組共用 `DP_USER`；登入後各自依角色載入功能 |
| ET / DM | 資安稽核 | 各模組之資安稽核事件（帳號 / 角色權限 / 系統操作）統一寫入 `DP_AUDIT_LOG`；業務歷程（文件變更、閱讀、學習紀錄）留各模組 |
| ET / DM | 系統參數 + 清單定義 | 值與清單（分類/標籤/func_name/可見對象）存 `DP_PARAM`（前綴分模組），平台提供唯讀查詢服務；維護畫面於 DP 後台按模組過濾；業務規則歸模組 |
| ET / DM | 通知範本 + 發信 | 範本存 `DP_NOTIFY_TEMPLATE`（`MODULE` 分模組）；各模組呼叫平台唯一發信服務並傳 `template_code`，經 `DP_EMAIL_LOG` outbox 非同步寄送 |
| ET / DM | 角色指派 | 指派畫面於 DP 後台（按模組過濾），寫入模組角色表 `ET_USER_ROLE` / `DM_USER_ROLE`；判定與 enforce 在模組 |
| ET / DM | 排程 | 各模組 job 於 `DP_SCHEDULE` 註冊、平台引擎執行、`DP_SCHEDULE_LOG` 記錄；job 反向 import 模組 service |
| 外部 eMail 系統 | SMTP | 平台發信引擎透過它寄發所有信件（密碼重設 / 帳號驗證 / ET / DM 業務通知），統一走 `DP_NOTIFY_TEMPLATE` 範本 |

## 需求編號說明

- **無 RFP RQ 編號**：EDMS 平台模組（DP）為模組拆分後新增之共用層，RFP／`RQ0.md` 中無屬於 EDMS 平台自身之需求編號。TBMS `RQ0.md` 的 `RQDP001–063` 屬**主專案「資訊模組」**（多站點、全域 RBAC、血袋、看板、外部 API 等），範圍與 EDMS 平台不同，**不沿用其編號**。
- **來源一律「分析資料」**：本文件所有需求以 `_refs/09-平台模組.md` 分析為權威來源。
- **資安法規對應**：密碼、稽核、加密、傳輸安全等項目對應國防部資安法規（需求說明書通則），EDMS 同樣適用；如日後需正式 RQ 追蹤，另於 EDMS 之 `RQ0.md` 建立平台需求編號後回填本文件。

## 範圍界定（明確不納入）

平台模組相對 TBMS「資訊模組」已裁剪，以下**不納入**（詳見 `_refs/09-平台模組.md`）：

- 多站點 / 院區架構（`DP_SITE`、單位維度）
- 全域 RBAC 與功能選單、集中式角色能力定義（`DP_ROLE` / `DP_MENU` / `DP_ROLE_MENUS`）— 權限完全分開做，角色能力定義 / 指派資料 / 判定皆由各模組（ET/DM）自理；角色**指派畫面**在 DP 後台，但指派資料寫入模組角色表（`ET_USER_ROLE` / `DM_USER_ROLE`）
- 平台管理員旗標（`IS_PLATFORM_ADMIN`）— 不另設平台管理員；DP 後台由 ET / DM 管理者進入，共用項（帳號 / 平台參數 / 稽核）兩者皆可存取
- MFA 多重驗證、Refresh Token / 伺服器端 session
- 資訊設備 / IP 對照 / 標籤印表機
- 對外系統 API、健檢資料介接、看板、多據點同步
- 血袋編碼 / ISBT128 / 周邊設備等 TBMS 血品業務

> 文件管理與教育訓練需求已於模組拆分時移至 [RQDM.md](RQDM.md) / [RQET.md](RQET.md)。
