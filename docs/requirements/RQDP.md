# 平台模組 需求清單（RQDP）

> 本文件以 `_refs/09-平台模組.md` 之章節結構為主體。
> **平台模組（DP）為 EDMS 模組拆分後新增之共用層，於 RFP／`RQ0.md` 中無屬於 EDMS 平台自身之 RQ 編號**（TBMS `RQ0.md` 的 RQDP001–063 屬**主專案「資訊模組」**，範圍不同、不沿用）。故本文件所有需求編號一律為「—」、來源標「分析資料」，以 `09-平台模組.md` 為權威來源。
> **2026-07-06 建立**。其中密碼 / 稽核 / 加密 / 傳輸安全等項目對應國防部資安法規（需求說明書通則），EDMS 同樣適用；詳見末段「需求編號說明」。

## 模組概述

平台模組（DP）為 EDMS 全系統共用基礎，非業務模組。ET（教育訓練）與 DM（文件管理）共用其**帳號、認證與稽核**；各模組的業務角色與權限**完全由各模組自行維護**（權限分開做；平台不碰業務角色）。表名沿用 `DP_` 前綴。涵蓋：使用者登入（簡單 JWT）、忘記密碼、密碼變更、使用者/帳號管理、帳號鎖定、**系統參數（全平台集中、前綴分模組）、通知範本與發信引擎、排程引擎**、操作記錄（稽核）。集中的是「儲存與引擎」，各模組的業務內容（參數值、範本文案、排程 job 邏輯）與維護 UI 仍由各模組自理（2026-07-08 集中化決策，見 `_refs/09-平台模組.md`）。

### 操作角色

| 角色 | 職責範圍 |
|------|---------|
| 平台管理員 | DP 自身功能之把關者（IT）：管理其他帳號（建立 / 停用 / 啟用 / 基本資料）、維護平台級 `DP_` 參數、查詢全平台操作記錄（稽核）；以 `DP_USER.IS_PLATFORM_ADMIN` 旗標判定，非業務角色體系 |
| 一般使用者 | 登入 / 登出、忘記密碼、變更密碼、維護自己的基本資料 |
| eMail Server | 外部郵件系統，平台發信引擎透過它寄發所有信件（密碼重設 / 帳號驗證 / 各模組業務通知）|

> ET / DM 的業務角色（管理者 / 教師 / 學員、DM 各角色）不在此定義；平台只認「帳號」與「是否為平台管理員」。平台管理員**看不到** ET / DM 的業務參數與範本（那些由各模組管理者於自己的系統設定頁維護）。

### 作業流程

```
使用者登入
─────────
1. 輸入帳號（Email）+ 密碼 → 系統驗證帳密
2. 失敗累計次數，達上限自動鎖定；成功則核發 JWT、寫登入稽核
3. 登入後由各模組依角色導向預設首頁

忘記密碼
─────────
1. 登入頁點「忘記密碼」，輸入帳號
2. 系統產生具時效之一次性重設 token，寄重設信（防帳號列舉）
3. 使用者點連結、驗證 token 與期限、輸入新密碼（檢核複雜度/重複性）→ 更新並記錄稽核

使用者/帳號管理
─────────
1. 使用者自助註冊即用（自動取得所屬模組預設角色）
2. 管理者可建立 / 停用 / 啟用帳號、維護基本資料
3. 帳號（Email）變更採新信箱驗證後切換（延遲生效）

系統維運
─────────
1. 平台管理員維護平台級 DP_ 參數（密碼策略、鎖定、發信引擎調校等）；DP 系統信範本亦於此維護
2. 操作記錄統一寫入稽核日誌，供資安查詢
```

## 跨模組介接

| 介接對象 | 介接方式 | 說明 |
|---------|---------|------|
| ET / DM | 共用帳號 | 兩模組共用 `DP_USER`；登入後各自依角色載入功能 |
| ET / DM | 資安稽核 | 各模組之資安稽核事件（帳號 / 角色權限 / 系統操作）統一寫入 `DP_AUDIT_LOG`；業務歷程（文件變更、閱讀、學習紀錄）留各模組 |

## 需求編號說明

- **無 RFP RQ 編號**：EDMS 平台模組（DP）為模組拆分後新增之共用層，RFP／`RQ0.md` 中無屬於 EDMS 平台自身之需求編號。TBMS `RQ0.md` 的 `RQDP001–063` 屬**主專案「資訊模組」**（多站點、全域 RBAC、血袋、看板、外部 API 等），範圍與 EDMS 平台不同，**不沿用其編號**。
- **來源一律「分析資料」**：本文件所有需求以 `_refs/09-平台模組.md` 分析為權威來源。
- **資安法規對應**：密碼、稽核、加密、傳輸安全等項目對應國防部資安法規（需求說明書通則），EDMS 同樣適用；如日後需正式 RQ 追蹤，另於 EDMS 之 `RQ0.md` 建立平台需求編號後回填本文件。

## 範圍界定（明確不納入）

平台模組相對 TBMS「資訊模組」已裁剪，以下**不納入**（詳見 `_refs/09-平台模組.md`）：

- 多站點 / 院區架構（`DP_SITE`、單位維度）
- 全域 RBAC 與功能選單、業務角色指派（`DP_ROLE` / `DP_USER_ROLE` / `DP_MENU`）— 權限完全分開做，業務角色定義 / 指派 / 判定皆由各模組（ET/DM）自理，平台不碰（平台僅以 `IS_PLATFORM_ADMIN` 旗標把關 DP 自身功能，非業務角色體系）
- MFA 多重驗證、Refresh Token / 伺服器端 session
- 資訊設備 / IP 對照 / 標籤印表機
- 對外系統 API、健檢資料介接、看板、多據點同步
- 血袋編碼 / ISBT128 / 周邊設備等 TBMS 血品業務

> 文件管理與教育訓練需求已於模組拆分時移至 [RQDM.md](RQDM.md) / [RQET.md](RQET.md)。
