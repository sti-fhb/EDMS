# User Story 1 — 使用者登入 / 登出（UCDP001）

> 返回總檔：[spec.md](spec.md) | 模組：平台（DP）| Priority：P1 | Wireframe：[dp/index.html](../../wireframes/dp/index.html)（登入頁）

## User Story

作為 EDMS 使用者（ET / DM 共用帳號），我要以帳號（Email）+ 密碼登入系統、使用結束後登出，以便安全地存取兩模組授權範圍內的功能。

**Priority**: P1 — 全系統存取基礎；無登入機制無法進入任何功能。

**Independent Test**: 以有效帳密登入取得 JWT 並進入模組；錯誤密碼累計達上限（預設 5 次）後帳號自動鎖定；閒置 15 分鐘後 token 失效需重新登入；登出後前端 token 丟棄並留登出稽核。

### Acceptance Scenarios

1. **Given** 使用者於登入頁輸入正確帳號與密碼，**When** 點「登入」，**Then** 系統核發 JWT（token TTL＝閒置逾時，預設 15 分鐘）、重設失敗計數、寫入登入稽核（LOGIN）；自 ET / DM 被導來者返回原模組（由模組依角色導向預設首頁），無來源者顯示模組入口頁
2. **Given** 模組入口頁載入，**When** 使用者僅具 ET 學員角色，**Then** 顯示 ET 入口與個人資料維護，**不顯示** DM 入口與 DP 後台入口；具任一 DM 角色者加顯 DM 入口
3. **Given** 使用者輸入未註冊之 Email，**When** 點「登入」，**Then** 提示「查無此帳號，請先註冊」（DP-MSG-LOGIN-001）並提供註冊入口
4. **Given** 使用者輸入存在帳號但密碼錯誤，**When** 點「登入」，**Then** 失敗計數 +1 並提示（DP-MSG-LOGIN-002）
5. **Given** 連續失敗達設定次數（預設 5），**When** 再次登入失敗，**Then** 帳號自動鎖定、寫入稽核，提示（DP-MSG-LOGIN-003）
6. **Given** 帳號鎖定中（未逾自動解鎖時間，預設 30 分鐘）或已停用或閒置逾 90 日被禁用，**When** 以正確帳密登入，**Then** 拒絕登入並提示（DP-MSG-LOGIN-003 / 004）
7. **Given** 密碼已逾效期（預設 90 日）或以管理者代建之初始密碼登入，**When** 帳密驗證通過，**Then** 導向強制變更密碼頁（DP-MSG-LOGIN-005），未完成變更不得存取其他功能
8. **Given** 使用者已登入且持續操作，**When** 前端於 token 到期前發出請求，**Then** 系統靜默換發新 token；**When** 閒置逾 15 分鐘無操作，**Then** token 自然失效、自動登出並提示重新登入（DP-MSG-LOGIN-006）
9. **Given** 使用者自本次登入起算已連續換發達單日上限（預設 8 小時），**When** 再請求換發，**Then** 拒絕換發、需重新登入
10. **Given** 同一來源 IP + 帳號對登入端點高頻嘗試，**When** 超過速率限制，**Then** 暫時拒絕並提示（DP-MSG-LOGIN-007）
11. **Given** 使用者點「登出」，**When** 確認，**Then** 前端丟棄 token、寫入登出稽核（LOGOUT）、返回登入頁
12. **Given** 帳號於登入期間被管理者停用，**When** 其持有之 token 發出下次請求，**Then** 伺服器檢核帳號狀態並拒絕（ET / DM 兩端同步失效）

## Functional Requirements

- **FR-DP-US1-01**: 登入頁 MUST 提供帳號（Email）+ 密碼欄位；密碼輸入 MUST 遮蔽顯示；帳密 MUST 加密傳輸（HTTPS），不以明文傳輸
- **FR-DP-US1-02**: 系統 MUST 查 `DP_USER` 驗證帳密（密碼以不可逆雜湊比對），並區分「帳號不存在」與「密碼錯誤」兩種訊息
- **FR-DP-US1-03**: 驗證通過 MUST 核發 JWT access token——token TTL＝閒置逾時（平台級參數，預設 15 分鐘、不得超過 15 分鐘）；前端有操作即靜默換發；自本次登入起算之換發 MUST NOT 超過單日上限（平台級參數，預設 8 小時）；不採 MFA / Refresh Token / 伺服器端 session
- **FR-DP-US1-04**: 密碼錯誤 MUST 累計失敗計數；達設定次數（平台級參數，預設 5）MUST 自動鎖定帳號；成功登入 MUST 重設計數；鎖定後經管理者手動解鎖（US4）或逾時（平台級參數，預設 30 分鐘）自動解鎖
- **FR-DP-US1-05**: 鎖定中 / 已停用 / 閒置逾 90 日被禁用之帳號 MUST 拒絕登入並提示對應訊息
- **FR-DP-US1-06**: 密碼逾效期或以初始密碼（管理者代建、標記首次登入強制變更）登入者，MUST 於驗證通過後導向強制變更密碼頁；未完成變更 MUST NOT 允許存取其他功能
- **FR-DP-US1-07**: 登入後導向 MUST 依來源決定：自 ET / DM 導來者返回原模組；無來源者顯示模組入口頁——ET 入口恆顯示、DM 入口僅對具任一 DM 角色者顯示、個人資料維護恆顯示；模組入口頁 MUST NOT 顯示 DP 後台入口（後台入口僅設於 ET / DM 介面之管理選單）
- **FR-DP-US1-08**: 登入成功 / 失敗、登出、鎖定 / 解鎖 MUST 寫入 `DP_AUDIT_LOG`（LOGIN / LOGOUT 類別，含來源 IP、時間至秒）
- **FR-DP-US1-09**: 登入端點 MUST 實施伺服器端速率限制（以來源 IP + 帳號為維度）；不採 CAPTCHA
- **FR-DP-US1-10**: 登出＝前端丟棄 token 並寫入登出稽核；已停用帳號之未逾期 token MUST 於下次請求時被伺服器端拒絕
- **FR-DP-US1-11**: 除匿名功能（登入 / 註冊 / 忘記密碼）外，所有端點 MUST 通過身分驗證後方可存取

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DP-MSG-LOGIN-001 | 錯誤 | 查無此帳號，請先註冊 | FR-DP-US1-02 帳號不存在 |
| DP-MSG-LOGIN-002 | 錯誤 | 密碼錯誤 | FR-DP-US1-02 密碼錯誤 |
| DP-MSG-LOGIN-003 | 錯誤 | 帳號已鎖定，請洽管理者或稍後再試 | FR-DP-US1-04 / 05 鎖定 |
| DP-MSG-LOGIN-004 | 錯誤 | 帳號已停用，請洽管理者 | FR-DP-US1-05 停用 / 閒置禁用 |
| DP-MSG-LOGIN-005 | 警告 | 密碼已逾效期（或首次登入），請立即變更密碼 | FR-DP-US1-06 強制變更 |
| DP-MSG-LOGIN-006 | 提示 | 閒置逾時已自動登出，請重新登入 | FR-DP-US1-03 閒置失效 |
| DP-MSG-LOGIN-007 | 錯誤 | 嘗試次數過多，請稍後再試 | FR-DP-US1-09 速率限制 |

## 前置依賴

- `DP_USER` 帳號存在（US2 自助註冊 / US4 管理者代建）
- 密碼策略、鎖定次數 / 時間、token TTL / 換發上限等平台級參數（US5）
- 強制變更密碼頁之密碼檢核規則同 US8 密碼變更（複雜度 + 重複性）
- 登入後各模組依角色導向預設首頁——判定由 ET / DM 實作（跨模組）
