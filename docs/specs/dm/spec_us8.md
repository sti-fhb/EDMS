# User Story 8 — 文件廢止申請（UCDM05 / DM02）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

編輯者於文件詳細頁對已發布文件發起整份廢止申請，必填廢止原因並選指定審核者送交簽核。送審期間文件仍正常對外提供，直到審核者核准才轉已廢止。同一文件不可同時進行新版本與廢止兩種送審；撰寫者可於個人專區撤回。

**Priority**: P2 — 文件下架之正式途徑，屬完整生命週期之終態流程；MVP 可後於新增 / 簽核上線。

**Independent Test**: 編輯者於 DM02 點「廢止此文件」、填廢止原因、選審核者（排除自己）送出，文件轉「廢止待簽核」但仍顯示於文件庫且可下載；新版本送審中之文件無法發起廢止；後續經 [spec_us6.md](spec_us6.md) 核准轉已廢止、退回回已發布。

### Acceptance Scenarios

1. **Given** 編輯者於 DM02 檢視已發布文件，**When** 點「廢止此文件」，**Then** 開啟廢止申請對話框
2. **Given** 廢止對話框，**When** 未填廢止原因即送出，**Then** 系統阻擋並提示（DM-MSG-DM02-011）
3. **Given** 編輯者填妥廢止原因並選指定審核者，**When** 下拉渲染，**Then** 名單為具審核者角色之使用者且**排除自己**；送出後文件轉「廢止待簽核」並通知該審核者（DM-MSG-DM02-013）
4. **Given** 文件處於廢止待簽核，**When** 使用者於文件庫 / 詳細頁存取，**Then** 文件仍持續對外提供閱讀與下載（直到核准才下架）
5. **Given** 文件正進行新版本送審，**When** 編輯者嘗試發起廢止，**Then** 系統阻擋並提示（不可兩種送審並存）（DM-MSG-DM02-012）
6. **Given** 廢止申請送審後，**When** 審核者於 [spec_us6.md](spec_us6.md) 處理，**Then** 核准 → 已廢止；退回 → 回到已發布
7. **Given** 撰寫者欲取消廢止申請，**When** 至 [spec_us9.md](spec_us9.md) 個人專區撤回，**Then** 文件回到已發布

## Functional Requirements

- **FR-001**: 編輯者 MUST 可於 DM02 對「已發布」文件發起整份廢止；開啟對話框並**必填廢止原因**
- **FR-002**: 廢止申請 MUST 選一名指定審核者（具審核者角色且**排除撰寫者本人**）；送出後文件轉 PENDING_OBSOLETE 並通知該指定審核者
- **FR-003**: 廢止待簽核期間文件 MUST 仍正常對外提供（顯示於文件庫、可下載目前發布版本），直到核准後才轉「已廢止」並自文件庫下架
- **FR-004**: 文件正進行新版本送審時系統 MUST 阻擋發起廢止（同一文件不可同時兩種送審）
- **FR-005**: 後續核准 / 退回由 [spec_us6.md](spec_us6.md) 處理（核准 → 已廢止 + 版本歷程末尾新增廢止紀錄含申請人 / 申請時間 / 原因 / 核准者 / 廢止時間；退回 → 回已發布）；撰寫者可於 [spec_us9.md](spec_us9.md) 撤回廢止申請（回已發布）

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DM-MSG-DM02-011 | 錯誤 | 請填寫廢止原因 | FR-001 廢止原因必填 |
| DM-MSG-DM02-012 | 錯誤 | 此文件正進行新版本送審，無法同時發起廢止 | FR-004 送審週期衝突 |
| DM-MSG-DM02-013 | 成功 | 已送出廢止申請，已通知指定審核者 | FR-002 送出完成 |
| DM-MSG-DM02-014 | 錯誤 | 請選擇指定審核者 | FR-002 未選審核者 |

## 前置依賴

- 廢止對象為 [spec_us6.md](spec_us6.md) 已發布之文件；核准 / 退回於簽核中心處理
- 撤回廢止申請見 [spec_us9.md](spec_us9.md)；已廢止文件查閱見 [spec_us10.md](spec_us10.md)
