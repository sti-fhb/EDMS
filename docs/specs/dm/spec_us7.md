# User Story 7 — 系統儀表板（UCDM02 / DM00）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

使用者登入後落地於系統儀表板，快速掌握各類型文件總數與近期更新動態。上方呈現 4 類內建分類之已發布文件總數，下方列出近 30 天內已發布文件之公告，點擊可進入詳細頁。

**Priority**: P2 — 提升掌握度與導覽便利性，但非核心作業；缺此頁仍可由文件庫完成檢索與閱讀。

**Independent Test**: 使用者登入後自動導向 DM00，見四類統計卡與總計（僅計已發布目前版本），下方公告列出近 30 天發布事件；點公告進詳細頁、點「查看全部文件」進文件庫；無近期發布時顯示空狀態。

### Acceptance Scenarios

1. **Given** 使用者登入成功，**When** 完成驗證，**Then** 系統自動導向 DM00 系統儀表板（無 sidebar 入口；可由 home icon 或系統名稱返回本頁）
2. **Given** DM00 載入，**When** 渲染上方區塊，**Then** 顯示 4 張統計卡（SOP / 系統操作手冊 / 訓練教材 / 其他）之已發布目前版本數量與總計
3. **Given** 統計卡，**When** 計數，**Then** 僅計入「已發布」之目前版本，不含送審中 / 草稿 / 已被取代之舊版本 / 已廢止；卡片為純資訊、不可點擊
4. **Given** DM00 載入，**When** 渲染下方區塊，**Then** 列出近 30 天內已發布文件（含新增首版與新版本兩類事件），每筆顯示文件名 + 類型 badge（新增 / 新版本）+ 摘要 + 發布日期 + 撰寫者 + 分類
5. **Given** 使用者點任一公告，**When** 點擊，**Then** 進入該文件詳細頁（[spec_us4.md](spec_us4.md)）；點「查看全部文件」進文件庫（[spec_us3.md](spec_us3.md)）
6. **Given** 近 30 天無任何發布事件，**When** 渲染公告區，**Then** 顯示「近期無新發布文件」（DM-MSG-DM00-001）

## Functional Requirements

- **FR-001**: DM00 MUST 為登入後落地頁，所有可進入 DM 之使用者皆可瀏覽；本頁無 sidebar 入口，MUST 可由 home icon 或系統名稱返回
- **FR-002**: 上方「各類型文件總數」MUST 對應 4 內建分類，顯示各分類已發布目前版本數量與總計；計數 MUST 僅計入已發布目前版本（排除送審中 / 草稿 / 舊版 / 已廢止）；卡片為純資訊、不可點擊
- **FR-003**: 下方「最新更新公告」MUST 列出近 30 天內已發布文件（含新增首版與新版本兩類事件），每筆含文件名 / 類型 badge / 摘要 / 發布日期 / 撰寫者 / 分類
- **FR-004**: 點公告 MUST 進入 [spec_us4.md](spec_us4.md) 詳細頁；點「查看全部文件」MUST 進入 [spec_us3.md](spec_us3.md) 文件庫；近 30 天無事件時顯示空狀態提示
