## 模組定位

文件管理模組（DM）提供院方 **SOP、系統操作手冊、訓練教材、其他**等文件之集中儲存、版本管控、單層簽核發布與線上閱覽機制。本模組**獨立於主系統 TBMS 部署**，與教育訓練模組（ET）共用同一帳號系統與資料庫（SSO）；ET 之訓練教材統一引用 DM 提供之文件（依 DOC_ID 取最新發布版），避免教材與規範文件重複維護。

### 主要角色

| 角色 | 代碼 | 職責 |
|------|------|------|
| 管理者 | DM_ADMIN | 系統設定（分類 / func_name / 標籤 / 角色指派 / 通知範本 / 催辦門檻）、查詢已廢止文件（DM06）與文件變更歷程（DM08）|
| 編輯者 | DM_EDITOR | 新增文件、編輯新版本、上傳檔案、發起廢止申請、送交簽核（送審時必選一名指定審核者）|
| 審核者 | DM_REVIEWER | 於簽核中心處理指派給自己之送審項目（新增 / 新版本 / 廢止三類），核准或退回 |
| 閱覽者 | DM_VIEWER | 閱讀文件庫、下載目前發布版本、預覽舊版本（首次登入自動授予此角色）；**僅能看到與自己被授予之「可見對象/單位」標籤相符或掛「全體」之文件**（可見對象由管理者授予）|
| 系統自動 | — | 狀態流轉、版本切換、自動催辦、通知寄送、首次登入授予閱覽者、ET 端取最新版 |

> 四種角色可重複指派、互不互斥、權限取聯集；新使用者首次登入 DM 自動授予「閱覽者」角色（per 跨模組共用規則）。
>
> **術語**：「**編輯者**」指角色（DM_EDITOR）；「**撰寫者**」指發起某文件 / 版本送審之編輯者（即該版本作者）；「**作者**」即版本之 `CREATED_USER`。三者於本模組語意一致，文中視語境交替使用。

## Clarifications

### Session 2026-06-29（客戶會議變動）

- Q: 文件是否全部開放給閱覽者？ → A: 否。新增「可見對象/單位」權限標籤組（必填≥1、含「全體」）；閱覽者僅看得到掛「全體」或與其被授予可見對象相符（OR）之文件；管理者維護閱覽者授權（`DM_USER_TAG`）
- Q: 可見性用哪個標籤維度？ → A: 專用「可見對象/單位」組（與檢索標籤分離）；比對採 OR；原檢索組「適用角色」移除（併入可見對象）
- Q: 未掛標籤文件如何處理？ → A: 發布強制至少 1 個可見對象（demo 皆已具備，系統未上線無 migration）
- Q: 可見對象值可否停用？ → A: 保留停用但採 soft-retire（不收回既有可見性、僅擋後續指派、停用時提示受影響數）；不開放刪除
- Q: 版本號是否保留自動建議？ → A: 取消。一律撰寫者自行輸入（自由文字，如 2026v1.0）；系統不建議、無 Major / Minor；僅檢核非空且同文件不重複
- Q: 廢止原因可否附檔？ → A: 可。廢止申請之原因欄新增選填單檔附件（格式 / 大小比照文件上傳）；於簽核明細與 US10 已廢止查詢可下載
- Q: 文件發布是否通知閱覽者？ → A: 是。核准並發布（新增首版 / 新版本，廢止不含）時，**非同步 Email** 通知可見對象相符之閱覽者（掛「全體」則所有閱覽者，照實全寄）；只要具閱覽者角色且可見對象相符即通知（不排除兼具編輯 / 審核者）；僅 Email；收件名單為發布當下快照；為不阻塞發布交易，改呼叫平台唯一發信服務、經 outbox `DP_EMAIL_LOG` 由背景批次寄送
- Q: 「文件發布」與「發布通知閱覽者」兩範本是否合併？ → A: 合併為單一「**文件發布通知**」，收件對象＝**撰寫者 + 相符閱覽者**、**僅 Email、非同步**；撰寫者原有之站內發布通知取消（改由 US9 我的文件動態呈現）；通知範本由 10 → **9** 項（2026-06-29）
- Q: 是否統計閱讀 KPI？ → A: 是（US13）。下載目前發布版即「已看」（`DM_DOC_READ`，預覽不記）；應看＝可見對象相符之閱覽者（含「全體」、不排除任何人）；發新版重置；管理者 KPI 儀表板（DM10）逐文件呈現；範圍全部已發布文件
- Q: KPI 如何回報？ → A: 排程 `SCHDM001` 每週執行（星期＋時間可設定，預設週一 10:00）：KPI 週報寄管理者（Email，內文摘要 + CSV）、未讀提醒寄未看閱覽者（Email，一人一信彙整）；經平台發信服務 + outbox `DP_EMAIL_LOG` 非同步；新增 `KPI_WEEKLY` / `UNREAD_REMIND` 兩事件（存 `DP_NOTIFY_TEMPLATE` MODULE=DM）
- Q: 未讀提醒會不會太吵？如何控制？ → A: 未讀提醒涵蓋全部已發布文件；**是否寄送由管理者於 DM09 通知範本「未讀提醒」啟用 / 停用統一控制**（比照 ET；2026-07-02 移除原「文件層每週未看提醒旗標」，不再逐文件開關）

### Session 2026-06-24

- Q: DOC_ID 的格式要怎麼定？ → A: `DM-{分類碼}-{6 位流水號}`（如 `DM-SOP-000123`；分類碼於建立後唯讀、代碼鎖定，故識別碼永久穩定）
- Q: 一個「版本」對應幾個檔案？ → A: 單一版本＝單一檔案（上傳新檔即建立新版本；需多份內容請併為單一 PDF，不支援單一版本多附件）
- Q: 一個 func_name 可對應幾份系統操作手冊？ → A: 同一 func_name 至多對應一份「已發布」手冊（送審 / 發布時檢核，確保於 DM01 依作業項目檢索必得唯一）
- Q: DOC_ID 流水號採分類獨立或全域連續？ → A: 流水號**依分類各自獨立**起號（如 DM-SOP-000001、DM-MANUAL-000001 各自從 1 起）
- Q: 公開變更歷程 / 角色異動紀錄保留期？ → A: **永久保留**（與版本一致；「至少 1 年」為資安稽核法規下限）
- Q: 自訂分類的分類碼如何決定？ → A: 管理者新增分類時手動指定英數短碼（系統檢核唯一、建立後鎖定），供 DOC_ID 嵌入
- Q: 撰寫者可否選自己為指定審核者？ → A: 禁止（指定審核者下拉排除當前使用者，確保職責分離）

> 其他低影響項採合理預設（不另設問）：文件名稱允許重複（DOC_ID 為唯一鍵）；DOC_ID 於草稿建立時配號（容許跳號）；不檢核「至少 1 名管理者」（0 名時由 IT 經 DB 恢復）；自訂分類為平面、不分層。

## Requirements

### Functional Requirements

各 US 之 Functional Requirements 與 Acceptance Scenarios 置於對應 `spec_usN.md`。本節定義全模組共用之訊息類型與代碼格式。

**訊息類型定義**

各 US 之「系統訊息」表使用以下 5 種類型（與 BS / MA / ET 模組一致）：

| 類型 | 視覺（Bootstrap class）| UI 形式 | 是否需使用者決策 | 用途 |
|------|---------------------|--------|----------------|------|
| 錯誤 | 🔴 紅（`alert-danger`）| 對話框 / Inline 表單驗證 | 否（最多單一【關閉】）| 操作失敗或不允許，使用者必須修正後繼續 |
| 警告 | 🟡 黃（`alert-warning`）| 對話框 / Inline alert | 視情境（對話框需二選一決策、inline 純顯示需注意）| 系統判斷有風險或異常情況 |
| 確認 | ⚪ 中性（Bootstrap modal 白底）| 對話框 | 是（二選一）| 不可逆動作之最終防呆 |
| 成功 | 🟢 綠（`alert-success`）| Inline alert（顯示於按鈕附近）| 否 | 主動操作完成之回饋 |
| 提示 | 🔷 淺藍（`alert-info`）| Inline alert / 列表空白狀態提示 | 否 | 正常流程告知（查詢結果、彙總、引導）|

訊息代碼格式：`DM-MSG-{功能作業}-{流水號}`，例 `DM-MSG-DM03-001`。`{功能作業}` 原則用畫面 / 功能碼（DM00~DM10）；**登入等無對應 DMxx 畫面者以功能語意碼替代**（如 `LOGIN`）；**同一畫面承載多 UC 時共用畫面碼、以流水號號段區隔**（如 DM02 詳細頁瀏覽用 001~、廢止申請用 011~）。

### Key Entities

| 實體 | 業務定位 |
|------|---------|
| 共用使用者主檔（DP_USER）| 由平台模組 DP 定義（非 DM 自有）；DM 僅以 USER_ID（VARCHAR(20)、系統內穩定識別碼，永久不變）為 FK 引用；帳號（Email）/ 密碼雜湊 / 姓名等完整帳號欄位見平台 DP data-model；註冊一次即可登入兩系統 |
| DM 使用者角色（DM_USER_ROLE）| 使用者於 DM 之角色指派（管理者 / 編輯者 / 審核者 / 閱覽者）；同一使用者可多角色，權限取聯集 |
| 角色異動紀錄（DM_USER_ROLE_LOG）| 角色勾選 / 取消之完整歷史（append-only、永久保留、不可竄改）；供 IT / 稽核由 DB 查核，DM 不另設查詢介面 |
| 文件主檔（DM_DOCUMENT）| 每份文件之識別與身份屬性；含 DOC_ID（對外引用與檢索基準，格式 `DM-{分類碼}-{6 位流水號}`，如 `DM-SOP-000123`；流水號依分類各自獨立起號）、文件名稱、分類、關聯作業項目（func_name，僅系統操作手冊）、撰寫者、目前發布版本（CURRENT_VERSION_ID）、文件狀態 |
| 文件版本（DM_DOC_VERSION）| 文件之各版本；含版本號、變更摘要 / 首版摘要（同欄 CHANGE_SUMMARY）、作者、核准者、發布時間（即核准時間）、**單一檔案**（每版本恰好一個檔案）、版本狀態；所有版本永久保留（DELETED=0）|
| 文件分類（DM_CATEGORY）| 4 類內建（SOP / 系統操作手冊 / 訓練教材 / 其他）+ 管理者自訂（平面、不分層）；每類具唯一英數分類碼（內建固定、自訂由管理者指定且建立後鎖定，供 DOC_ID 嵌入）；可啟用 / 停用、不開放刪除 |
| 關聯作業項目（DM_FUNC）| func_name 受控清單（對應主系統作業功能代號，如 BS04）；系統操作手冊類文件可標記，供 DM01 依作業項目檢索 |
| 標籤組（DM_TAG_GROUP）| 4 內建組：**可見對象/單位（權限）** + 適用模組 / 文件性質 / 法規關聯（檢索）；原「適用角色」檢索組移除（併入可見對象/單位）|
| 標籤（DM_TAG）| 受控標籤庫；撰寫者只能挑選不可自由輸入；可啟用 / 停用（可見對象組之停用採 soft-retire）、不開放刪除 |
| 文件標籤關聯（DM_DOC_TAG）| 文件 × 標籤多對多；**可見對象組必填至少 1 項**（含通用值「全體」）、檢索組選填多選 AND |
| 閱覽者可見對象授權（DM_USER_TAG）| 使用者 × 「可見對象/單位」標籤之授權（多對多）；由管理者維護，決定閱覽者於文件庫之可見範圍；異動寫入稽核欄位 |
| 送審紀錄（DM_REVIEW）| 一次送審週期；含送審類型（新增 / 新版本 / 廢止）、指定審核者、核准者、送審時間、完成時間、退回原因 / 廢止原因、**廢止附件（選填、單檔，僅廢止類）**、狀態 |
| 公開變更歷程（DM_CHANGE_LOG）| 對外發布版本之實質變更事件（發布 / 廢止兩類）；不可竄改、不可刪除、**永久保留**（≥1 年法規下限）|
| 閱讀紀錄（DM_DOC_READ）| 使用者下載「目前發布版」之閱讀事件（使用者 × 文件 × 版本 × 時間）；作為閱讀 KPI「已看」判定；預覽不記錄；獨立於公開變更歷程 |
| 通知範本（`DP_NOTIFY_TEMPLATE` MODULE=DM）| DM 9 項內建事件之範本集中於平台 DP（「文件發布通知」發撰寫者+相符閱覽者、「KPI 週報」「未讀提醒」皆僅 Email、非同步）；事件固定不可新增，個別可啟用 / 停用、主旨 / 內文可編輯；**編輯 UI 仍在 DM09「通知範本」分頁**（DM 管理者只編輯 MODULE=DM 的列），完整欄位見平台 DP data-model |
| 寄件 outbox（平台 `DP_EMAIL_LOG`）| DM 之非同步寄送（文件發布通知 / KPI 週報 / 未讀提醒）改呼叫平台唯一發信服務（傳 template_code），由平台 outbox 非同步寄送並記錄狀態，使發布 / 排程不因大量寄信而阻塞；DM 不自建佇列表 |
| 系統參數（平台 `DP_PARAM` 前綴 `DM_`）| DM 參數集中於平台 DP；**自動催辦門檻**（`DM_REMIND_THRESHOLD`）、**每週排程時間**（`DM_WEEKLY_SCHED_DAY_TIME`）由管理者於 DM09 維護、**檔案大小上限 / 可上傳格式**（`DM_FILE_MAX_MB` / `DM_FILE_TYPES`）等由 IT 設定；平台提供唯讀查詢服務、維護 UI 仍在 DM09；發信引擎調校參數屬平台級 `DP_`（見〈跨模組共用規則〉） |

> 角色指派異動採稽核欄位記錄（異動者 / 異動時間，寫入異動紀錄）；DM 不另設權限變更查詢介面，完整歷史保留於 DB 供稽核（per §角色指派與異動紀錄）。各實體完整欄位定義於 `data-model.md`（待 `/speckit.plan` 階段產出）。

### 跨模組共用規則

- **帳號共用（SSO）**：DM 與 ET 共用 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名），使用者註冊一次即可登入兩系統；DM 端首次登入自動授予「閱覽者」角色。SSO 為登入方式而非獨立模組，與主系統 TBMS 各業務模組之帳號完全切開。**認證（登入 / 註冊 / 忘記密碼）由平台模組 DP 以簡單 JWT 統一實作（UCDP001–003、`DP_USER`），DM 不自建認證**
- **角色分治**：ET / DM 各自管理自己的角色（角色不共用、功能權限不共用）
- **多重角色**：角色（管理者 / 編輯者 / 審核者 / 閱覽者）可重複指派、互不互斥，權限取聯集
- **稽核欄位**：所有資料異動自動記錄建立者 / 異動者、建立時間 / 異動時間
- **文件編號**：每份文件具備唯一識別碼 DOC_ID（格式 `DM-{分類碼}-{6 位流水號}`，如 `DM-SOP-000123`，流水號依分類各自獨立），作為對外引用與檢索之基準；版號 / 名稱可變，DOC_ID 永久不變
- **歷史保留**：所有版本以邏輯方式永久保留（DELETED=0），不開放實體刪除，僅允許廢止整份文件
- **獨立部署**：本模組與 ET 同屬獨立部署，僅與 ET 共用帳號系統
- **系統參數 / 通知範本 / 排程 / 發信集中於平台 DP（2026-07-08 集中化）**：DM 不自持 `DM_PARAM` / `DM_NOTIFY_TEMPLATE` / `DM_NOTIFY_QUEUE`。系統參數存平台 `DP_PARAM`（`PARAM_ID` 前綴 `DM_`，平台提供唯讀查詢服務）、通知範本存 `DP_NOTIFY_TEMPLATE`（`MODULE=DM`）、非同步寄送改呼叫平台唯一發信服務並經 outbox `DP_EMAIL_LOG`、排程 `SCHDM001` 於 `DP_SCHEDULE` 註冊由平台引擎執行（`DP_SCHEDULE_LOG` 記錄、job handler 仍由 DM 提供）。**維護 / 編輯 UI 仍留 DM09**（DM 管理者只看 / 編輯 `MODULE=DM` / `DM_` 前綴的列，與 ET / 平台互不可見）；發信引擎調校參數（重試 / 速率 / 失敗告警）因發信引擎集中於平台，屬平台級 `DP_` 參數

### 文件狀態機

DM 同時處理三類送審：**新增 / 新版本 / 廢止**

```
[新增 / 新版本]
  草稿 (DRAFT)
    └─送審→ 送審中 (PENDING_REVIEW)
              ├─核准並發布→ 已發布 (PUBLISHED)
              └─退回→ 退回 (REJECTED) ─修改→ 草稿

[廢止]
  已發布 (PUBLISHED)
    └─發起廢止→ 廢止待簽核 (PENDING_OBSOLETE)
                  ├─核准廢止→ 已廢止 (OBSOLETE)
                  └─退回→ 已發布（回復原狀態）
```

- **廢止待簽核期間文件仍正常對外提供**（仍顯示於文件庫、仍可下載目前發布版本），直到核准後才轉「已廢止」並下架
- **同一文件不可同時進行兩種送審**：新版本送審中時不可同時發起廢止；廢止待簽核時不可同時編輯新版本（DM02 對應入口暫時失效）。此擋擋判定**以「該文件是否已存在進行中（PENDING）之送審週期」為準**（查 DM_REVIEW），非以文件狀態判定——新增 / 新版本 / 廢止送審皆會建立一筆 PENDING 送審週期

> **狀態以三個正交維度表達**：① 文件生命週期（DM_DOCUMENT.STATUS：PUBLISHED / PENDING_OBSOLETE / OBSOLETE）；② 版本自身結果（DM_DOC_VERSION.STATUS：DRAFT / PENDING_REVIEW / PUBLISHED / SUPERSEDED / REJECTED）；③ 進行中送審週期（DM_REVIEW）。
> - **已發布文件之新版本送審期間**：文件層維持 **PUBLISHED**（現行版仍在架可下載），新版本以版本層 PENDING_REVIEW + 一筆 PENDING 送審週期表示；文件層 PENDING_REVIEW **僅用於首版**（尚無目前發布版）送審。
> - **文件廢止後**：文件層轉 **OBSOLETE**（整份下架、僅 DM06 read-only 查），廢止前最後發布版之版本狀態**維持 PUBLISHED**（廢止屬文件層動作、該版未被新版取代故非 SUPERSEDED）；不可下載由 DM06 read-only 模式控制。

### 單層簽核與指定審核者

- 簽核流程**單層**：撰寫者 → 指定審核者 → 發布 / 廢止生效
- 送審（三類）時撰寫者**必選一名指定審核者**（具審核者角色之名單）；系統僅通知該指定審核者，不廣播
- 指定審核者下拉**排除當前使用者本人**：撰寫者不可選自己為審核者（即使兼具審核者角色），確保職責分離
- 簽核中心**僅顯示「指定審核者 = 當前登入者」之送審項目**；因本頁為審核者本人視角，清單不另列「指定審核者」欄
- 審核者可核准（發布 / 廢止）或退回；退回必填退回原因；核准動作須二次確認
- 送審紀錄記錄「指定審核者」（送審時寫入）與「核准者」（該指定審核者按核准 / 退回時自 Session 取 USER_ID，不可覆寫）

### 撤回送審機制

- 撰寫者可於個人專區（US9）對送審中項目（含廢止申請）執行「撤回送審」，適用於指定審核者請假 / 調職 / 離職 / 長期未處理等簽核停滯情況
- 撤回後文件狀態：新增 / 新版本送審 → 回到草稿；廢止申請 → 回到已發布
- 撤回後重送：原送審週期之指定審核者紀錄保留不被改寫；重新送審時記入新指定審核者
- 撤回僅以**系統內訊息**通知原指派審核者（不發 Email、不另設通知範本）

### 版本號規則

- 版本號**一律由撰寫者自行輸入**（自由文字，如 v1.0、2026v1.0、v2.0-RC1）；**系統不自動建議、無「Major / Minor」升版選項**（2026-06-29 客戶會議：取消原自動建議 / 大小號機制）
- 系統僅檢核：**非空**、且**同一文件內版本號不重複**（跨文件不限）
- 變更摘要欄位於新增首版時 UI 顯示為「首版摘要」、編輯新版本時顯示為「變更摘要」，DB 為同一欄位 CHANGE_SUMMARY

### 檔案預覽與下載規則

| 檔案格式 | 線上預覽 | 下載 |
|---------|---------|------|
| PDF | ✅ 內嵌瀏覽器預覽 | 目前發布版本可下載 |
| 圖片（PNG / JPG / GIF 等）| ✅ 內嵌瀏覽器預覽 | 目前發布版本可下載 |
| Office（Word / Excel / PPT）| ❌ 顯示「不支援預覽」，僅能下載原檔以本機應用程式開啟 | 目前發布版本可下載 |

- 上傳支援 PDF / Word / Excel / PowerPoint / 圖片，單檔上限預設 50 MB
- 上傳**不支援預覽之 Office 格式**時系統跳提醒視窗（PDF 與圖片不觸發）；撰寫者可改傳 PDF 或選「仍使用此檔案」繼續

### 受控標籤庫

- 採受控標籤庫（controlled vocabulary）；撰寫者只能從管理者維護之清單挑選，不允許自由輸入（避免同義詞碎裂影響搜尋與授權一致性）
- 4 內建標籤組，分**權限**與**檢索**兩用途：
  - **可見對象/單位（AUDIENCE，權限）**：預設值 護理師 / 軍人 / 醫檢師 / 行政人員 / **全體**；撰寫者於 DM03 **必填至少 1 項**、可多選；決定文件之可見範圍（見〈標籤式可見性〉）
  - **適用模組 / 文件性質 / 法規關聯（檢索）**：撰寫者跨組複選、**選填**；純供 DM01 搜尋，不影響可見性
- 原「適用角色」檢索組移除（角色語意併入「可見對象/單位」）
- DM01 列表 / DM02 詳細頁以**灰色文字頓號分隔**顯示（不用彩色 pill）

### 標籤式可見性（閱覽者存取控制）

- **可見對象/單位** 標籤同時作為閱覽者之存取控制依據：管理者於 US1 對每位使用者授予可見對象（`DM_USER_TAG`）
- **可見性規則**：某閱覽者看得到某文件 ⇔ 文件掛「**全體**」**或**（文件的可見對象 ∩ 該閱覽者被授予的可見對象 ≠ 空，**OR 比對**）
- 「**全體**」為文件端通用值（掛上＝所有閱覽者可見）；閱覽者端不需被授予「全體」；未被授予任何可見對象之閱覽者僅能看到掛「全體」之文件
- 可見性**只作用於閱覽者**；編輯者 / 審核者 / 管理者不受限（作業需要），且因權限取聯集，兼具其他角色之使用者亦見全部
- 可見對象標籤庫之維護沿用受控標籤庫機制，惟**停用採 soft-retire**：不收回既有文件與既有授權之可見性，僅停止後續指派，停用時系統提示受影響之文件 / 閱覽者數
- 檢索標籤（適用模組 / 文件性質 / 法規關聯）**不參與**可見性判斷

### 參數共通維護機制

分類、func_name、標籤組與標籤三類參數採相同維護模式：

- 可新增、編輯、**啟用 / 停用**；**不開放刪除**（避免既有文件失去引用），淘汰改為停用
- **停用後既有引用 100% 保留**：DB 既有文件欄位不變、DM01 / DM02 仍正常顯示原值；停用僅影響後續入口（DM03 新增 / 編輯下拉與 DM01 搜尋篩選只列啟用中）
- 內建項目可改名稱，代碼鎖定

### 閱讀統計與 KPI（2026-06-29 客戶會議）

- **閱讀追蹤**：使用者**下載「目前發布版」**時記一筆 `DM_DOC_READ`（使用者 × 文件 × 版本 × 時間）；**預覽不記**；此紀錄獨立於公開變更歷程，僅供 KPI
- **應看（分母）**：該文件可見對象相符之閱覽者（掛「全體」→ 所有閱覽者；OR 比對）；**與可見性 / 發布通知名單一致，不排除兼具編輯 / 審核 / 管理者者**
- **已看 / 未看 / 百分比**：已看＝分母中已下載目前發布版者；未看＝其餘；百分比＝已看 ÷ 應看
- **版本更新即重置**：發布新版本後 KPI 改以新版計算，原已看舊版者對新版視為未看
- **管理者儀表板（DM10）**：僅 DM_ADMIN；逐文件列 文件 / 分類 / 應看 / 已看 / 未看 / 百分比，可查詢排序；統計範圍為全部已發布文件
- **每週排程（`SCHDM001`，每週執行；星期＋時間可由管理者於通知範本設定，預設週一 10:00，存 `DP_PARAM.DM_WEEKLY_SCHED_DAY_TIME`；於平台 `DP_SCHEDULE` 註冊、平台引擎執行、`DP_SCHEDULE_LOG` 記錄，job handler 由 DM 提供；KPI 週報與未讀提醒共用）**：① **KPI 週報**寄管理者（Email；內文摘要〔總數 / 平均閱讀率 / 閱讀率最低前 N / 儀表板連結〕+ CSV 逐文件附件；涵蓋全部已發布文件）；② **未讀提醒**寄未看之閱覽者（Email；一人一信、彙整其所有未看文件；**涵蓋全部已發布文件**）
- **未讀提醒之統一控制**：是否寄送由管理者於 DM09 通知範本「未讀提醒」**啟用 / 停用**統一控制（比照 ET）；**不設文件層個別開關**（2026-07-02 移除原 `WEEKLY_REMIND_VIEWER` 旗標）
- 排程寄信皆經平台發信服務 + outbox `DP_EMAIL_LOG` **非同步批次**，不同步阻塞

### 公開變更歷程定義

- 公開變更歷程只記錄「對外發布版本」之實質變更，分兩類：**發布**（核准並發布，申請人 = 撰寫者、核准人 = 指定審核者、備註 = 變更摘要）、**廢止**（核准並廢止，申請人 = 廢止發起人、核准人 = 指定審核者、備註 = 廢止原因）
- 不可竄改、不可刪除（即使文件廢止亦保留）；**永久保留**（與版本一致；「至少 1 年」為資安稽核法規下限）；支援 CSV 匯出
- **撰寫過程動作**（上傳 / 編輯 / 送審 / 退回 / 撤回）為撰寫者個人視角，由 US9 個人專區呈現，**不寫入公開變更歷程**；**閱讀類動作不寫入公開變更歷程**（惟**下載**另記於 `DM_DOC_READ` 供閱讀 KPI，見〈閱讀統計與 KPI〉；**預覽仍不記錄**）
- **系統設定變更**（分類 / 角色 / 通知範本）不在本查詢頁顯示（僅記錄異動者 / 異動時間於各設定，per §角色指派與異動紀錄）

- **同一 func_name 至多對應一份「已發布」系統操作手冊**：送審 / 發布時檢核，若該 func_name 已有其他已發布手冊則阻擋發布，確保於 DM01 依作業項目檢索必得唯一手冊（草稿 / 已廢止不佔用 func_name）

> **2026-06-26 變更**：移除「主系統各畫面『線上系統操作手冊』按鈕以 func_name 跨系統反查 DM」之鏈路；DM 與主系統間**不再有此介接**。func_name 保留為 DM 內部欄位，僅供 DM01 依作業項目檢索。

### 角色指派與異動紀錄

- 管理者於 US1 權限管理對共用 `DP_USER` 之使用者勾選 / 取消 DM 角色，**即時生效**並**寫入異動紀錄**（記錄異動者 / 異動時間）
- **DM 不另設權限變更查詢介面**；DM09 權限管理清單以「最後異動」欄顯示最後一次變更，完整歷史永久保留於 DB 供 IT / 稽核查核
- **平台對齊（DP）**：角色異動屬資安類事件，對齊寫入平台共用稽核表 `DP_AUDIT_LOG`（DM 端不另設查詢 UI）；`DM_USER_ROLE_LOG` 仍為 DM 業務層角色指派歷史，維持不變
- 建置時系統預設指派 1–2 位初始管理者；之後管理者可彼此指派 / 取消管理者角色
- **當前登入之管理者無法停用自己之管理者角色**（自我保護）；管理者間可互相停用，系統**不檢核「至少保留 1 名管理者」**，若意外造成 0 名管理者由 IT 經 DB 恢復（比照 ET）
- 離職 / 調職由共用 `DP_USER` 之管理介面停用該帳號即可，DM 不另外處理

### 帳號（Email）變更流程

> **平台對齊（DP）**：帳號 / 個人資料維護（姓名 / Email / 密碼）操作共用 `DP_USER`，已統一由平台模組 DP 提供（UCDP004）；本節保留為情境描述，實作與帳號實體以平台為準。

使用者於 US9 變更帳號（Email）採「新 Email 驗證後切換」之延遲生效機制：

- 提交變更後系統寄驗證信至**新 Email**（30 分鐘內有效）；點驗證連結後新帳號才生效、舊帳號失效
- 驗證信逾時未點 → 新帳號不生效，舊帳號維持有效
- 因與 ET 共用 `DP_USER`，變更後 ET 端同步生效；密碼變更須提供舊密碼 + 新密碼 + 確認新密碼

### 個人專區分區可見性

- **個人資料維護**：所有 4 角色可用
- **文件草稿 / 撤回送審**：編輯者
- **我的文件動態**：編輯者（撰寫者視角）/ 審核者（審核者視角），同時具兩角色者兩 tab 皆顯示
- **純閱覽者 / 純管理者**（未兼具編輯者或審核者角色）僅看得到「個人資料維護」分區

## Success Criteria

### Measurable Outcomes

- **SC-001**: 文件庫檢索（US3）載入時間 P95 ≤ 2 秒（含關鍵字 / 分類 / 作者 / 標籤 AND / 日期 / func_name 條件）
- **SC-002**: PDF 與圖片於文件詳細頁（US4）以內嵌瀏覽器預覽，閱讀者不需安裝外部軟體；Office 檔提供下載
- **SC-003**: 送審項目停留 ≥ 催辦門檻天數時，系統每日自動以系統內訊息催辦指定審核者，並於 DM04 / DM07 標紅警示
- **SC-004**: 簽核核准後（US6）即時完成版本切換（新版本轉已發布、舊版自動標記已被取代）或整份文件廢止，並通知撰寫者、寫入公開變更歷程
- **SC-005**: 角色變更（US1）寫入後即時生效並寫入異動紀錄；下次使用者操作依新角色 / 新權限生效
- **SC-006**: DM 文件發布新版本後（US12），ET 端引用該 DOC_ID 之教材於下次開啟時即取得最新發布版（無快取延遲）
- **SC-007**: 密碼重設連結（US2）30 分鐘內有效，逾時拒絕並提示重新申請
- **SC-008**: 帳號（Email）變更（US9）須透過寄至新 Email 之驗證信點擊後方生效，避免誤改鎖死帳號
- **SC-009**: 已廢止文件（US10）不出現於文件庫主清單，且所有版本僅可預覽、不開放下載
- **SC-010**: 閱覽者於文件庫（US3）僅看得到掛「全體」或與其被授予之可見對象相符（OR）之已發布文件；未授予任何可見對象者僅見「全體」文件；編輯者 / 審核者 / 管理者不受限
- **SC-011**: 使用者下載目前發布版即記入 `DM_DOC_READ`；管理者 KPI 儀表板（US13）逐文件正確呈現應看 / 已看 / 未看 / 百分比（發新版後重置）；`SCHDM001` 於管理者設定之每週時間（預設週一 10:00）寄出 KPI 週報（管理者）與未讀提醒（未看閱覽者，涵蓋全部已發布文件、由管理者以範本啟用/停用統一控制、執行時間亦由範本設定），大量寄信經平台 outbox `DP_EMAIL_LOG` 非同步不阻塞

## Assumptions

- **SSO 範圍與權責切分**：DM 以 SSO 方式與 ET 共用 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名）；但 ET / DM 各自管理自己的角色（角色不共用、功能權限不共用）。SSO 為登入方式而非獨立模組，無獨立模組編碼；**認證能力由平台模組 DP 以簡單 JWT 統一提供**。與主系統 TBMS 各業務模組之帳號完全切開（per 2026-04-28 設計決策）
- **文件分類**：預設 4 類（SOP / 系統操作手冊 / 訓練教材 / 其他），管理者可新增自訂分類（平面、不分層）；每類具唯一英數分類碼供 DOC_ID 嵌入（內建固定、自訂由管理者指定且鎖定）
- **文件名稱可重複**：DOC_ID 為唯一鍵，文件名稱不強制唯一（允許不同文件同名）
- **DOC_ID 配號時機**：於草稿建立時即配號（分類此時已選定）；草稿刪除造成之號碼跳號可接受
- **檔案上傳**：每個版本單一檔案；支援 PDF / Word / Excel / PowerPoint / 圖片，單檔大小上限預設 50 MB；可預覽格式為 PDF 與圖片，Office 格式僅下載。格式與大小上限由 **DM 系統參數**（`DM_FILE_MAX_MB` / `DM_FILE_TYPES`，存平台 `DP_PARAM` 前綴 `DM_`）控制，維護 UI 在 DM09 / IT 可調整；與 TBMS 主系統各業務模組無介接
- **外部文件報表範圍**：交付確認書將「外部文件報表與儀表板」界定為**系統儀表板（US7）+ 文件變更歷程查詢（US11）+ 閱讀統計 KPI（US13，2026-06-29 客戶會議新增）**；仍不含分類分佈 / 廢止統計 / 簽核耗時 / 撰寫者-審核者活躍度等其他統計報表
- **權限變更稽核**：交付確認書要求「角色勾選即時生效並寫入異動紀錄」；本 spec 採「寫 DB、不提供 DM 查詢 UI」，完整歷史供 IT / 稽核查核
- **個資處理**：使用者 Email 與姓名為一般個資，不需加密儲存；密碼採雜湊儲存
- **跨模組互動細節**：請求 / 回應格式、錯誤碼由 `specs/dm/contracts/` 規範（待 `/speckit.plan` 階段產出）
- **數量 / 長度上限延遲決定**：文件 / 版本 / 標籤等之數量上限，以及文件名稱 / 變更摘要 / 廢止原因等長度上限，於 `/speckit.plan` 與 data-model 階段依效能與業務情境決定，spec 階段不預先綁定
