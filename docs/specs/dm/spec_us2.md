# User Story 2 — 登入 / 註冊 / 忘記密碼（UCDM01）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

> **平台對齊（DP）**：**登入 / 註冊 / 忘記密碼由平台模組 DP 統一提供（UCDP001–003、單一登入、簡單 JWT、`DP_USER`），DM 不自設登入畫面**——登入頁與認證流程皆為平台 DP 畫面，使用者一次登入即可存取 ET / DM 兩系統。本 US 原保留之 DM 專屬規則「首次進入 DM 自動授予閱覽者」**已作廢**（2026-07-08 平台釐清：新帳號預設僅 **ET 學員**、DM 全部四角色〔含閱覽者〕一律由管理者於平台 DP 後台權限管理開通，見 FR-003）；帳號 / 密碼 / 姓名之變更於 [spec_us9.md](spec_us9.md)（亦由平台 DP 提供）。下述情境 / FR 中屬純登入 UI 者為 DP 提供之行為描述、DM 不實作，僅「無 DM 角色者拒絕進入」為 DM 實作。

## User Story

使用者於登入頁以帳號（Email）+ 密碼登入 DM；未有帳號者可自行註冊（跳回登入頁重新登入；不自動授予 DM 角色，DM 存取由管理者開通）；忘記密碼者可申請重設連結。DM 與 ET 共用同一 `DP_USER`，使用者註冊一次即可登入兩系統。

**Priority**: P1 — 登入為所有 DM 作業之存取基礎；無登入機制無法進入任何功能。

**Independent Test**: 以既有帳密登入成功並導向 DM00 儀表板；以未註冊 Email 登入得到「查無此帳號」提示；完成註冊後以新帳號登入；申請忘記密碼並於連結有效期內重設密碼後登入。

### Acceptance Scenarios

1. **Given** 使用者於登入頁輸入正確帳號與密碼，**When** 點「登入」，**Then** 系統驗證通過、寫入 session；具任一 DM 角色者導向 DM00 系統儀表板，無任何 DM 角色者拒絕進入 DM 並提示洽管理者開通（DM-MSG-LOGIN-007，無自動授予）
2. **Given** 使用者輸入未註冊之 Email，**When** 點「登入」，**Then** 系統提示「查無此帳號，請先註冊」（DM-MSG-LOGIN-001）並提供註冊入口
3. **Given** 使用者輸入存在帳號但密碼錯誤，**When** 點「登入」，**Then** 系統提示「密碼錯誤」（DM-MSG-LOGIN-002）
4. **Given** 使用者切換至「註冊」頁籤並填寫 Email / 密碼 / 確認密碼 / 姓名，**When** Email 未被註冊且兩次密碼一致，**Then** 系統建立共用 `DP_USER` 紀錄（預設角色僅 ET 學員、由平台 DP 授予；不授予 DM 角色）、提示註冊成功並跳回登入頁預填 Email（DM-MSG-LOGIN-003）
5. **Given** 註冊時 Email 已存在，**When** 送出，**Then** 系統阻擋並提示（DM-MSG-LOGIN-004）
6. **Given** 註冊時兩次密碼不一致，**When** 送出，**Then** 系統阻擋並提示（DM-MSG-LOGIN-005）
7. **Given** 使用者點「忘記密碼」並輸入註冊 Email，**When** 送出，**Then** 系統寄發密碼重設連結至該 Email（30 分鐘內有效）
8. **Given** 使用者點重設連結，**When** 連結未逾時，**Then** 進入重設密碼頁、輸入新密碼後儲存生效；**When** 連結逾時，**Then** 提示「連結已失效，請重新申請」（DM-MSG-LOGIN-006）

## Functional Requirements

- **FR-001**: 系統 MUST 於登入頁提供「登入」與「註冊」兩入口；登入以帳號（Email）+ 密碼驗證共用 `DP_USER`
- **FR-002**: 登入驗證 MUST 區分兩種失敗並給明確訊息：帳號不存在（DM-MSG-LOGIN-001）、密碼錯誤（DM-MSG-LOGIN-002）；密碼以雜湊比對（不可逆儲存）
- **FR-003**: 登入成功 MUST 取得該使用者之 DM 角色清單；具任一 DM 角色者導向 DM00 系統儀表板，無任何 DM 角色者 MUST 拒絕進入 DM 並提示洽管理者開通（MUST NOT 自動授予；2026-07-08 作廢原「自動授予閱覽者」，DM 角色一律由管理者於平台 DP 後台權限管理開通）
- **FR-004**: 註冊 MUST 檢核 Email 未被註冊、兩次密碼一致；通過後建立 `DP_USER` 紀錄（不授予 DM 角色；預設僅 ET 學員、由平台 DP 授予），並跳回登入頁請使用者以新帳號重新登入（驗證密碼可用）
- **FR-005**: 忘記密碼 MUST 寄發密碼重設連結至註冊 Email，連結 **30 分鐘**內有效；逾時連結 MUST 拒絕並提示重新申請（認證由平台模組 DP 提供，重設 token 有效時間為**平台級 `DP_` 參數**、重設信範本為 `DP_NOTIFY_TEMPLATE` MODULE=DP，皆於平台 DP 系統管理後台維護〔平台級 `DP_` 參數與 MODULE=DP 系統信為共用項，ET / DM 管理者皆可維護，系統信可編輯主旨 / 內文但不可停用〕）
- **FR-006**: 帳號（Email）/ 密碼 / 姓名變更於 [spec_us9.md](spec_us9.md) 個人專區處理；因與 ET 共用 `DP_USER`，變更於兩系統同步生效

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DM-MSG-LOGIN-001 | 錯誤 | 查無此帳號，請先註冊 | FR-002 帳號不存在 |
| DM-MSG-LOGIN-002 | 錯誤 | 密碼錯誤 | FR-002 密碼錯誤 |
| DM-MSG-LOGIN-003 | 成功 | 註冊成功，請以新帳號登入 | FR-004 註冊完成 |
| DM-MSG-LOGIN-004 | 錯誤 | 此 Email 已被註冊 | FR-004 Email 重複 |
| DM-MSG-LOGIN-005 | 錯誤 | 兩次輸入之密碼不一致 | FR-004 密碼不一致 |
| DM-MSG-LOGIN-006 | 錯誤 | 連結已失效，請重新申請 | FR-005 重設連結逾時 |
| DM-MSG-LOGIN-007 | 提示 | 尚未開通文件管理權限，請洽管理者 | FR-003 無任何 DM 角色 |

- 登入成功導向之 DM00 儀表板見 [spec_us7.md](spec_us7.md)
