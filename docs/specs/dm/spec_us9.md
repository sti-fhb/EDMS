# User Story 9 — 個人專區（UCDM09 / DM07）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

> **平台對齊（DP）**：本頁『個人資料維護』（姓名 / 帳號 Email / 密碼）操作共用 `DP_USER`，已統一由平台 DP 提供（UCDP004），經右上使用者選單進入；本區塊保留為情境描述、實作以平台為準。文件草稿 / 我的文件動態 / 撤回送審為 DM 業務，維持不變。

## User Story

使用者於個人專區管理個人資料（帳號 Email / 姓名 / 密碼，所有角色）、文件草稿與撤回送審（編輯者）、我的文件動態（編輯者 / 審核者）。帳號變更採新 Email 驗證後切換；密碼變更須驗舊密碼。純閱覽者 / 純管理者僅看得到個人資料維護分區。

**Priority**: P2 — 個人資料維護為所有使用者共用；草稿與撤回送審為編輯者處理卡關送審之重要手段，但非首發必要。

**Independent Test**: 使用者於個人專區改姓名直接儲存、變更 Email 後收驗證信點擊生效、變更密碼需驗舊密碼；編輯者於草稿區續編 / 刪除草稿、對卡住送審項目撤回（回草稿 / 已發布）；編輯者 / 審核者於我的文件動態見近 30 天事件；純閱覽者僅見個人資料維護。

### Acceptance Scenarios

1. **Given** 任一角色使用者進入個人專區，**When** 載入，**Then** 顯示「個人資料維護」分區（帳號 Email / 姓名 / 密碼變更入口）
2. **Given** 使用者編輯姓名，**When** 儲存，**Then** 直接生效
3. **Given** 使用者變更帳號（Email），**When** 按「變更」，**Then** 系統寄驗證信至**新 Email**（30 分鐘內有效）；點連結後新帳號生效、舊帳號失效；逾時未點則新帳號不生效、舊帳號維持（DM-MSG-DM07-002 / 003）
4. **Given** 使用者變更密碼，**When** 填舊密碼 + 新密碼 + 確認新密碼且舊密碼正確，**Then** 儲存成功並同步更新共用 `DP_USER`（ET 端亦生效）；舊密碼錯誤則提示（DM-MSG-DM07-001）
5. **Given** 編輯者進入個人專區，**When** 載入，**Then** 顯示「文件草稿」分區，列未送審草稿 / 被退回待修改 / 已撤回送審三類
6. **Given** 編輯者於草稿區，**When** 按「繼續編輯 / 修改」進 [spec_us5.md](spec_us5.md)；按「刪除」，**Then** 跳確認後刪除（不可復原）（DM-MSG-DM07-004）
7. **Given** 編輯者對卡住之送審項目按「撤回送審」，**When** 確認，**Then** 新增 / 新版本送審 → 回草稿；廢止申請 → 回已發布；以系統內訊息通知原指派審核者；可重新編輯並改選新審核者再送（DM-MSG-DM07-005）
8. **Given** 編輯者 / 審核者進入「我的文件動態」，**When** 載入，**Then** 依角色 tab 呈現近 30 天事件（撰寫者視角：送審中 / 核准發布 / 退回 / 廢止待簽核 / 已廢止 / 已撤回；審核者視角：待處理 / 催辦中 / 已被撰寫者撤回 / 已處理歷史）
9. **Given** 純閱覽者 / 純管理者（未兼具編輯者或審核者角色），**When** 進入個人專區，**Then** 僅看得到「個人資料維護」分區；文件草稿與我的文件動態不顯示

## Functional Requirements

- **FR-001**: 個人資料維護 MUST 對所有 4 角色開放，可檢視 / 編輯帳號（Email）、姓名、密碼
- **FR-002**: 帳號（Email）變更 MUST 採延遲生效：寄驗證信至新 Email（30 分鐘內有效），點連結後新帳號生效、舊帳號失效；PENDING 期間舊 Email 仍可登入；逾時未點則新帳號不生效、舊帳號維持（帳號 / 個資維護與此驗證信由平台模組 DP 提供，有效時間為**平台級 `DP_` 參數**、驗證信範本為 `DP_NOTIFY_TEMPLATE` MODULE=DP）
- **FR-003**: 密碼變更 MUST 要求舊密碼 + 新密碼 + 確認新密碼；舊密碼錯誤 MUST 阻擋；成功後同步更新共用 `DP_USER`（ET 端生效）；忘記密碼改走 [spec_us2.md](spec_us2.md) 登入頁
- **FR-004**: 文件草稿分區（編輯者）MUST 列出未送審草稿 / 被退回待修改 / 已撤回送審三類；提供「繼續編輯 / 修改」（→ [spec_us5.md](spec_us5.md)）與「刪除」（須確認、不可復原、僅影響草稿不影響已發布版本）
- **FR-005**: 撤回送審（編輯者）MUST 將送審中項目回復狀態（新增 / 新版本 → 草稿；廢止 → 已發布），以**系統內訊息**通知原指派審核者（不發 Email），並允許重新編輯與改選新審核者再送；原送審週期之指定審核者紀錄保留不被改寫
- **FR-006**: 我的文件動態 MUST 依角色 tab 呈現近 30 天事件（撰寫者視角 / 審核者視角）；同時具兩角色者兩 tab 皆顯示
- **FR-007**: 個人專區可見性 MUST 依角色：純閱覽者 / 純管理者（未兼編輯者 / 審核者）僅顯示個人資料維護；文件草稿與我的文件動態僅對編輯者 / 審核者顯示對應內容

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DM-MSG-DM07-001 | 錯誤 | 舊密碼不正確 | FR-003 舊密碼錯誤 |
| DM-MSG-DM07-002 | 成功 | 驗證信已寄至新 Email，點擊連結後新帳號才生效 | FR-002 Email 變更送出 |
| DM-MSG-DM07-003 | 警告 | 驗證連結已逾時，帳號未變更 | FR-002 驗證逾時 |
| DM-MSG-DM07-004 | 確認 | 確定刪除此草稿？刪除後不可復原 | FR-004 刪除草稿 |
| DM-MSG-DM07-005 | 成功 | 已撤回送審，已通知原指派審核者 | FR-005 撤回送審 |
