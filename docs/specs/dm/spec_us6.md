# User Story 6 — 簽核處理（UCDM07 / DM04）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

審核者於簽核中心處理指派給自己之送審項目（新增 / 新版本 / 廢止三類），檢視送審檔案與變更摘要（或廢止原因）後核准或退回。核准發布即建立新版本並轉已發布；核准廢止即轉已廢止；退回則回對應狀態並通知撰寫者。清單僅顯示指派給當前登入者之項目。

**Priority**: P1 — 簽核為文件對外生效（發布 / 廢止）之唯一閘門；無簽核則文件無法發布、廢止無法生效。

**Independent Test**: 審核者登入後於簽核中心「待簽核」頁籤見指派給自己之項目，點開明細下載送審檔案，按「核准並發布」使文件轉已發布並寫入變更歷程；按「退回」填原因後文件回草稿 / 已發布並通知撰寫者；「已完成」頁籤可回顧處理歷史。

### Acceptance Scenarios

1. **Given** 審核者已登入，**When** 進入簽核中心，**Then** 預設「待簽核」頁籤列出**指派給自己**之送審項目（新增 / 新版本 / 廢止），欄位含文件名稱 / 分類 / 送審版本 / 送審者 / 送審時間 / 停留天數（**不列「指定審核者」欄**，因本頁皆為自己之項目）
2. **Given** 審核者點任一筆，**When** 展開明細，**Then** 顯示變更摘要（或廢止原因）與送審檔案資訊，**可下載**檔案查看（新版本申請另提供新舊版並列下載比對）；不提供線上預覽
3. **Given** 審核者按「核准並發布」（新增 / 新版本），**When** 二次確認後執行，**Then** 系統建立新版本並轉「已發布」、舊版自動標記已被取代，寫入公開變更歷程（DM-MSG-DM04-001），**並以單一「文件發布通知」非同步 Email 通知撰寫者 + 該文件可見對象相符之閱覽者**（掛「全體」則所有閱覽者）
3a. **Given** 文件核准並發布，**When** 系統組出收件名單，**Then** 名單為**撰寫者 + 發布當下所有具閱覽者角色且可見對象相符（或掛「全體」）之使用者**（不排除兼具編輯 / 審核者）；收件人逐筆入 `DM_NOTIFY_QUEUE` 由背景批次寄送（僅 Email），不阻塞核准交易；**廢止不觸發此通知**
4. **Given** 審核者按「核准並廢止」（廢止申請），**When** 二次確認後執行，**Then** 文件轉「已廢止」、自文件庫下架、版本歷程末尾新增廢止紀錄，通知撰寫者並寫入公開變更歷程（DM-MSG-DM04-002）
5. **Given** 審核者按「退回」，**When** 開啟對話框，**Then** 必填退回原因；確認後文件轉回對應狀態（新增 / 新版本 → 草稿；廢止 → 已發布）並通知撰寫者（DM-MSG-DM04-003 / 004）
6. **Given** 送審項目停留 ≥ 催辦門檻天數，**When** 每日批次觸發，**Then** 系統以系統內訊息催辦該指定審核者，清單同步標紅警示已停留天數
7. **Given** 撰寫者已撤回某送審，**When** 審核者檢視，**Then** 該項目自待簽核清單消失；審核者於 [spec_us9.md](spec_us9.md) 我的文件動態可見「已被撰寫者撤回」事件
8. **Given** 審核者切至「已完成」頁籤，**When** 載入，**Then** 呈現自己過往處理（已核准 / 已退回）之項目、提供搜尋分頁，且不可再操作

## Functional Requirements

- **FR-001**: 簽核中心 MUST 僅顯示「指定審核者 = 當前登入者」之送審項目；待簽核清單欄位為文件名稱 / 分類 / 送審版本 / 送審者 / 送審時間 / 停留天數，**不列「指定審核者」欄**
- **FR-002**: 明細 MUST 提供送審檔案之**下載**（不提供線上預覽）；新版本申請 MUST 提供新舊版並列下載比對；廢止申請顯示廢止對象（目前發布版）與廢止原因，並（如有）提供**廢止附件下載**
- **FR-003**: 核准動作 MUST 二次確認；「核准並發布」MUST 原子完成版本切換（新版本轉已發布、舊版標記已被取代），「核准並廢止」MUST 將文件轉已廢止並自文件庫下架、版本歷程末尾新增廢止紀錄
- **FR-004**: 「退回」MUST 必填退回原因；新增 / 新版本退回 → 文件回草稿（出現於撰寫者個人專區草稿區）；廢止退回 → 文件回已發布
- **FR-005**: 退回 / 廢止核准 / 廢止退回結果 MUST 通知撰寫者；發布結果由 FR-008 之「文件發布通知」一併通知撰寫者（不另發站內）；發布 / 廢止 MUST 寫入公開變更歷程（永久保留、不可竄改）；核准者 MUST 自登入 Session 取 USER_ID、不可覆寫
- **FR-006**: 送審項目停留 ≥ 催辦門檻天數時系統 MUST 每日自動以系統內訊息催辦指定審核者，並於清單標紅警示停留天數
- **FR-007**: 「已完成」頁籤 MUST 呈現當前審核者過往處理（已核准 / 已退回）之項目，提供搜尋分頁；已完成項目 MUST NOT 可再操作
- **FR-008**: 核准並發布（新增首版 / 新版本）時系統 MUST 於**發布當下**組出收件名單——**撰寫者 + 所有具閱覽者角色且該文件可見對象相符（或掛「全體」）之使用者**（不排除兼具編輯 / 審核者、不追溯後續授權）——並將單一「**文件發布通知**」（`DOC_PUBLISH`）逐筆寫入 `DM_NOTIFY_QUEUE`，由**背景批次非同步僅以 Email 寄送**，不阻塞核准發布交易；該範本停用時不寄；**廢止 MUST NOT 觸發此通知**

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DM-MSG-DM04-001 | 成功 | 已核准並發布，已通知撰寫者 | FR-003 核准發布 |
| DM-MSG-DM04-002 | 成功 | 已核准並廢止，文件已自文件庫下架 | FR-003 核准廢止 |
| DM-MSG-DM04-003 | 確認 | 確定核准此項目？此動作將立即生效 | FR-003 二次確認 |
| DM-MSG-DM04-004 | 錯誤 | 請填寫退回原因 | FR-004 退回必填 |
| DM-MSG-DM04-005 | 成功 | 已退回並通知撰寫者 | FR-004 退回完成 |
| DM-MSG-DM04-006 | 提示 | 此項目已被撰寫者撤回，已自清單移除 | FR-006 撰寫者撤回 |
