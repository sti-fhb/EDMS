# User Story 1 — 系統設定（UCDM11 / DM09）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

管理者於系統設定（DM09）維護全模組可配置項目，含三分區：**參數設定**（文件分類、關聯作業項目 func_name、受控標籤庫、自動催辦門檻）、**權限管理**（對共用 user table 之使用者指派 DM 四角色）、**通知範本**（7 項內建事件之主旨 / 內文與啟用停用）。角色指派即時生效並寫入異動紀錄；分類 / func_name / 標籤採共通維護機制（不開放刪除、淘汰改停用、停用後既有引用保留）。

**Priority**: P1 — 權限管理為編輯者 / 審核者取得作業權限之入口；參數（分類 / func_name / 標籤 / 催辦門檻）為其他作業之基礎設定，無此設定文件無法正確分類、簽核無法催辦。

**Independent Test**: 管理者登入後進入 DM09，於參數分頁新增一個自訂分類（指定唯一分類碼）、於權限分頁對某使用者勾選「編輯者」並儲存、於通知範本分頁調整催辦門檻為 10 天；該使用者下次操作即具編輯權限，新分類出現於 DM03 下拉。

### Acceptance Scenarios

1. **Given** 管理者已登入，**When** 進入 DM09 參數設定分頁，**Then** 系統列出文件分類（4 內建 + 自訂）、關聯作業項目（func_name）、標籤組與標籤，各項皆可新增 / 改名 / 啟用 / 停用，且**無刪除**功能
2. **Given** 管理者新增自訂分類，**When** 未填分類碼或填入已存在之分類碼，**Then** 系統阻擋並提示（DM-MSG-DM09-001 / 002）
3. **Given** 管理者新增自訂分類並填入唯一英數分類碼與名稱，**When** 儲存，**Then** 分類建立成功、**分類碼建立後鎖定**（不可再改，僅可改名），供 DOC_ID 嵌入
4. **Given** 管理者停用某分類 / 標籤 / func_name，**When** 儲存，**Then** 既有文件之標記不受影響，但 DM03 新增 / 編輯與 DM01 搜尋之下拉不再顯示該項（DM-MSG-DM09-003）
5. **Given** 管理者設定自動催辦門檻，**When** 輸入值落於 1–30 天，**Then** 儲存成功；輸入 0 或 > 30，**Then** 阻擋並提示（DM-MSG-DM09-004）
6. **Given** 管理者進入權限管理分頁，**When** 系統載入，**Then** 列出共用 user table 之使用者，每人顯示 4 角色（管理者 / 編輯者 / 審核者 / 閱覽者）核取方塊與「最後異動」欄
7. **Given** 管理者對某使用者勾選 / 取消角色，**When** 儲存，**Then** 即時生效並寫入異動紀錄（異動者 / 異動時間），下次該使用者操作依新權限生效（DM-MSG-DM09-005）
8. **Given** 當前登入之管理者取消**自己**之管理者角色，**When** 儲存，**Then** 系統阻擋並提示（DM-MSG-DM09-006）
9. **Given** 管理者 A 取消管理者 B 之管理者角色，**When** 儲存，**Then** 系統允許（不檢核「至少保留 1 名管理者」）
10. **Given** 管理者進入通知範本分頁，**When** 系統載入，**Then** 列出 7 項內建事件（6 工作流 + 1 自動催辦），各可編輯主旨 / 內文、啟用 / 停用；自動催辦另含門檻天數設定
11. **Given** 新使用者首次登入 DM，**When** 管理者檢視權限清單，**Then** 該使用者已存在且預設具「閱覽者」角色（系統自動授予，毋須管理者勾選）

## Functional Requirements

- **FR-001**: 系統 MUST 於參數設定維護文件分類（4 內建：SOP / 系統操作手冊 / 訓練教材 / 其他 + 管理者自訂）、關聯作業項目（func_name）、標籤組與標籤；三類採**共通維護機制**：可新增 / 編輯 / 啟用 / 停用，**不開放刪除**（淘汰改停用），停用後既有文件引用 100% 保留、僅影響後續新增 / 編輯與搜尋之下拉選項
- **FR-002**: 新增自訂分類時系統 MUST 要求唯一英數**分類碼**（檢核唯一、建立後鎖定，供 DOC_ID `DM-{分類碼}-{流水號}` 嵌入）與分類名稱；內建分類碼固定、僅可改名；分類為平面、不分層
- **FR-003**: 系統 MUST 維護 func_name 受控清單（對應主系統作業功能代號），供 DM03 系統操作手冊類文件單選指定、DM01 依作業項目檢索；func_name 不可自由輸入
- **FR-004**: 系統 MUST 提供自動催辦門檻設定，值域 **1–30 天（預設 7）**、單一全域；超出值域時阻擋並提示
- **FR-005**: 系統 MUST 於權限管理列出共用 user table 之使用者，對每人提供 4 角色（管理者 / 編輯者 / 審核者 / 閱覽者）**複選**勾選（互不互斥、權限取聯集）；勾選 / 取消**即時生效**並**寫入異動紀錄**（記異動者 / 異動時間）；清單顯示「最後異動」欄；DM **不另設**權限變更查詢介面，完整歷史永久保留於 DB 供稽核
- **FR-006**: 系統 MUST 阻擋當前登入管理者停用**自己**之管理者角色；管理者間可互相停用，系統 MUST NOT 檢核「至少保留 1 名管理者」（若意外造成 0 名，由 IT 經 DB 恢復）
- **FR-007**: 系統 MUST 提供 7 項內建通知範本事件（文件送審 / 退回 / 發布、廢止申請送審 / 核准 / 退回、自動催辦），主旨 / 內文可編輯、可啟用 / 停用（停用後不發 Email，系統內訊息仍顯示）；事件固定**不可新增**；自動催辦僅以系統內訊息發送並含門檻天數設定
- **FR-008**: 系統 MUST 於使用者首次登入 DM 時自動授予「閱覽者」角色（不需管理者手動勾選）；離職 / 調職由共用 user table 之管理介面停用帳號，DM 不另處理

## 系統訊息

| 訊息代碼 | 類型 | 訊息內容 | 觸發 / 對應 FR |
|---------|------|---------|---------------|
| DM-MSG-DM09-001 | 錯誤 | 請填寫分類碼與分類名稱 | FR-002 必填未填 |
| DM-MSG-DM09-002 | 錯誤 | 此分類碼已存在，請使用其他分類碼 | FR-002 分類碼重複 |
| DM-MSG-DM09-003 | 提示 | 已停用；既有文件之標記不受影響，新增 / 搜尋下拉將不再顯示此項 | FR-001 停用分類 / func_name / 標籤 |
| DM-MSG-DM09-004 | 錯誤 | 催辦門檻須介於 1–30 天 | FR-004 值域超出 |
| DM-MSG-DM09-005 | 成功 | 角色已更新並即時生效 | FR-005 角色指派 |
| DM-MSG-DM09-006 | 錯誤 | 無法停用自己之管理者角色 | FR-006 自我保護 |
| DM-MSG-DM09-007 | 成功 | 通知範本已儲存 | FR-007 範本編輯 |
