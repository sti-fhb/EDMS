# User Story 1 — 系統設定（UCDM11）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

> **平台對齊（DP，2026-07-08 集中化）**：**DM 系統設定之維護介面已移至平台 DP 系統管理後台**——**權限管理**（DM 四角色＋可見對象授權指派）於 DP 後台「權限管理」畫面、**系統參數與清單**（文件分類 / func_name / 可見對象 / 檢索標籤定義、催辦門檻 / 排程參數）於 DP 後台「系統參數與清單」畫面、**通知範本**（9 項事件）於 DP 後台「通知範本」畫面；DM 管理者於 DP 後台操作、**按模組過濾**（只見 / 只改 `MODULE=DM` / `DM_` 前綴的項，與 ET / 平台互不可見）。**DM 不自設系統設定畫面（原 DM09）**。
> 本 US 保留之內容為 **DM 業務規則**：分類碼嵌入 DOC_ID 並鎖定、func_name 唯一手冊、標籤式可見性與四角色 / 可見對象授權語意、9 項通知事件與觸發、催辦 / 週報排程語意——DP 後台依此維護、DM 執行時依此運作。**「定義」（分類 / func_name / 可見對象 / 檢索標籤清單、範本）存 `DP_PARAM` / `DP_NOTIFY_TEMPLATE`；「指派 / 關聯」（使用者×角色 `DM_USER_ROLE`、使用者×可見對象授權 `DM_USER_TAG`）與判定仍屬 DM**。

## User Story

管理者於平台 DP 系統管理後台（按模組過濾）維護 DM 全模組可配置項目，含三面向：**系統參數與清單**（文件分類、關聯作業項目 func_name、受控標籤庫、自動催辦門檻）、**權限管理**（對共用 `DP_USER` 之使用者指派 DM 四角色，並維護**閱覽者可見對象/單位授權**）、**通知範本**（9 項內建事件之主旨 / 內文與啟用停用，「文件發布通知」發撰寫者+相符閱覽者，含「KPI 週報」「未讀提醒」）。角色指派與可見對象授權即時生效並寫入異動紀錄；分類 / func_name / 標籤採共通維護機制（不開放刪除、淘汰改停用、停用後既有引用保留；可見對象組之停用採 soft-retire）。下述 Acceptance Scenarios / FR 描述之為 **DM 業務規則與 DP 後台之運作行為**（畫面在 DP 後台）。

**Priority**: P1 — 權限管理為編輯者 / 審核者取得作業權限之入口；參數（分類 / func_name / 標籤 / 催辦門檻）為其他作業之基礎設定，無此設定文件無法正確分類、簽核無法催辦。

**Independent Test**: 管理者登入後進入平台 DP 系統管理後台（DM 模組），於系統參數與清單新增一個自訂分類（指定唯一分類碼）、於權限管理對某使用者勾選「編輯者」並儲存、於系統參數調整催辦門檻為 10 天；該使用者下次操作即具編輯權限，新分類出現於 DM03 下拉。

### Acceptance Scenarios

1. **Given** 管理者已登入，**When** 進入平台 DP 後台「系統參數與清單」（DM 模組），**Then** 系統列出文件分類（4 內建 + 自訂）、關聯作業項目（func_name）、標籤組與標籤，各項皆可新增 / 改名 / 啟用 / 停用，且**無刪除**功能
2. **Given** 管理者新增自訂分類，**When** 未填分類碼或填入已存在之分類碼，**Then** 系統阻擋並提示（DM-MSG-DM09-001 / 002）
3. **Given** 管理者新增自訂分類並填入唯一英數分類碼與名稱，**When** 儲存，**Then** 分類建立成功、**分類碼建立後鎖定**（不可再改，僅可改名），供 DOC_ID 嵌入
4. **Given** 管理者停用某分類 / 標籤 / func_name，**When** 儲存，**Then** 既有文件之標記不受影響，但 DM03 新增 / 編輯與 DM01 搜尋之下拉不再顯示該項（DM-MSG-DM09-003）
5. **Given** 管理者設定自動催辦門檻，**When** 輸入值落於 1–30 天，**Then** 儲存成功；輸入 0 或 > 30，**Then** 阻擋並提示（DM-MSG-DM09-004）
6. **Given** 管理者進入平台 DP 後台「權限管理」（DM 模組），**When** 系統載入，**Then** 列出共用 `DP_USER` 之使用者，每人顯示 4 角色（管理者 / 編輯者 / 審核者 / 閱覽者）核取方塊與「最後異動」欄
7. **Given** 管理者對某使用者勾選 / 取消角色，**When** 儲存，**Then** 即時生效並寫入異動紀錄（異動者 / 異動時間），下次該使用者操作依新權限生效（DM-MSG-DM09-005）
8. **Given** 當前登入之管理者取消**自己**之管理者角色，**When** 儲存，**Then** 系統阻擋並提示（DM-MSG-DM09-006）
9. **Given** 管理者 A 取消管理者 B 之管理者角色，**When** 儲存，**Then** 系統允許（不檢核「至少保留 1 名管理者」）
10. **Given** 管理者進入平台 DP 後台「通知範本」（DM 模組），**When** 系統載入，**Then** 列出 9 項內建事件（6 工作流〔含文件發布通知＝撰寫者+相符閱覽者〕 + 2 每週排程〔KPI 週報 / 未讀提醒〕 + 1 自動催辦），各可編輯主旨 / 內文、啟用 / 停用；自動催辦另含門檻天數設定；**KPI 週報 / 未讀提醒另含「每週執行時間」（星期＋時間，兩者共用，預設週一 10:00）設定**；「文件發布通知 / KPI 週報 / 未讀提醒」標示為僅 Email
11. **Given** 新使用者首次登入 DM，**When** 管理者檢視權限清單，**Then** 該使用者已存在且預設具「閱覽者」角色（系統自動授予，毋須管理者勾選）
12. **Given** 管理者於平台 DP 後台「權限管理」維護「閱覽者可見對象授權」，**When** 對某使用者勾選 / 取消可見對象（護理師 / 軍人 / 醫檢師 / 行政人員…），**Then** 即時生效並寫入異動紀錄，該閱覽者於 DM01 之可見範圍隨即改變（DM-MSG-DM09-008）
13. **Given** 管理者停用某「可見對象/單位」值，**When** 儲存，**Then** 系統採 soft-retire——既有文件與既有授權之可見性不變、僅停止後續指派，並提示受影響之文件 / 閱覽者數（DM-MSG-DM09-009）

## Functional Requirements

- **FR-001**: 系統 MUST 於參數設定維護文件分類（4 內建：SOP / 系統操作手冊 / 訓練教材 / 其他 + 管理者自訂）、關聯作業項目（func_name）、標籤組與標籤；三類採**共通維護機制**：可新增 / 編輯 / 啟用 / 停用，**不開放刪除**（淘汰改停用），停用後既有文件引用 100% 保留、僅影響後續新增 / 編輯與搜尋之下拉選項
- **FR-002**: 新增自訂分類時系統 MUST 要求唯一英數**分類碼**（檢核唯一、建立後鎖定，供 DOC_ID `DM-{分類碼}-{流水號}` 嵌入）與分類名稱；內建分類碼固定、僅可改名；分類為平面、不分層
- **FR-003**: 系統 MUST 維護 func_name 受控清單（對應主系統作業功能代號），供 DM03 系統操作手冊類文件單選指定、DM01 依作業項目檢索；func_name 不可自由輸入
- **FR-004**: 系統 MUST 提供自動催辦門檻設定，值域 **1–30 天（預設 7）**、單一全域；超出值域時阻擋並提示；值存平台 `DP_PARAM.DM_REMIND_THRESHOLD`（前綴 `DM_`），維護 UI 於平台 DP 系統管理後台（按模組過濾，見〈spec.md 跨模組共用規則〉集中化）
- **FR-005**: 系統 MUST 於權限管理列出共用 `DP_USER` 之使用者，對每人提供 4 角色（管理者 / 編輯者 / 審核者 / 閱覽者）**複選**勾選（互不互斥、權限取聯集）；勾選 / 取消**即時生效**並**寫入異動紀錄**（記異動者 / 異動時間）；清單顯示「最後異動」欄；DM **不另設**權限變更查詢介面，完整歷史永久保留於 DB 供稽核
- **FR-006**: 系統 MUST 阻擋當前登入管理者停用**自己**之管理者角色；管理者間可互相停用，系統 MUST NOT 檢核「至少保留 1 名管理者」（若意外造成 0 名，由 IT 經 DB 恢復）
- **FR-007**: 系統 MUST 提供 9 項內建通知範本事件（文件送審 / 退回、**文件發布通知**、廢止申請送審 / 核准 / 退回、**KPI 週報**、**未讀提醒**、自動催辦），主旨 / 內文可編輯、可啟用 / 停用（停用後不發該 Email，系統內訊息仍顯示）；事件固定**不可新增**；自動催辦僅以系統內訊息發送並含門檻天數設定；**「KPI 週報」「未讀提醒」另含「每週執行時間」（星期＋時間，兩者共用，寫 `DP_PARAM.DM_WEEKLY_SCHED_DAY_TIME`，預設週一 10:00）設定**；**「文件發布通知」（發撰寫者+相符閱覽者）「KPI 週報」「未讀提醒」皆僅 Email、非同步批次寄送**（詳見 [spec_us6.md](spec_us6.md) FR-008、[spec_us13.md](spec_us13.md)）。範本本體集中存平台 `DP_NOTIFY_TEMPLATE`（`MODULE=DM`），DM 管理者只編輯 MODULE=DM 的列、**編輯 UI 於平台 DP 系統管理後台「通知範本」**（按模組過濾，集中化見〈spec.md 跨模組共用規則〉）
- **FR-008**: 系統 MUST 於使用者首次登入 DM 時自動授予「閱覽者」角色（不需管理者手動勾選）；離職 / 調職由共用 `DP_USER` 之管理介面停用帳號，DM 不另處理
- **FR-009**: 系統 MUST 於權限管理提供「閱覽者可見對象授權」維護（使用者 × 「可見對象/單位」標籤，多選）；勾選 / 取消**即時生效**並**寫入異動紀錄**；未被授予任何可見對象之閱覽者僅能看到掛「全體」之文件；此授權即〈標籤式可見性〉之比對基準（見 [spec_us3.md](spec_us3.md)）
- **FR-010**: 「可見對象/單位」標籤組之維護沿用共通機制，惟**停用採 soft-retire**：不收回既有文件與既有授權之可見性，僅停止後續指派；停用時系統 MUST 提示受影響之文件 / 閱覽者數；此組不開放刪除
