# User Story 11 — 文件變更歷程查詢（UCDM10 / DM08）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

管理者跨文件查詢公開變更紀錄（發布 / 廢止兩類事件），依日期 / 申請人 / 核准人 / 操作類型篩選，並匯出 CSV 供資安稽核。撰寫過程動作與閱讀類動作不寫入此公開歷程；系統設定變更不於本頁顯示。變更歷程永久保留、不可竄改。

**Priority**: P3 — 跨文件稽核查詢，屬管理 / 合規輔助；單一文件之變更可由詳細頁版本歷程查得，故優先級最低。

**Independent Test**: 管理者進入 DM08，以日期區間與操作類型（發布 / 廢止）查詢，列出依時間排序之變更紀錄（時間 / 申請人 / 核准人 / 操作 / 文件 / 版本 / 備註）；匯出 CSV；確認上傳 / 編輯 / 下載等動作不出現於此清單。

### Acceptance Scenarios

1. **Given** 管理者已登入，**When** 自導覽進入 DM08，**Then** 顯示文件變更歷程查詢頁（僅 DM_ADMIN 可進）
2. **Given** 管理者輸入搜尋條件，**When** 以日期區間、申請人 / 核准人（帳號或姓名）、操作類型（全部 / 發布 / 廢止）查詢，**Then** 系統列出符合之變更紀錄，依時間由新到舊
3. **Given** 查詢結果，**When** 顯示，**Then** 欄位含時間 / 申請人 / 核准人 / 操作（發布 / 廢止）/ 文件名稱 / 版本號 / 備註（發布 → 變更摘要；廢止 → 廢止原因）
4. **Given** 管理者按「匯出 CSV」，**When** 執行，**Then** 系統匯出查詢結果供資安稽核
5. **Given** 撰寫過程動作（上傳 / 編輯 / 送審 / 退回 / 撤回），**When** 管理者查詢 DM08，**Then** 該等動作**不出現**（屬撰寫者個人視角，於 [spec_us9.md](spec_us9.md) 呈現）；閱讀類動作（下載 / 預覽）亦不記錄
6. **Given** 系統設定變更（分類 / 角色 / 通知範本），**When** 管理者查詢 DM08，**Then** **不於本頁顯示**（僅記錄異動者 / 異動時間於各設定，per [spec_us1.md](spec_us1.md)）

## Functional Requirements

- **FR-001**: DM08 MUST 僅 DM_ADMIN 可進入；提供跨文件之公開變更歷程查詢，僅含**發布 / 廢止**兩類事件
- **FR-002**: 系統 MUST 提供搜尋：日期區間、申請人 / 核准人（帳號或姓名）、操作類型（全部 / 發布 / 廢止）
- **FR-003**: 紀錄欄位 MUST 含時間 / 申請人 / 核准人 / 操作 / 文件名稱 / 版本號 / 備註（發布 = 變更摘要；廢止 = 廢止原因）；依時間 DESC 呈現
- **FR-004**: 系統 MUST 支援 CSV 匯出供資安稽核
- **FR-005**: 撰寫過程動作（上傳 / 編輯 / 送審 / 退回 / 撤回）MUST NOT 寫入公開變更歷程（改由 [spec_us9.md](spec_us9.md) 個人動態呈現）；閱讀類動作（下載 / 預覽）MUST NOT 記錄；系統設定變更 MUST NOT 於本頁顯示
- **FR-006**: 公開變更歷程 MUST **永久保留**、不可竄改、不可刪除（「至少 1 年」為法規下限）
