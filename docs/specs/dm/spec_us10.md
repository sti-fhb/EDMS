# User Story 10 — 已廢止文件查詢（UCDM08 / DM06）

> 返回總檔：[spec.md](spec.md) | 模組：文件管理（DM）

## User Story

管理者查詢已廢止文件，供稽核、醫療糾紛追溯、法規查核。可依關鍵字 / 分類 / 廢止日期區間搜尋，點入文件詳細頁之 read-only 模式檢視歷史脈絡，並可匯出 CSV 封存。已廢止文件不出現於文件庫主清單，一般使用者無法存取。

**Priority**: P2 — 稽核 / 法規追溯之必要查閱途徑，但非日常作業；於廢止流程上線後配套。

**Independent Test**: 管理者進入 DM06，以關鍵字與廢止日期搜尋得到已廢止文件清單；點任一筆進入 read-only 詳細頁（版本歷程自動展開、所有版本僅預覽）；匯出 CSV；一般使用者導覽看不到此項且 URL 直接進入被擋。

### Acceptance Scenarios

1. **Given** 管理者已登入，**When** 自導覽選單進入 DM06，**Then** 顯示已廢止文件查詢頁
2. **Given** 管理者輸入搜尋條件，**When** 以關鍵字（文件名 / 廢止原因）、分類、廢止日期區間查詢，**Then** 系統列出符合之已廢止文件
3. **Given** 查詢結果清單，**When** 顯示，**Then** 欄位含文件名稱（含末版版號）/ 分類 / 原撰寫者 / 廢止時間 / 廢止申請人 / 核准者 / 廢止原因
4. **Given** 管理者點任一筆，**When** 點擊，**Then** 進入文件詳細頁 **read-only 模式**（[spec_us4.md](spec_us4.md) FR-006：隱藏文件檔案 + 文件資訊、版本歷程自動展開、所有版本僅可預覽）
5. **Given** 管理者按「匯出 CSV」，**When** 執行，**Then** 系統匯出當前查詢結果供稽核封存
6. **Given** 一般使用者（無管理者角色），**When** 嘗試由導覽或直接 URL 進入 DM06，**Then** 導覽不顯示此項且後端 MUST 擋下直接存取（DM-MSG-DM06-002）

## Functional Requirements

- **FR-001**: DM06 MUST 僅 DM_ADMIN 可進入；一般使用者導覽不顯示且**後端 MUST 擋下直接 URL 存取**（非僅前端隱藏）
- **FR-002**: 系統 MUST 提供搜尋：關鍵字（文件名 / 廢止原因）、分類、廢止日期區間
- **FR-003**: 清單欄位 MUST 含文件名稱（含末版版號）/ 分類 / 原撰寫者 / 廢止時間 / 廢止申請人 / 核准者 / 廢止原因
- **FR-004**: 點任一筆 MUST 進入 [spec_us4.md](spec_us4.md) read-only 詳細頁（隱藏文件檔案 + 文件資訊整段、版本歷程自動展開、所有版本僅可預覽不可下載）
- **FR-005**: 系統 MUST 支援 CSV 匯出當前查詢結果供稽核資料封存

## 系統訊息
