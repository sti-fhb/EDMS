# User Story 3 — UCET002 課程建立與編輯（ET02）

> 對應 UC：UCET002 ｜ 功能選項：ET02（新增／編輯課程）｜ Priority：P1 ｜ Wireframe：[課程建立／編輯畫面](../../wireframes/et/index.html) ｜ 返回總檔：[spec.md](spec.md)

教師於 ET01 課程列表點「新增課程」進入新增模式，或點既有課程卡片進入編輯模式。本作業為 ET 最核心之教師端作業，涵蓋課程基本資料（課程名稱、**受訓單位標籤多選**、**課程起訖時間（閱課期間）**、課程描述）、章節編排（拖拉式調整章節 / 教材 / 測驗順序）、教材管理（影片本地上傳 / DM 文件引用 / 說明文字三類媒材自由組合）、測驗建立與題庫管理（單選 / 多選題、配分總和 100、題目順序系統內建洗牌、軟刪除）、**課後回饋問卷建立（0～1 份 / 課程，選配）**、儲存草稿與發布（發布時觸發**標籤自動邀請＋寄通知信**）、關閉與再開課（**可逆**，屬 [spec_us11.md](spec_us11.md) US11）、邀請學員（屬 [spec_us8.md](spec_us8.md) US8）等子功能。多裝置同時編輯採樂觀鎖（含問卷本體）。

**Priority**: P1

**Why this priority**: 教師核心作業，學員端所有學習活動之資料來源；無課程建立即無學員學習與測驗。

**Independent Test**: 教師完整建立一門課程（含至少 1 章節 + 1 教材 + 1 測驗、至少 1 個受訓單位標籤、課程起訖時間）並發布，標籤對應學員自動加入課程並收到通知信，於起始時間後可於 ET05 開始學習；亦可於 ET04 透過邀請碼加入。

**Acceptance Scenarios**:

### 基本資料

1. **Given** 教師於 ET01 點「新增課程」，**When** 進入 ET02 新增模式，**Then** 系統顯示課程名稱（必填）、受訓單位標籤（**多選**，自 `ET_TAG` 啟用中標籤選取；發布時至少 1 個）、課程起訖時間（閱課期間，起、訖兩個時間；發布時必填）、課程描述（選填，至多 500 字）等欄位，皆為空白
2. **Given** 教師僅填寫課程名稱後點「儲存草稿」（標籤與起訖時間留空），**When** 系統儲存，**Then** 寫入 ET_COURSE 狀態為「草稿」、擁有者為當前使用者；**受訓單位標籤與起訖時間於草稿狀態允許留空**（發布時才檢核）
3. **Given** 課程為「草稿」狀態，**When** 教師增刪受訓單位標籤，**Then** 系統允許**自由增刪**
4. **Given** 課程為「已發布」狀態，**When** 教師**新增**一個受訓單位標籤並儲存，**Then** 系統對該標籤對應且尚未加入之學員執行**補邀請＋每人寄通知信**（統一範本）；既有學員不受影響
5. **Given** 課程為「已發布」狀態，**When** 教師嘗試**移除**既有標籤，**Then** 系統阻擋（移除鈕不顯示）；已發布課程之標籤僅可新增、不可移除
6. **Given** 某標籤已被管理者停用，**When** 教師開啟標籤多選清單，**Then** 停用標籤不出現於可選清單；課程既有已掛之停用標籤保留、不受影響

### 章節編排

7. **Given** 教師於章節編排區點「新增章節」，**When** 系統建立空章節並追加至最末，**Then** 可立即拖拉調整順序
8. **Given** 教師對既有章節點「刪除」，**When** 跳 confirm modal 確認，**Then** 章節本體軟刪除（DELETED=1）；學員於該章節之學習紀錄與測驗成績連同移除；後續章節順序自動往前遞補

### 教材管理

9. **Given** 教師於章節中點「新增項目 → 教材」，**When** 開啟教材編輯視窗，**Then** 可組合三類媒材：影片（本地上傳 mp4 / webm，單檔 ≤ `ET_PARAM.VIDEO_MAX_SIZE_MB`，預設 500 MB）、文件（從 DM「訓練教材」分類下拉選取）、說明文字（WYSIWYG）
10. **Given** 教師選取一份 DM 文件加入教材，**When** DM 端發布新版本，**Then** 學員端下次開啟章節即取得最新版（無快取延遲）
11. **Given** 教材引用之 DM 文件已被廢止，**When** 教師進入 ET02 編輯，**Then** 該引用旁顯示「此文件已廢止」警告標籤；發布時系統阻擋並提示「請先移除廢止文件之引用後方可發布」
12. **Given** 教師欲移除廢止文件引用，**When** 開啟該教材編輯視窗，**Then** 須**逐筆刪除**該引用（系統不提供批次移除）

### 測驗建立

13. **Given** 教師於章節中點「新增項目 → 測驗」，**When** 開啟測驗編輯視窗，**Then** 可設定測驗名稱（必填）、及格分數（必填，0–100，預設 80）、作答時間限制（選填，分鐘；**空白 = 不限時**、≥ 1 = 限時分鐘）、重考次數上限（必填，0–999，預設 3，0 = 不允許重考）
14. **Given** 教師建立多選題並僅勾選錯誤選項（無正確選項），**When** 點儲存，**Then** 系統阻擋並提示「請至少指定 1 個正確選項」
15. **Given** 同一測驗下各題配分總和 ≠ 100，**When** 教師欲發布課程，**Then** 系統阻擋並提示「配分總和須等於 100」
16. **Given** 教師對既有題目點「刪除」，**When** 跳 confirm modal 確認，**Then** 題目本體軟刪除；學員於該題之作答紀錄與得分連同移除

### 課後問卷建立（2026-07-02 新增）

17. **Given** 課程尚無課後問卷，**When** 教師於 ET02 點「新增課後問卷」，**Then** 系統建立問卷（掛課程層級，不掛章節）；可設定問卷名稱並新增題目
18. **Given** 教師於問卷中新增題目，**When** 編輯題目，**Then** 每題為**單選題**；選項由教師自行新增 / 編輯 / 刪除（如：滿意 / 普通 / 不滿意）；每題至少 2 個選項
19. **Given** 某題選項不足 2 個，**When** 教師點儲存，**Then** 系統阻擋並提示「每題至少需 2 個選項」
20. **Given** 問卷尚無任何學員填答，**When** 教師修改題目或選項，**Then** 系統允許**自由編修**（新增 / 編輯 / 刪除）
21. **Given** 問卷已有任何學員填答，**When** 教師嘗試修改題目或選項，**Then** 系統阻擋並提示「已有學員填答，題目與選項不可修改」；此時教師僅可**停用問卷**（題目凍結規則）
22. **Given** 課程已存在 1 份問卷，**When** 教師嘗試再新增問卷，**Then** 系統阻擋（一門課程 0～1 份）
23. **Given** 課程未建立問卷，**When** 教師點發布，**Then** 系統**不因此阻擋**（問卷為選配；填寫問卷不是完課條件、不計入學習進度 %）

### 儲存與發布

24. **Given** 教師完成編輯點「發布」，**When** 系統檢核「至少 1 章節 + 1 教材」＋「**至少 1 個受訓單位標籤**」＋「**課程起訖時間已填**」＋「各測驗配分總和 = 100」＋「無引用之廢止 DM 文件」，**Then** 通過後狀態變更為「已發布」、寫入首次發布時間戳、系統自動產生 8 碼邀請碼
25. **Given** 發布檢核通過完成發布，**When** 系統處理，**Then** 依課程標籤**自動邀請**對應人員（限具「學員」角色者；多標籤取聯集、去重；「全體」= 全部具學員角色者）並**每人寄一封通知信**（統一範本，教師不可編輯主旨內文）
26. **Given** 教師發布時未掛任何標籤或未填起訖時間，**When** 點發布，**Then** 系統阻擋並提示具體缺漏項目
27. **Given** 課程已發布但尚未到起始時間（`OPEN_START_AT`），**When** 學員開啟 ET04 我的課程，**Then** 課程**不顯示、無法進入學習**（起始時間前對學員不可見）；教師端正常可見可編輯
28. **Given** 已發布課程教師繼續編輯，**When** 點儲存，**Then** **變更即時生效**，無需重新發布
29. **Given** 已發布課程教師新增章節，**When** 儲存，**Then** 系統自動寄信通知所有已加入學員（包含已完課學員）；已完課學員之完課狀態自動回退為「進行中」（學員已填之課後問卷不因回退而失效或重填）

### 樂觀鎖

30. **Given** 教師於 A 裝置開啟編輯（記錄 VERSION=5），同時於 B 裝置編輯同一課程並儲存（VERSION → 6），**When** A 裝置點儲存，**Then** 系統拒絕並提示「內容已被其他裝置變更，請重新整理後再儲存」
31. **Given** A、B 裝置編輯**同一課程之不同章節**，**When** 各自儲存，**Then** 互不衝突（樂觀鎖檢核粒度為單一實體；課程 / 章節 / 教材 / 測驗 / 題目 / **問卷本體**皆各自維護 VERSION）

## Functional Requirements

- **FR-ET-US3-01**: 系統 MUST 提供課程基本資料欄位——課程名稱（必填）、受訓單位標籤（多選，自 ET_TAG 啟用中標籤選取）、課程起訖時間（閱課期間，起、訖兩個時間）、課程描述（選填，至多 500 字）；草稿狀態允許受訓單位標籤與起訖時間留空，發布時始檢核；儲存草稿時寫入 ET_COURSE 狀態「草稿」、擁有者為當前使用者。
- **FR-ET-US3-02**: 系統 MUST 於「草稿」課程允許自由增刪受訓單位標籤；於「已發布」課程僅允許新增標籤（不可移除，移除鈕不顯示），新增標籤時對該標籤對應且尚未加入之學員執行補邀請並每人寄通知信（統一範本），既有學員不受影響。
- **FR-ET-US3-03**: 系統 MUST 於標籤多選清單排除已停用標籤；課程既有已掛之停用標籤保留、不受影響。
- **FR-ET-US3-04**: 系統 MUST 支援章節新增（追加至最末並可拖拉調整章節 / 教材 / 測驗順序）與刪除；刪除章節須經 confirm 確認，章節本體軟刪除（DELETED=1），學員於該章節之學習紀錄與測驗成績一併移除，後續章節順序自動遞補。
- **FR-ET-US3-05**: 系統 MUST 支援教材自由組合三類媒材——影片（本地上傳 mp4 / webm，單檔 ≤ `ET_PARAM.VIDEO_MAX_SIZE_MB`，預設 500 MB）、DM「訓練教材」分類文件引用、說明文字（WYSIWYG）；引用之 DM 文件恆取最新發布版本（無快取延遲）。
- **FR-ET-US3-06**: 系統 MUST 於教材引用之 DM 文件已廢止時於編輯頁顯示「此文件已廢止」警告標籤，並於發布時阻擋並提示須先移除廢止文件之引用；廢止引用僅可逐筆刪除（不提供批次移除）。
- **FR-ET-US3-07**: 系統 MUST 於測驗提供設定——測驗名稱（必填）、及格分數（必填，0–100，預設 80）、作答時間限制（選填，分鐘；空白 = 不限時、≥ 1 = 限時分鐘）、重考次數上限（必填，0–999，預設 3，0 = 不允許重考）；題目順序由系統內建洗牌。
- **FR-ET-US3-08**: 系統 MUST 於多選題無正確選項時阻擋並提示至少指定 1 個正確選項；於各題配分總和 ≠ 100 時阻擋發布並提示配分總和須等於 100；刪除題目須經 confirm 確認，題目本體軟刪除，學員於該題之作答紀錄與得分一併移除。
- **FR-ET-US3-09**: 系統 MUST 限制一門課程 0～1 份課後問卷（掛課程層級，不掛章節）；問卷題目為單選題、選項由教師自行新增 / 編輯 / 刪除且每題至少 2 個選項；重複新增問卷時阻擋。
- **FR-ET-US3-10**: 系統 MUST 於問卷尚無任何學員填答時允許自由編修題目與選項；一旦存在任何填答紀錄（ET_SURVEY_RESPONSE_M）即凍結，阻擋修改並提示，此時教師僅可停用問卷。
- **FR-ET-US3-11**: 系統 MUST 於發布時檢核「至少 1 章節 + 1 教材」＋「至少 1 個受訓單位標籤」＋「課程起訖時間已填」＋「各測驗配分總和 = 100」＋「無引用之廢止 DM 文件」，未通過時阻擋並提示具體缺漏項目；課後問卷為選配，系統 MUST NOT 因未建立問卷而阻擋發布，且問卷不作為完課條件、不計入學習進度 %。
- **FR-ET-US3-12**: 系統 MUST 於發布檢核通過後將狀態變更為「已發布」、寫入首次發布時間戳、自動產生 8 碼邀請碼，並依課程標籤自動邀請對應人員（限具「學員」角色者；多標籤取聯集、去重；「全體」= 全部具學員角色者）且每人寄一封通知信（統一範本，教師不可編輯主旨內文）。
- **FR-ET-US3-13**: 系統 MUST 於課程已發布但尚未到起始時間（`OPEN_START_AT`）時，對學員端 ET04 不顯示、不可進入學習；教師端正常可見可編輯。
- **FR-ET-US3-14**: 系統 MUST 使已發布課程之編輯變更即時生效（無需重新發布）；新增章節時自動寄信通知所有已加入學員（含已完課者），並將已完課學員之完課狀態回退為「進行中」（已填之課後問卷不因回退而失效或重填）。
- **FR-ET-US3-15**: 系統 MUST 對課程 / 章節 / 教材 / 測驗 / 題目 / 問卷本體各自維護 VERSION，儲存時以樂觀鎖檢核版本，衝突則拒絕並提示「內容已被其他裝置變更，請重新整理後再儲存」；不同實體之並行編輯互不衝突。

## 系統訊息

各訊息類型（錯誤 / 警告 / 確認 / 成功 / 提示）定義見 [spec.md](spec.md) §Requirements。ET02 為多 US 共用畫面，本 US（課程建立與編輯）使用流水號區段 001–099。

| 訊息代碼 | 類型 | 訊息內容 | 觸發情境 |
|---------|------|---------|---------|
| ET-MSG-ET02-001 | 成功 | 草稿已儲存 | 場景 2：儲存草稿 |
| ET-MSG-ET02-002 | 警告 | 此文件已廢止 | 場景 11：教材引用之 DM 文件已廢止（編輯頁警告標籤）|
| ET-MSG-ET02-003 | 錯誤 | 請先移除廢止文件之引用後方可發布 | 場景 11：發布時仍引用廢止文件 |
| ET-MSG-ET02-004 | 錯誤 | 請至少指定 1 個正確選項 | 場景 14：多選題無正確選項 |
| ET-MSG-ET02-005 | 錯誤 | 配分總和須等於 100 | 場景 15：測驗各題配分總和 ≠ 100 |
| ET-MSG-ET02-006 | 確認 | 確定刪除此章節？學員於本章節之學習與成績將一併移除 | 場景 8：刪除章節 |
| ET-MSG-ET02-007 | 確認 | 確定刪除此題目？學員於本題之作答紀錄將一併移除 | 場景 16：刪除題目 |
| ET-MSG-ET02-008 | 錯誤 | 每題至少需 2 個選項 | 場景 19：問卷題目選項不足 2 個 |
| ET-MSG-ET02-009 | 錯誤 | 已有學員填答，題目與選項不可修改 | 場景 21：問卷已有填答仍嘗試修改 |
| ET-MSG-ET02-010 | 提示 | 一門課程僅可建立 1 份課後問卷 | 場景 22：重複新增問卷 |
| ET-MSG-ET02-011 | 錯誤 | 發布條件未滿足：{缺漏項目}（受訓單位標籤 / 起訖時間 / 章節 / 教材 / 配分）| 場景 24/26：發布檢核未通過 |
| ET-MSG-ET02-012 | 成功 | 課程已發布，系統將依標籤自動邀請對應學員 | 場景 24/25：發布成功 |
| ET-MSG-ET02-013 | 成功 | 已對新標籤對應之學員補邀請並寄送通知信 | 場景 4：已發布課程新增標籤 |
| ET-MSG-ET02-014 | 錯誤 | 內容已被其他裝置變更，請重新整理後再儲存 | 場景 30：樂觀鎖版本衝突 |

## 相關 Clarifications 摘錄

- 課程擁有者於建立當下記錄；本擁有者欄位永久不可變更（管理者代為轉讓為例外，詳見 [spec.md](spec.md) §擁有權判定）
- 「關聯模組」欄位已於 2026-07-02 移除，改採受訓單位標籤（詳見 [spec.md](spec.md) §受訓單位標籤規則）
- 課程狀態機為 DRAFT → PUBLISHED ⇄ CLOSED，**關閉可逆**（原「停課不可逆」與 PENDING_CLOSE 廢除）；關閉與再開課詳見 [spec_us11.md](spec_us11.md) US11
- 課後問卷「題目凍結」以是否存在任何填答紀錄（ET_SURVEY_RESPONSE_M）判定，應用層檢核
- 教材文件統一由 DM 模組「訓練教材」分類提供
- 影片格式與大小上限由 ET 內部 `ET_PARAM` 控制；新教材下拉不出現已廢止文件
