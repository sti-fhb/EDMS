# User Story 3 — UCET002 課程建立與編輯（ET02）

> 返回總檔：[spec.md](spec.md) | 模組：教育訓練文件管理（ET）

教師於 ET01 課程列表點「新增課程」進入新增模式，或點既有課程卡片進入編輯模式。本作業為 ET 最核心之教師端作業，涵蓋課程基本資料、章節編排（拖拉式調整章節 / 教材 / 測驗順序）、教材管理（影片本地上傳 / DM 文件引用 / 說明文字三類媒材自由組合）、測驗建立與題庫管理（單選 / 多選題、配分總和 100、題目順序系統內建洗牌、軟刪除）、儲存草稿與發布、停課（屬 [spec_us11.md](spec_us11.md) US11）、邀請學員（屬 [spec_us8.md](spec_us8.md) US8）等子功能。多裝置同時編輯採樂觀鎖。

**Priority**: P1

**Why this priority**: 教師核心作業，學員端所有學習活動之資料來源；無課程建立即無學員學習與測驗。

**Independent Test**: 教師完整建立一門課程（含至少 1 章節 + 1 教材 + 1 測驗）並發布，學員端可於 ET04 透過邀請碼加入並於 ET05 開始學習。

**Acceptance Scenarios**:

### 基本資料

1. **Given** 教師於 ET01 點「新增課程」，**When** 進入 ET02 新增模式，**Then** 系統顯示課程名稱（必填）、關聯模組（必填，下拉選 ET_MODULE）、課程描述（選填）等欄位，皆為空白
2. **Given** 教師填寫課程名稱、關聯模組、描述後點「儲存草稿」，**When** 系統儲存，**Then** 寫入 ET_COURSE 狀態為「草稿」、擁有者為當前使用者、開課日期欄位保留空白
3. **Given** 已建立之課程，**When** 教師重新進入編輯，**Then** **關聯模組欄位不可變更**（避免影響已依模組預設帶入之學員名單）

### 章節編排

4. **Given** 教師於章節編排區點「新增章節」，**When** 系統建立空章節並追加至最末，**Then** 可立即拖拉調整順序
5. **Given** 教師對既有章節點「刪除」，**When** 跳 confirm modal 確認，**Then** 章節本體軟刪除（DELETED=1）；學員於該章節之學習紀錄與測驗成績連同移除；後續章節順序自動往前遞補

### 教材管理

6. **Given** 教師於章節中點「新增項目 → 教材」，**When** 開啟教材編輯視窗，**Then** 可組合三類媒材：影片（本地上傳 mp4 / webm，單檔 ≤ `ET_PARAM.VIDEO_MAX_SIZE_MB`，預設 500 MB）、文件（從 DM「訓練教材」分類下拉選取）、說明文字（WYSIWYG）
7. **Given** 教師選取一份 DM 文件加入教材，**When** DM 端發布新版本，**Then** 學員端下次開啟章節即取得最新版（無快取延遲）
8. **Given** 教材引用之 DM 文件已被廢止，**When** 教師進入 ET02 編輯，**Then** 該引用旁顯示「此文件已廢止」警告標籤；發布時系統阻擋並提示「請先移除廢止文件之引用後方可發布」
9. **Given** 教師欲移除廢止文件引用，**When** 開啟該教材編輯視窗，**Then** 須**逐筆刪除**該引用（系統不提供批次移除）

### 測驗建立

10. **Given** 教師於章節中點「新增項目 → 測驗」，**When** 開啟測驗編輯視窗，**Then** 可設定測驗名稱（必填）、及格分數（必填，0–100，預設 80）、作答時間限制（必填，分鐘，0 = 不允許作答）、重考次數上限（必填，預設 3，0 = 不允許重考，-1 = 不限）
11. **Given** 教師建立多選題並僅勾選錯誤選項（無正確選項），**When** 點儲存，**Then** 系統阻擋並提示「請至少指定 1 個正確選項」
12. **Given** 同一測驗下各題配分總和 ≠ 100，**When** 教師欲發布課程，**Then** 系統阻擋並提示「配分總和須等於 100」
13. **Given** 教師對既有題目點「刪除」，**When** 跳 confirm modal 確認，**Then** 題目本體軟刪除；學員於該題之作答紀錄與得分連同移除

### 儲存與發布

14. **Given** 教師完成編輯點「發布」，**When** 系統檢核「至少 1 章節 + 1 教材」，**Then** 通過後狀態變更為「已發布」、寫入發布時間戳（即開課日期）；開放學員加入
15. **Given** 教師發布時未滿足「至少 1 章節 + 1 教材」，**When** 點發布，**Then** 系統阻擋並提示具體缺漏項目
16. **Given** 已發布課程教師繼續編輯，**When** 點儲存，**Then** **變更即時生效**，無需重新發布
17. **Given** 已發布課程教師新增章節，**When** 儲存，**Then** 系統自動寄信通知所有已加入學員（包含已完課學員）；已完課學員之完課狀態自動回退為「進行中」

### 樂觀鎖

18. **Given** 教師於 A 裝置開啟編輯（記錄 VERSION=5），同時於 B 裝置編輯同一課程並儲存（VERSION → 6），**When** A 裝置點儲存，**Then** 系統拒絕並提示「內容已被其他裝置變更，請重新整理後再儲存」
19. **Given** A、B 裝置編輯**同一課程之不同章節**，**When** 各自儲存，**Then** 互不衝突（樂觀鎖檢核粒度為單一實體）

## 相關 Clarifications 摘錄

- 課程擁有者於建立當下記錄；本擁有者欄位永久不可變更（管理者代為轉讓為例外，詳見 [spec.md](spec.md) §擁有權判定）
- 教材文件統一由 DM 模組「訓練教材」分類提供
- 影片格式與大小上限由 ET 內部 `ET_PARAM` 控制；新教材下拉不出現已廢止文件
