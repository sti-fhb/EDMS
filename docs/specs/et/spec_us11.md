# User Story 11 — UCET003 課程關閉與再開課（ET02）

> 返回總檔：[spec.md](spec.md) | 模組：教育訓練文件管理（ET）

課程關閉有**兩種觸發、效果相同**：（1）**到期自動關閉**——課程到達訖止時間（`OPEN_END_AT`）時，由 SCHET002 每日排程自動將狀態 `PUBLISHED` → `CLOSED`，並輔以應用層即時判定（學員存取時檢核 `OPEN_END_AT` 已過即視同關閉），避免排程間隔造成的邊界誤差；（2）**手動關閉**——教師於 ET02 頁面右上點「關閉課程」按鈕（僅已發布狀態顯示）＋ confirm modal。關閉後課程進入**唯讀模式**：學員仍可見課程並回看已學過的內容，但不能再累積進度、作答測驗、解鎖新章節、填寫問卷；關閉當下作答中之 attempt 沿用 Attempt Snapshot **允許完成並計分**，之後不可開新 attempt。**關閉可逆**：教師可於已關閉課程執行「再開課」，強制重設一組新的起訖時間後狀態回 `PUBLISHED`，學員進度接續保留；關閉 / 再開課可重複多次（2026-07-02 變更：原「停課不可逆」與 `PENDING_CLOSE` 過渡狀態廢除）。

**Priority**: P3

**Why this priority**: 課程時窗管理之收尾流程；到期自動關閉為主要路徑、手動關閉與再開課為輔助情境，不影響日常學習作業；作答中 attempt 以 Attempt Snapshot 保護，無需過渡狀態。

**Independent Test**: 教師對已發布課程手動點「關閉課程」，課程立即轉「已關閉」、學員端轉唯讀回看；教師再點「再開課」並重設起訖時間，課程回「已發布」，學員進度接續保留。另將課程 `OPEN_END_AT` 設為已過時間，SCHET002 執行（或學員存取觸發即時判定）後課程自動轉「已關閉」。

**Acceptance Scenarios**:

### 顯示條件

1. **Given** 課程狀態為「草稿」，**When** 教師進入 ET02，**Then** 「關閉課程」按鈕**隱藏**
2. **Given** 課程狀態為「已發布」，**When** 教師進入 ET02，**Then** 「關閉課程」按鈕顯示於頁面右上
3. **Given** 課程狀態為「已關閉」，**When** 教師（owner）進入 ET02，**Then** 「關閉課程」按鈕**隱藏**、改顯示「**再開課**」按鈕；**課程內容仍可編輯**（非唯讀，供準備下次開課；2026-07-02 變更）

### 手動關閉

4. **Given** 課程狀態為「已發布」，**When** 教師點「關閉課程」，**Then** 系統跳 confirm modal「確定關閉此課程？關閉後學員僅可回看已學內容，不能再累積進度與作答」；確認後課程狀態**立即**變更為「已關閉」（CLOSED）、ET_COURSE.CLOSED_AT 寫入
5. **Given** 課程有學員 IN_PROGRESS attempt，**When** 教師點「關閉課程」並確認，**Then** 課程狀態**立即轉 CLOSED（不延遲、不進入過渡狀態）**；作答中 attempt 沿用 Attempt Snapshot **允許完成並計分**（含 timeout 自動提交），計入歷史紀錄；提交後不可再開新 attempt

### 到期自動關閉

6. **Given** 已發布課程到達 `OPEN_END_AT`，**When** SCHET002 每日排程執行，**Then** 系統自動將課程狀態 PUBLISHED → CLOSED、ET_COURSE.CLOSED_AT 寫入（無需教師操作）
7. **Given** 課程 `OPEN_END_AT` 已過但排程尚未執行，**When** 學員存取該課程，**Then** 應用層**即時判定視同關閉**（轉唯讀回看），不受排程間隔影響
8. **Given** 到期自動關閉當下有學員作答中，**When** 系統處理，**Then** 與手動關閉一致：作答中 attempt 沿用快照允許完成並計分，之後不可開新 attempt

### 關閉後學員端（唯讀回看）

9. **Given** 學員於 ET04 我的課程列表，**When** 系統載入，**Then** 已關閉課程**仍顯示**於列表，並標示「已關閉」狀態
10. **Given** 學員開啟已關閉課程，**When** 進入 ET05，**Then** 可**唯讀回看已學過的內容**（含教材重看、歷次作答明細、已填問卷內容）；**不能**再累積進度、作答測驗、解鎖新章節、填寫問卷
11. **Given** 課程已關閉，**When** 學員嘗試開始新測驗 attempt，**Then** 系統阻擋並提示「此課程已關閉，無法作答」
12. **Given** 課程已關閉，**When** 學員於 ET04 輸入該課程邀請碼欲加入，**Then** 系統阻擋並提示課程已關閉（邀請碼於關閉期間**失效**）

### 關閉後教師端

13. **Given** 教師（owner）查看已關閉課程，**When** 點擊進入 ET02，**Then** **課程內容仍可編輯**（非唯讀），右上顯示「再開課」按鈕；ET01 課程卡片顯示「已關閉」狀態 pill（2026-07-02 變更：原「唯讀 / 編輯控制項禁用」改為可編輯）
14. **Given** 課程已關閉，**When** 教師進入 US9 學員學習狀況追蹤，**Then** 仍可查看學員歷史完課狀態 / 成績 / 每次作答明細 / 問卷結果；**不可**再邀請新學員（標籤補邀請 / Email 邀請停用）、**不可**重置重考次數
15. **Given** 課程已關閉，**When** SCHET001 / SCHET002 排程執行，**Then** 該課程**不納入**每週統計、週報與各項提醒（排程停止）

### 再開課

16. **Given** 課程狀態為「已關閉」，**When** 教師點「再開課」，**Then** 系統**強制要求重新設定一組新的起訖時間**（`OPEN_START_AT` / `OPEN_END_AT`）；未填妥不可送出
17. **Given** 教師填妥新起訖時間並確認再開課，**When** 系統處理，**Then** 課程狀態 CLOSED → PUBLISHED；學員**進度接續保留**（不歸零）；ET_COURSE.URGENT_REMIND_SENT 歸 false（加急提醒依新起訖重新計）
18. **Given** 課程已再開課，**When** 學員於 ET04 輸入邀請碼，**Then** 邀請碼**恢復有效**、可正常加入；SCHET001 / SCHET002 排程恢復納入該課程
19. **Given** 課程已再開課，**When** 教師再次手動關閉或課程再次到達新 `OPEN_END_AT`，**Then** 課程再次轉 CLOSED（關閉 / 再開課**可重複多次**）

## Functional Requirements

- **FR-ET-US11-01**: 「關閉課程」按鈕 MUST 僅於課程狀態為「已發布」時顯示於 ET02 頁面右上；草稿與已關閉狀態 MUST NOT 顯示；已關閉狀態改顯示「再開課」按鈕
- **FR-ET-US11-02**: 教師手動關閉須先經 confirm modal 確認；確認後系統 MUST 立即將課程狀態 PUBLISHED → CLOSED 並寫入 ET_COURSE.CLOSED_AT（不延遲、不進入過渡狀態）
- **FR-ET-US11-03**: 課程到達 OPEN_END_AT 時，系統 MUST 由 SCHET002 每日排程自動將狀態 PUBLISHED → CLOSED 並寫入 CLOSED_AT，並輔以應用層即時判定（學員存取時檢核 OPEN_END_AT 已過即視同關閉），不受排程間隔影響
- **FR-ET-US11-04**: 關閉當下（手動或到期）作答中之 attempt，系統 MUST 沿用 Attempt Snapshot 允許完成並計分（含 timeout 自動提交）計入歷史紀錄；提交後 MUST NOT 再開新 attempt
- **FR-ET-US11-05**: 課程關閉後對學員端 MUST 進入唯讀模式：可唯讀回看已學過內容（教材重看、歷次作答明細、已填問卷），但 MUST NOT 累積進度、作答測驗、解鎖新章節或填寫問卷；已關閉課程 MUST 仍顯示於 ET04 我的課程列表並標示「已關閉」
- **FR-ET-US11-06**: 課程關閉期間邀請碼 MUST 失效；學員以邀請碼加入時系統 MUST 阻擋並提示課程已關閉
- **FR-ET-US11-07**: 課程關閉後教師（owner）端 MUST 保留課程內容可編輯（非唯讀）並可於 US9 查看學員歷史完課狀態 / 成績 / 作答明細 / 問卷結果，但 MUST NOT 再邀請新學員或重置重考次數；ET01 課程卡片 MUST 顯示「已關閉」狀態
- **FR-ET-US11-08**: 課程關閉期間 SCHET001 / SCHET002 MUST NOT 將該課程納入每週統計、週報與各項提醒
- **FR-ET-US11-09**: 教師執行「再開課」時系統 MUST 強制要求重新設定一組新的 OPEN_START_AT / OPEN_END_AT，未填妥 MUST NOT 送出；確認後 MUST 將狀態 CLOSED → PUBLISHED、學員進度接續保留（不歸零）、ET_COURSE.URGENT_REMIND_SENT 歸 false、邀請碼恢復有效、SCHET001 / SCHET002 排程恢復納入
- **FR-ET-US11-10**: 關閉 / 再開課 MUST 可重複多次；CLOSED_AT MUST 記錄最近一次關閉時間，再開課後保留供追溯

## 系統訊息

各訊息類型（錯誤 / 警告 / 確認 / 成功 / 提示）定義見 [spec.md](spec.md) §Requirements。ET02 為多 US 共用畫面，本 US（課程關閉與再開課）使用流水號區段 201–299。

| 訊息代碼 | 類型 | 訊息內容 | 觸發情境 |
|---------|------|---------|---------|
| ET-MSG-ET02-201 | 確認 | 確定關閉此課程？關閉後學員僅可回看已學內容，不能再累積進度與作答 | 場景 4：手動關閉課程 |
| ET-MSG-ET02-202 | 成功 | 課程已關閉 | 場景 4：關閉完成 |
| ET-MSG-ET02-203 | 錯誤 | 此課程已關閉，無法作答 | 場景 11：關閉後學員嘗試開新測驗 attempt |
| ET-MSG-ET02-204 | 錯誤 | 請重新設定一組新的起訖時間後方可再開課 | 場景 16：再開課未填妥起訖時間 |
| ET-MSG-ET02-205 | 成功 | 課程已再開課 | 場景 17：再開課完成（狀態回已發布）|

## 相關 Clarifications 摘錄

- 2026-07-02 變更：課程狀態機改為 `DRAFT → PUBLISHED ⇄ CLOSED`，**關閉可逆**；原「停課不可逆」與 `PENDING_CLOSE` 過渡狀態**廢除**（關閉已可逆、無需終態保護，作答中改以 Attempt Snapshot 保護，狀態即時轉 CLOSED）
- 關閉當下作答中 attempt 依 [spec.md](spec.md) §並發處理規則之 Attempt Snapshot 原則完成並計分；已加入學員之歷史紀錄完整保留
- 到期自動關閉 = SCHET002 每日排程 ＋ 應用層即時判定雙保險（詳見 [spec.md](spec.md) §排程作業總覽）
- CLOSED_AT 記錄**最近一次**關閉時間，再開課後保留供追溯
- 再開課不重發邀請、不重產邀請碼（沿用原 8 碼邀請碼，僅恢復有效）
