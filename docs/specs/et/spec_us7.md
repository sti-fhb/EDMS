# User Story 7 — UCET001 課程列表瀏覽（ET01）

> 對應 UC：UCET001 ｜ 功能選項：ET01（課程列表瀏覽）｜ Priority：P2 ｜ Wireframe：[課程列表畫面](../../wireframes/et/index.html) ｜ 返回總檔：[spec.md](spec.md)
> 2026-07-02 變更：關聯模組移除，改受訓單位標籤（多標籤 badges）；卡片新增起訖時間；狀態改為 草稿 / 已發布 / 已關閉（可逆）；列表不再依模組分組。

教師登入後預設首頁為 ET01 課程列表。本頁為新增與編輯之入口。教師於頁面上方分頁切換「我建立的 / 全部課程」：**「我建立的」顯示當前使用者建立之全部狀態課程（草稿 / 已發布 / 已關閉），供本人管理**；**「全部課程」僅顯示所有教師之「已發布」課程（草稿與已關閉不列入）**，供跨教師瀏覽觀摩。下方搜尋區提供關鍵字（課程名稱）與**受訓單位標籤**篩選，「全部課程」分頁額外提供「建立者」篩選。課程以卡片網格呈現（**不分組**）；卡片含課程名稱、**受訓單位標籤 badges（可多個）**、章節數、課程狀態（草稿 / 已發布 / **已關閉**）、**課程起訖時間**、學員人數、建立者姓名。他人建立之課程可進入瀏覽（唯讀），無法編輯；自己建立之已關閉課程可於「我建立的」進入並執行「再開課」（→ [spec_us11.md](spec_us11.md) US11）。

**Priority**: P2

**Why this priority**: 教師日常作業入口，但因本質為「列表瀏覽」屬輔助查詢功能；核心 CRUD 由 [spec_us3.md](spec_us3.md) US3 提供。

**Independent Test**: 多名教師建立各自之課程（含草稿 / 已發布 / 已關閉）後，於 ET01「我建立的」分頁見自己全部狀態課程；切「全部課程」僅見所有教師之**已發布**課程（自己與他人之草稿 / 已關閉皆不出現）；以受訓單位標籤篩選可正確過濾；點他人卡片進入後為唯讀模式。

**Acceptance Scenarios**:

1. **Given** 教師登入，**When** 系統依角色導向，**Then** 進入 ET01 課程列表（教師預設首頁）
2. **Given** 教師於 ET01 預設分頁為「我建立的」，**When** 系統載入，**Then** 僅顯示當前使用者建立之課程（**含草稿 / 已發布 / 已關閉全部狀態**，供教師管理自己的課程）
3. **Given** 教師切至「全部課程」分頁，**When** 系統載入，**Then** 顯示所有教師建立之課程、但**僅限「已發布」狀態**（**草稿與已關閉不列入**）；搜尋區額外出現「建立者」篩選欄位
4. **Given** 教師於搜尋區輸入關鍵字「採血」，**When** 點查詢，**Then** 系統依課程名稱模糊比對列出結果
5. **Given** 教師選取受訓單位標籤「護理師」，**When** 點查詢，**Then** 系統列出有掛該標籤之課程（一課程多標籤，任一命中即列出）
6. **Given** 系統載入清單，**When** 卡片渲染，**Then** 課程以卡片網格呈現（不分組），卡片含課程名稱、受訓單位標籤 badges（可多個）、章節數、課程狀態、課程起訖時間、學員人數、建立者姓名
7. **Given** 課程為當前使用者建立，**When** 卡片載入，**Then** 卡片右上**不顯示**「檢視」標籤；點擊進入 ET02 編輯模式
8. **Given** 課程為他人建立，**When** 卡片載入，**Then** 卡片右上顯示「檢視」標籤；點擊進入唯讀模式（所有編輯按鈕隱藏）
9. **Given** 課程狀態為「已關閉」，**When** 卡片載入，**Then** 卡片標示「已關閉」pill（**僅出現於「我建立的」分頁**，因「全部課程」不列入已關閉）；點擊進入後**課程內容仍可編輯**（非唯讀），並顯示「再開課」按鈕（→ [spec_us11.md](spec_us11.md) US11）
10. **Given** 教師於搜尋區右側點「新增課程」按鈕，**When** 系統載入，**Then** 進入 ET02 新增模式（屬 [spec_us3.md](spec_us3.md) US3）

## Functional Requirements

- **FR-ET-US7-01**: 系統 MUST 依分頁範圍過濾課程：「我建立的」僅回傳當前使用者建立之課程（含草稿 / 已發布 / 已關閉全部狀態），「全部課程」回傳所有教師之課程但僅限「已發布」狀態（草稿與已關閉不列入）
- **FR-ET-US7-02**: 系統 MUST 提供關鍵字（課程名稱模糊比對）與受訓單位標籤篩選，標籤篩選採任一命中即列出（一課程多標籤 OR），標籤下拉來源為 `ET_TAG` 全部標籤（含停用者）；「全部課程」分頁 MUST 額外提供「建立者」篩選欄位
- **FR-ET-US7-03**: 課程 MUST 以卡片網格呈現（不分組），每張卡片 MUST 含課程名稱、受訓單位標籤 badges（可多個）、章節數、課程狀態（草稿 / 已發布 / 已關閉）、課程起訖時間、學員人數、建立者姓名
- **FR-ET-US7-04**: 系統 MUST 依課程擁有者決定進入模式：當前使用者建立之課程進入 ET02 編輯模式且卡片不顯示「檢視」標籤；他人建立之課程進入唯讀模式且卡片顯示「檢視」標籤，並隱藏所有編輯與操作按鈕
- **FR-ET-US7-05**: 對當前使用者建立之「已關閉」課程，系統 MUST 於「我建立的」分頁標示「已關閉」pill，進入後課程內容仍可編輯（非唯讀）並顯示「再開課」按鈕（→ [spec_us11.md](spec_us11.md) US11）；「已關閉」pill MUST NOT 出現於「全部課程」分頁
- **FR-ET-US7-06**: 當篩選無符合結果時系統 MUST 顯示空狀態提示（ET-MSG-ET01-001）；並 MUST 於搜尋區提供「新增課程」入口，點擊進入 ET02 新增模式（屬 [spec_us3.md](spec_us3.md) US3）

## 相關 Clarifications 摘錄

- 課程擁有者於建立當下記錄；本擁有者欄位永久不可變更
- 他人課程開啟時，畫面以唯讀模式呈現，所有編輯與其他操作按鈕皆不顯示
- 受訓單位標籤篩選之下拉來源為 `ET_TAG` 全部標籤（含停用者，供查歷史課程）；標籤停用僅影響新課程掛載（per [spec.md](spec.md) §受訓單位標籤規則）
- 課程狀態「已關閉」為可逆狀態（2026-07-02 變更）；原「停課中（PENDING_CLOSE）/ 已停課（終態）」廢除
