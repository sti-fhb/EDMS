## 模組定位

教育訓練文件管理模組（ET）提供血液中心內部人員之線上教育訓練功能：教師建立課程、上傳教材、安排測驗與課後問卷，系統依受訓單位標籤自動邀請學員；學員透過線上方式完成學習與考核，並可隨時查閱個人學習進度與成績；系統每週自動統計看課狀況並提醒未看者。本模組**獨立於主系統部署**，與文件管理模組（DM）共用同一帳號系統與資料庫；教材文件統一由 DM 模組提供，避免教材與規範文件重複維護。

### 主要角色

| 角色 | 職責 |
|------|------|
| 管理者 | 權限管理（設定其他帳號於 ET 之角色與受訓單位標籤）；標籤庫維護；通知信範本與排程參數維護 |
| 教師 | 課程安排（建立 / 編輯 / 關閉 / 再開課程、章節、教材、測驗、課後問卷；設定起訖時間；邀請學員；查看學員學習狀況、每次作答明細、問卷結果）|
| 學員 | 加入課程、觀看教材、進行線上測驗、填寫課後問卷、查看個人學習進度 |
| 系統自動 | 章節解鎖判定、測驗閱卷、發布標籤自動邀請＋寄信、到期自動關閉、每週統計與提醒排程、截止前加急提醒 |

> 角色可重複指派；新使用者首次登入自動授予「學員」角色。

## Requirements

### Functional Requirements

**訊息類型定義**

各 US 之「系統訊息」表使用以下 5 種類型（與 BS / MA 模組一致）：

| 類型 | 視覺（Bootstrap class）| UI 形式 | 是否需使用者決策 | 用途 |
|------|---------------------|--------|----------------|------|
| 錯誤 | 🔴 紅（`alert-danger`）| 對話框 / Inline 表單驗證 | 否（最多單一【關閉】）| 操作失敗或不允許，使用者必須修正後繼續 |
| 警告 | 🟡 黃（`alert-warning`）| 對話框 / Inline alert | 視情境（對話框需二選一決策、inline 純顯示需注意）| 系統判斷有風險或異常情況 |
| 確認 | ⚪ 中性（Bootstrap modal 白底）| 對話框 | 是（二選一）| 不可逆動作之最終防呆 |
| 成功 | 🟢 綠（`alert-success`）| Inline alert（顯示於按鈕附近）| 否 | 主動操作完成之回饋 |
| 提示 | 🔷 淺藍（`alert-info`）| Inline alert / 列表空白狀態提示 | 否 | 正常流程告知（查詢結果、彙總、引導）|

訊息代碼格式：`ET-MSG-{功能作業}-{流水號}`，例 `ET-MSG-ET01-001`。多個 US 共用同一功能作業畫面時，各 US 分配獨立流水號區段以避免跨檔衝突（`ET02`：US3 = 001–099、US8 = 101–199、US11 = 201–299；`ET03`：US9 = 001–099、US12 = 101–199；`ET05`：US5 = 001–099、US13 = 101–199）；無畫面作業代碼之 US2（登入）以 `LOGIN`、US14（排程）以 `SCH` 為功能作業碼。各 US 之系統訊息表詳見對應 `spec_us{N}.md`。

### 跨模組共用規則

- **帳號共用**：ET 與 DM 共用平台模組 DP 之 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名，由平台 DP 定義），使用者註冊一次即可登入兩系統；ET 僅以 USER_ID 引用，端首次登入自動授予「學員」角色
- **多重角色**：角色（管理者 / 教師 / 學員）可重複指派，權限取聯集；登入後依角色預設首頁導向，多重角色由側邊選單切換
- **標籤不共用**：ET 自持受訓單位標籤庫（ET_TAG / ET_USER_TAG），**不與 DM「可見對象/單位」標籤共用**——兩系統使用者群相交而不相同，且後續可能各自新增對方沒有的單位（2026-07-02 設計決策）
- **教材文件來源**：所有教材引用之文件統一由 DM 模組之「訓練教材」分類提供；DM 端發布新版時 ET 教材自動取得最新版。DM 文件廢止後之 ET 端行為詳見下方「DM 文件廢止之 UI 規則」
- **擁有權判定**：課程之擁有者於建立當下記錄；他人課程僅可閱覽（唯讀），編輯與其他操作按鈕皆不顯示。**例外**：擁有者離職 / 帳號失能時，**管理者可代為轉讓擁有者**至其他教師（破例變更，需於稽核 log 記錄轉讓人 / 接收人 / 時間 / 原因）
- **稽核欄位**：所有資料異動自動記錄建立者 / 異動者、建立時間 / 異動時間
- **系統參數 / 通知範本 / 排程 / 發信集中於平台 DP**（2026-07-08 集中化）：ET 不自持參數表 / 範本表 / 寄件佇列 / 排程引擎——系統參數存平台 `DP_PARAM`（`PARAM_ID` 前綴 `ET_`）、通知範本存平台 `DP_NOTIFY_TEMPLATE`（`MODULE=ET`）、寄信呼叫平台唯一發信服務經 `DP_EMAIL_LOG` outbox、排程 SCHET001 / SCHET002 於平台 `DP_SCHEDULE` 註冊由平台引擎執行（`DP_SCHEDULE_LOG` 記錄，job handler 由 ET 提供）；**維護 UI 仍留在 ET**（參數於系統設定頁、範本於 ET09；ET 管理者只看 / 編輯前綴 `ET_` 或 `MODULE=ET` 的列）
- **獨立部署**：本模組與主系統 TBMS 各業務模組之帳號完全切開；僅與 DM 共用帳號系統，並共用平台 DP 之參數 / 範本 / 發信 / 排程基礎建設

### 受訓單位標籤規則（2026-07-02 新增）

| 項目 | 規則 |
|------|------|
| 標籤庫維護 | 管理者於 US1 新增 / 修改 / 停用 / 啟用；內建種子 5 筆：全體 / 護理師 / 行政人員 / 軍人 / 醫檢師 |
| 「全體」標籤 | 系統內建特殊標籤（IS_ALL=true），代表**所有具「學員」角色之使用者**；不可停用、不可刪除、不需逐人貼標 |
| 停用標籤 | 不可再掛至新課程；已掛之既有課程不受影響 |
| 人×標籤 | 多對多（一人可屬多個受訓單位）；管理者於 US1 指派 |
| 課程×標籤 | 多對多（一課程可掛多個標籤）；**發布前至少 1 個**（發布檢核）|
| 已發布課程之標籤調整 | 可**新增**標籤（觸發該標籤人員補邀請＋寄信）；**不可移除**既有標籤（已加入學員不受影響）；草稿狀態可自由增刪 |
| 貼標追溯 | 管理者**新增**人×標籤 → 系統自動補加入該標籤所有「已發布且未關閉」課程＋寄彙整一封通知信；**移除**人×標籤 → 既有課程學員名單**不變動**，之後新發布之該標籤課程不再自動邀請 |

### 課程起訖時間與狀態機（2026-07-02 新增）

**狀態機**：`DRAFT（草稿）→ PUBLISHED（已發布）⇄ CLOSED（已關閉）`；已發布與已關閉**可來回切換**（原 `PENDING_CLOSE` 過渡狀態與「停課不可逆」規則廢除）。

| 項目 | 規則 |
|------|------|
| 起訖時間 | `OPEN_START_AT` / `OPEN_END_AT`；**發布時必填**（草稿可留空）；即「閱課期間」 |
| 起始時間前 | 課程對學員**不可見**（我的課程清單不顯示、無法進入學習）；教師端正常可見可編輯 |
| 開放期間 | 學員正常學習、作答、累積進度 |
| 到期自動關閉 | 系統於到達 `OPEN_END_AT` 時自動將狀態轉 `CLOSED`（排程判定）|
| 手動關閉 | 教師於 ET02 點「關閉課程」（僅已發布狀態顯示）＋ confirm modal |
| 關閉後 — 學員端（唯讀）| 學員**仍可見**課程並**唯讀回看**已學過的內容；**不能**再累積進度、作答測驗、解鎖新章節、填寫問卷 |
| 關閉後 — 教師端（可編輯）| 教師（owner）於 ET02 **仍可編輯課程內容**（非唯讀，供準備下次開課）；隱藏「關閉課程」、顯示「再開課」；關閉期間不可邀請、不可重置重考（2026-07-02 變更：教師端由唯讀改為可編輯）|
| 關閉當下作答中 | 允許該次 attempt **完成並計分**（沿用 Attempt Snapshot）；之後禁止開始新的作答 |
| 邀請碼 | 關閉期間**失效**；再開課後恢復有效 |
| 排程 | 關閉期間停止每週統計與提醒（US14）；再開課後恢復 |
| 再開課 | 教師於已關閉課程點「再開課」→ **重設一組新的起訖時間** → 狀態回 `PUBLISHED`；學員進度接續保留；關閉 / 再開可重複多次 |

### 強制完成解鎖規則

學員於 US5 章節學習與 US6 線上測驗之解鎖規則：

| 章節包含內容 | 解鎖下一章節條件 |
|-------------|-----------------|
| 僅教材（含影片）| 章節內所有影片需播放覆蓋率達 80% |
| 僅教材（無影片，僅文件 / 說明文字）| 學員開啟該項目即視為已閱讀，純記錄不強制（系統不阻擋下一章節之解鎖）|
| 含測驗 | 上述教材條件達成 AND 測驗達到及格分數 |

**影片 80% 判定方式（2026-07-02 補充倍速與拉桿明文）**：

- **採累計覆蓋率**：學員實際播放過之區段聯集 ÷ 影片總長 ≥ 80% 即達成（非連續播放、非播放時數累加、**非「結束時間−開始時間」之牆鐘時間**）
- **允許跳轉**：學員可拖動進度條跳轉播放，不強制線性播放；但**只有實際播放過的區段計入**——直接拉到結尾不產生觀看區段、**不算看過**
- **允許倍速**：播放器提供倍速選項 0.75 / 1 / 1.25 / 1.5 / 2（**上限 2 倍**）；**倍速播放之區段照算覆蓋率**（2 倍速實際看完全片 = 覆蓋率 100%）
- **重複觀看不加成**：同一區段重複觀看不會多算覆蓋率（區段聯集去重）
- 教材內容可重複觀看；跨章節保持上次觀看位置，學員返回時自動恢復
- 課程於學習中被關閉 → 內容轉唯讀回看，不再累積進度（詳見「課程起訖時間與狀態機」）

> 實作建議：每段播放區間以**獨立資料列**儲存於 `ET_PROGRESS_INTERVAL`（避免 JSON 字串之 race condition、便於以 SQL 直接聚合）；每次播放結束 / 暫停時 `INSERT` 一筆，覆蓋率 = 區段聯集長度 ÷ 影片總長。

**區段儲存規則**：

| 項目 | 規則 |
|------|------|
| 儲存結構 | 獨立 table `ET_PROGRESS_INTERVAL`，每段播放為一資料列（含學員、影片教材 ID、起始秒、結束秒、寫入時間）|
| 區段上限 | 不限（不裁切，避免學員實際觀看資料遺失）|
| Normalize 時機 | **學員離開頁面時合併重疊 / 鄰近區段**（前端發送 `unload` 事件 → 後端執行合併：SELECT all → 排序 → 合併 → DELETE all → INSERT 合併後結果）；播放中不即時合併以避免高頻寫入 |
| 覆蓋率計算 | 即時聚合：SUM(END_SEC − START_SEC) ÷ VIDEO_DURATION（normalize 前後皆可正確計算，因聚合方法相同）|

> 學員未正常離開（瀏覽器強制關閉、斷網）時 normalize 不執行，但區段紀錄已 INSERT 完成，覆蓋率計算仍正確；下次學員再次開啟並離開時補做 normalize。

### 通知信統一範本（2026-07-02 新增）

- 課程 / 學習相關通知信採**管理者維護之統一範本**（`DP_NOTIFY_TEMPLATE`，`MODULE=ET`，US15 / ET09 維護；主旨＋內文，支援變數：課程名稱、起訖時間、學習連結等）；**教師不可逐課編輯信件內容**。範本集中於平台 DP，ET 寄信呼叫平台唯一發信服務（傳 `template_code`）經 `DP_EMAIL_LOG` outbox 寄送

### 多選題評分規則

採部分計分（Moodle 預設）：

```
該題得分 = max(0, (答對正確選項數 − 誤選數) ÷ 應選正確數 × 該題配分)
```

- 每答對一個正確選項加分
- 每選錯一個錯誤選項扣分
- 最低為 0 分（不會出現負分）
- 單選題仍為全有全無（答對得滿分、答錯 0 分）

**邊界規則**：

- **題目建立時必填檢核**：系統強制檢核「至少 1 個正確選項」；未滿足者阻擋儲存並提示「請至少指定 1 個正確選項」。「全為錯誤選項」題型不允許（避免公式分母為 0）
- **完全未作答之多選題**：學員提交時若該題完全未勾選任何選項，視為 0 分（不另保留空白讓教師審閱）

### 測驗作答機制（題目與選項洗牌）

| 範圍 | 行為 |
|------|------|
| 同一作答 session 內 | 題目順序與選項順序**保持一致**（學員跳題、暫存後回來，順序不變）|
| 重考時 | **重新洗牌**（題目順序與選項順序皆重新洗，避免學員記答案位置）|
| 多選題之選項順序 | **隨題目一同洗牌**（單選題之選項亦比照）|
| 順序快照 | DB 於 `ET_QUIZ_ATTEMPT_M` 存「該次作答之題目順序與選項順序快照」，供答題明細顯示與爭議釐清 |

> 實作建議：`ET_QUIZ_ATTEMPT_M` 新增 `QUESTION_ORDER` 與 `OPTION_ORDER`（每題之選項順序對應陣列）；學員作答中即依快照渲染題序，提交後依快照產出答題明細。

### 測驗設定邊界值

| 欄位 | 型別 | 值域 | 特殊值 |
|------|------|------|--------|
| 及格分數 | INT | 0–100 | 預設 80 |
| 作答時間限制（分鐘）| INT（可空）| NULL 或 ≥ 1 | **空白（NULL）= 不限時**（可作答、無倒數）；**≥ 1 = 限時 N 分鐘**（倒數歸零自動提交）；預設空白 |
| 重考次數上限 | INT | 0–999 | 預設 3；**0 = 不允許重考**（僅可作答 1 次）|

> **作答時間限制為兩態**：空白（NULL）= 不限時（答題介面不顯示倒數、僅手動提交）；≥ 1 = 限時該分鐘數（倒數歸零自動提交）。「不限時」以**留空**表示。停用某測驗請直接刪除該測驗項目（軟刪除），不以時間限制表達。

### 閱卷後正確答案顯示與作答明細（2026-07-02 擴充）

學員提交測驗並完成自動閱卷後，於答題明細頁**強制顯示正確答案**：

- 所有測驗皆強制顯示，無教師設定可關閉之選項（避免題庫不一致或教師設定遺漏）
- 學員「未及格但重考次數未用完」時**仍顯示**正確答案；重考時依「測驗作答機制」重新洗牌之題目順序與選項順序作答
- 顯示範圍：題幹原文 + 所有選項（含正確答案標示）+ 學員當次選擇 + 該題得分

**每次作答明細可回看（2026-07-02 新增）**：

- **學員端**：可回看自己**每一次** attempt 之成績與逐題明細（不限最近一次；依各 attempt 之順序快照渲染）
- **教師端**：於 US9 點擊學員展開其各測驗之**歷次 attempt 清單**（作答時間、總分、是否及格），點入單次 attempt 檢視逐題明細（題目、學員作答、正確答案、對錯、該題得分）

> 即便題庫外流風險存在，仍以「學習導向」為設計優先；未來若需限制可於 plan 階段討論加入教師設定。

### 軟刪除分流規則

刪除章節 / 題目時，**本體採軟刪除**（DELETED=1，前台不顯示但 DB 保留供稽核）；**學員學習紀錄與測驗成績連同 hard delete**（避免孤兒紀錄，且完課率以當前章節數為分母重算）：

| 刪除動作 | 軟刪除對象（DB 保留）| Hard delete 對象（DB 移除）|
|---------|---------------------|--------------------------|
| 刪除章節 | 章節本體、章節下教材與測驗本體 | 學員於本章節之學習紀錄與測驗成績 |
| 刪除題目 | 題目本體 | 學員於該題之作答紀錄與得分 |

> **「移除學員」不同於上述**：學員 ET_ENROLLMENT 標記為已移除（前台不顯示），但學習歷史紀錄完整保留；移除後不計入完課率統計。
>
> **學員無法主動退出課程**：學員端 ET04 我的課程頁僅顯示課程清單，無「退出課程」按鈕；如需退出，由教師於 US9 學員管理頁執行「移除學員」。

### 並發處理規則（教師動作 vs. 學員進行中）

當教師端執行「關閉課程 / 移除學員 / 編輯課程內容」等動作時，若有學員正在學習或作答中，採用以下原則：

#### 原則 1：Attempt Snapshot（作答即凍結規則）

學員一旦點「開始測驗」進入答題介面，系統為該次 `ET_QUIZ_ATTEMPT_M` 拍一份規則快照：題目與選項本體、題目與選項順序、總題數、配分、及格分數、作答時間限制。在該 attempt 結束（提交 / timeout 自動提交）之前，教師端任何修改皆**不影響本次 attempt**。新開啟之 attempt 才套用最新版規則。

#### 原則 2：Graceful Termination（允許完成本次學習動作）

教師端「較大動作」（關閉課程 / 移除學員）採「允許完成 → 之後生效」之延遲處置，避免吞分爭議。

#### 各場景處理

| 教師 / 系統動作 | 學員狀態 | 系統行為 |
|---------|---------|---------|
| **課程關閉（US11；到期自動或手動）** | 無人作答中 | 直接轉 `CLOSED`；學員端轉唯讀回看模式 |
| | 有學員作答中 | 課程立即轉 `CLOSED`（狀態不延遲）；**作答中 attempt 沿用快照允許完成並計分**（含 timeout 自動提交）；提交後不可再開新 attempt；學員端其餘功能即時轉唯讀（2026-07-02 變更：原 `PENDING_CLOSE` 過渡狀態移除，因關閉已可逆、無需終態保護）|
| **移除學員（US9）** | 無作答中 attempt | 立即標記移除，學員 next request 被導至「您已被該課程移除」訊息頁；ET_ENROLLMENT 之 REMOVED_AT 寫入 |
| | 有作答中 attempt | 移除按鈕跳警告「該學員作答中，移除後其當前 attempt 將保留並計入歷史。確定移除？」；確認後立即標記移除；學員當前 attempt 沿用快照繼續，提交後正常閱卷並計入歷史紀錄；學員 next page navigation 被導至「您已被該課程移除」訊息頁 |
| **刪除章節 / 題目** | 學員觀看 / 作答該內容 | 學員端顯示「此內容已刪除」訊息；作答中之 attempt 沿用 snapshot 完成本次 |
| **修改章節 / 教材 / 題目本體** | 學員學習 / 作答中 | 已開啟之 attempt **沿用 snapshot**（題幹、選項、配分、順序皆凍結）；新開啟之 attempt 套用最新版（含重新洗牌）；學員學習中觀看之教材則於下次重新整理時取得最新版 |

> **`ET_QUIZ_ATTEMPT_M` snapshot 欄位**：除 `QUESTION_ORDER` / `OPTION_ORDER` 外，須額外保存題目 + 選項 + 配分 + 及格分數 + 作答時間限制之版本快照（具體欄位於 data-model 階段定義）。

### 多裝置同時編輯之衝突處理（樂觀鎖）

同一教師（owner）於 A、B 兩裝置同時開啟 ET02 編輯同一課程時，採**樂觀鎖**：

- 每筆課程 / 章節 / 教材 / 測驗 / 題目 / 問卷本體於 DB 維護一 `VERSION` 欄位（每次寫入即 `VERSION = VERSION + 1`）
- 教師於 A 裝置開啟編輯時，前端記錄當下 `VERSION`；儲存時 server 端檢核傳入 `VERSION` 是否等於 DB 當下 `VERSION`：
  - **相等** → 寫入成功，`VERSION` 遞增
  - **不等** → 拒絕寫入，回傳「**內容已被其他裝置變更**，請重新整理後再儲存」提示；學員 / 教師需自行決定保留哪版（系統不自動合併）
- 此檢核覆蓋粒度為**單一實體**（如某課程之單筆章節），非整個課程；同一課程不同章節之同時編輯不互相衝突

### 管理者角色之最後一個保護

spec.md 已規範：「**當前登入之管理者無法停用自己之管理者角色**」（自我保護）。其他保護機制為：

| 情境 | 系統行為 |
|------|---------|
| 管理者 A 停用其他管理者 B 之角色 | **允許**（不檢核「停用後系統是否仍有啟用中管理者」）|
| 多名管理者依序互相停用，最終 0 名管理者 | **允許**（不阻擋）；此情境恢復需透過系統初始化流程之 DB 直接寫入新管理者 |

> **設計決策**：不增「至少保留 1 名啟用中管理者」之系統檢核。若意外造成 0 管理者狀態，由 IT 透過 DB 寫入恢復（per「系統初始化（第一個管理者）」之同等流程）；此情境屬極少數，不值得增加系統檢核複雜度。

### DM 文件廢止之 UI 規則

當 DM 端將某份教材文件標記為「廢止」後，ET 端對該文件之引用採以下處置：

| 端點 | 行為 |
|------|------|
| **教師端編輯頁（ET02）** | 教材編輯視窗於該引用旁顯示「**此文件已廢止**」警告標籤 |
| **教師端發布檢核** | 發布按鈕點擊時，若課程仍引用廢止文件 → **阻擋發布**並提示「請先移除廢止文件之引用後方可發布」 |
| **教師端移除引用** | 教師須**開啟該教材編輯視窗逐筆刪除廢止文件之引用**（系統不主動清除、不提供批次移除）；移除後重新發布 |
| **學員端章節學習頁（ET05）** | 於該文件項目旁顯示「**此文件已廢止**」標籤，學員仍可閱讀廢止前最後版本（per 跨模組共用規則） |

> 教師建立**新教材時**不可從下拉選單選到已廢止文件（DM「訓練教材」分類之下拉清單僅顯示有效文件）；既有已發布課程之既有引用則依上述規則處理。

### 邀請與加入課程（2026-07-02 變更：標籤自動邀請為主）

| 加入來源 | 觸發時機 | 識別 | 備註 |
|---------|---------|------|------|
| **標籤帶入**（主要）| 課程發布時系統自動執行；已發布課程新增標籤時補執行 | 由 ET_COURSE_TAG × ET_USER_TAG 對應決定（限具「學員」角色者；多標籤取**聯集、去重**）| 加入來源標記「標籤帶入」；**同時每人寄一封通知信**（統一範本）；「全體」標籤 = 全部具學員角色者 |
| 事後貼標追溯 | 管理者新增人×標籤時 | 同上 | 補加入該標籤所有「已發布且未關閉」課程；寄**彙整一封**通知信（列出所有新加入課程）|
| Email 邀請（補件用）| 教師寄發邀請信，學員點擊邀請連結 | 邀請連結內含 token | 供臨時追加不在標籤內的人；信件採**統一範本**（教師不可編輯主旨內文）；學員可於 ET_INVITATION 追蹤狀態 |
| 邀請碼 | 課程發布時系統自動產生 8 碼唯一邀請碼；學員於 ET04 輸入加入 | **純數字 8 碼**（系統產生，唯一）| 邀請碼於課程**關閉期間失效、再開課後恢復**；教師**不可手動指定**、**不提供重新產生功能**（避免外洩管理複雜化）|

**標籤對應變更後對既有 / 新課程之影響**：

| 變更類型 | 對既有課程之學員名單影響 | 對之後新發布課程之邀請 |
|---------|--------------------|---------------------|
| **移除**使用者 × 標籤 | **不變動**（已加入之學員名單保留；學員可繼續學習）| 之後新發布之該標籤課程，**不會**自動邀請該使用者 |
| **新增**使用者 × 標籤 | **自動補加入該標籤所有「已發布且未關閉」課程**（寄彙整通知信）| 之後新發布之該標籤課程，**會**自動邀請該使用者 |

> 學員之主動退場僅能由教師於 US9 學員管理頁執行「移除學員」；標籤對應變更**不會自動移除既有學員**。

### 重考次數重置規則（US9）

教師於 US9「已加入」頁籤之**區塊 2 作答明細**（各測驗標題列）可對個別學員執行「重置重考次數」（2026-07-02：由學員清單列移入作答明細）：

- **重置範圍**：以**特定測驗**為單位（非整個課程之所有測驗一次重置）；教師於區塊 2 該測驗標題列點擊重置按鈕
- **重置動作**：將該學員於該測驗之「已用重考次數」歸 0，即恢復至滿配額（重考次數上限）狀態
- **可重置時機**：僅當學員於該測驗之「已用重考次數已達上限且尚未及格」時，重置按鈕方可點擊；其他狀態（仍有重考次數 / 已及格）下按鈕不顯示或不可點擊
- **不存在的情境**：因觸發條件已限制為「次數歸 0 時才可重置」，故不會發生「作答中重置」之競態狀況

### 重考時機（無冷卻）

學員測驗未及格、且剩餘重考次數 > 0 時，可**立即**點「重新作答」進入下一次 attempt：

- 系統**不設冷卻時間**（cooldown）；學員提交後即可立刻重考
- 重考開啟新 `ET_QUIZ_ATTEMPT_M` 並依「測驗作答機制」重新洗牌題目與選項順序
- 若需限制重考節奏，可透過「重考次數上限」（`ET_QUIZ.MAX_RETRY`）控制，不另以時間限制

### 完課定義與完課率計算（US9 / US14）

| 名詞 | 定義 |
|------|------|
| 完課 | 學員於該課程**所有章節**之教材與測驗皆達成解鎖條件，且**所有測驗皆達及格分數**（填寫問卷不是完課條件）|
| 完課狀態 | 已完成 / 進行中 / 未開始（依學員之 ET_PROGRESS 即時判定）|
| 完課率 | `已完課學員數 ÷ 已加入學員數 × 100%`；**分母不含已移除學員**（per UC「移除後不計入完課率統計」）|
| 看課進度 % | 已完成章節項目數 ÷ 總章節項目數（影片依覆蓋率 ≥ 80%、測驗依及格、文件 / 說明文字依開啟判完成）|
| 平均成績 | 該學員於該課程**所有測驗**之得分平均；每測驗以**該學員之最高分**為基礎（per RQ「以最高分為結業成績」），再對所有測驗取算術平均。**未作答之測驗排除於計算**（不視為 0 分；分母為已作答測驗數），學員完課前該欄位之意義為「目前為止之平均」|

> 學員完課狀態於每次學習動作 / 測驗提交後即時重算；不依賴排程批次更新。每週統計快照（US14）為報表用途，不回寫完課狀態。

### 已發布課程之章節異動規則

教師於已發布狀態下仍可繼續編輯課程內容（新增章節、新增教材 / 測驗、修改既有內容），對既有學員之影響如下：

| 異動類型 | 對既有學員之影響 |
|---------|------------------|
| 新增章節 | 已完課學員（狀態為「已完成」）**自動回退為「進行中」**；該學員需重新學習新章節並通過測驗方可重新達成完課 |
| 新增教材 / 測驗（既有章節下）| 該章節下之既有學員需補完新教材 / 測驗，未完成者於該章節進度回退；已完課學員一併回退為「進行中」|
| 修改既有教材 / 測驗（不改本體 ID）| 既有學員之學習紀錄與成績保留，但需重新觀看 / 作答否則該章節進度顯示為「需更新」（具體 UI 於 plan 階段定義）|
| 刪除章節 / 題目 | 依「軟刪除分流規則」處理（章節 / 題目本體軟刪除，學員紀錄 hard delete）|

**通知機制**：

- 教師於已發布課程新增章節後，系統**每次新增即自動寄信**通知所有已加入學員（包含已完課學員），告知課程內容有更新
- 信件內容：課程名稱、新增章節名稱、進入學習連結（採統一範本「課程內容更新通知」）
- 通知與 US8 邀請信走相同之 Email Server 介接
- 系統**不採 debounce / 合併寄送**亦**不提供學員端通知偏好設定**；實務上教師於初次發布前完成所有章節編排（發布時透過標籤自動邀請通知學員），後續對已發布課程之章節新增屬少數情境，每次寄信不易造成困擾

> **已完課學員回退與問卷**：學員已填之課後問卷不因完課狀態回退而失效或重填；問卷維持一人一次。

### 系統初始化（第一個管理者）

- **首次部署時**，由 IT 人員透過 DB 直接寫入第一筆管理者帳號（含帳號 Email、姓名、密碼雜湊、ET_USER_ROLE.ROLE=管理者）
- 後續所有管理者帳號之新增 / 變更皆透過 US1 權限管理畫面執行；系統不再提供 DB 寫入以外之初始化介面
- 與部署 / 維運文件約定：第一筆管理者帳號之 Email 與初始密碼於部署完成後由系統管理員另行通知

### 帳號（Email）變更流程

使用者於 US10 變更帳號（Email）時，採「**雙信箱共存 + 新 Email 驗證後切換**」之延遲生效機制：

| 階段 | 狀態 | 認證行為 |
|------|------|---------|
| 1. 提交變更 | 寫入 `DP_USER.EMAIL_PENDING_CHANGE` + `EMAIL_PENDING_TOKEN` + `EMAIL_PENDING_EXPIRES_AT`（30 分鐘後） | 舊 Email **仍可正常登入**（變更尚未生效）|
| 2. 系統寄驗證信至**新 Email** | 同上 | 同上 |
| 3a. 學員 30 分鐘內點驗證連結 | `DP_USER.EMAIL` 更新為新值；清除 PENDING 欄位 | 變更生效；學員當前 session 強制登出，**須以新 Email 重新登入** |
| 3b. 學員未點驗證連結 / 逾 30 分鐘 | PENDING 欄位於下次登入或排程清理時失效 | 舊 Email **永遠維持有效**；變更請求視為作廢 |
| 4. 學員於 PENDING 期間再次提交新變更 | 舊 PENDING 紀錄被**取代**，新驗證連結重新計時 30 分鐘 | 舊 Email 仍可登入 |

> 系統不主動「逾期回滾」之 cron；PENDING 欄位於下次與該帳號相關之認證 / 變更動作時即時檢核並清理。
>
> 對應實體：`DP_USER`（帳號 Email 變更 PENDING 欄位）由**平台模組 DP 定義與維護**；ET 僅以 USER_ID 引用，本節保留為 ET 端個資維護之情境描述，實作以平台 DP 為準（UCDP004）。

## Assumptions

- **共用帳號與認證權責切分**：本模組與 DM 模組共用平台模組 DP 之 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名，由平台 DP 定義），使用者註冊一次即可登入兩系統；認證（登入 / 註冊 / 忘記密碼）與個人資料維護能力**由平台模組 DP 統一提供（以簡單 JWT 實作、DP_USER）**，非 ET 自有中介層。ET / DM 各自管理自己的角色與標籤（角色不共用、標籤不共用、功能權限不共用）。與主系統 TBMS 各業務模組之帳號完全切開（per 2026-04-28 設計決策）
- **受訓單位標籤清單**：內建種子 5 筆（全體 / 護理師 / 行政人員 / 軍人 / 醫檢師），管理者可新增 / 修改 / 停用 / 啟用（2026-07-02 變更；原「業務模組七類系統內定」廢除）
- **影片儲存**：影片採本地上傳，不串接外部影音平台（YouTube、Vimeo 等）。可接受之格式（預設 mp4 / webm）與單檔大小上限（預設 500 MB）由**系統參數**（平台 `DP_PARAM`，`PARAM_ID` 前綴 `ET_`：`ET_VIDEO_ALLOWED_FORMATS` / `ET_VIDEO_MAX_SIZE_MB`）控制；ET 系統參數集中於平台 DP（2026-07-08），維護 UI 仍在 ET 系統設定頁
- **DM 文件分類**：教師可選用之文件限 DM「訓練教材」分類；其他分類不出現於下拉選項
- **完課定義**：學員完成課程中所有章節之學習與測驗即視為完課；完課率公式已於 spec §完課定義與完課率計算 定義；填寫問卷不是完課條件
- **排程執行環境**：SCHET001（每週）/ SCHET002（每日）於平台 `DP_SCHEDULE` 註冊、由平台單一排程引擎執行（`DP_SCHEDULE_LOG` 記錄）；job handler 由 ET 提供（需要業務資料時反向 import ET service）；執行時間與加急天數由 `DP_PARAM`（前綴 `ET_`：`ET_WEEKLY_STAT_DAY_TIME` / `ET_URGENT_REMIND_DAYS`）控制
- **個資處理**：使用者 Email 與姓名為一般個資，不需加密儲存；密碼採雜湊儲存；問卷填答具名屬一般個資
- **跨模組互動細節**：請求 / 回應格式、錯誤碼由 `specs/et/contracts/` 規範（待 `/speckit.plan` 階段產出）
- **數量 / 長度上限延遲決定**：課程 / 章節 / 教材 / 測驗 / 題目 / 問卷題目之**數量上限**，以及課程描述 / 章節名稱 / 教材說明文字 / 測驗名稱 / 學員人數等**長度與容量上限**，於 `/speckit.plan` 與 data-model 階段依效能與業務情境綜合決定，spec 階段不預先綁定

## 排程作業總覽（2026-07-02 新增）

| 排程編號 | 名稱 | 頻率 | 內容 |
|---------|------|------|------|
| SCHET001 | 每週統計與週報 | 每週一 10:00（`DP_PARAM.ET_WEEKLY_STAT_DAY_TIME` 可調）| 統計快照寫入 ET_WEEKLY_STAT；週報寄教師＋管理者；進度 0% 學員寄未看提醒（一人一信彙整）|
| SCHET002 | 每日課程時窗檢查 | 每日 | 到期課程自動轉 CLOSED；訖止前 3 天（`DP_PARAM.ET_URGENT_REMIND_DAYS` 可調）對未完課學員寄加急提醒（每課一次）|

> 課程「到期自動關閉」由 SCHET002 每日檢查搭配應用層即時判定（學員存取時檢核 `OPEN_END_AT` 已過即視同關閉），避免排程間隔造成的邊界誤差。
