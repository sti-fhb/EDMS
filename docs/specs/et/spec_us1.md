# User Story 1 — UCET010 權限管理與標籤維護（系統設定：參數設定 + 權限管理 分頁，ET07）

> 對應 UC：UCET010 ｜ 功能選項：ET07（系統設定：參數設定＋權限管理）｜ Priority：P1 ｜ Wireframe：[權限管理畫面](../../wireframes/et/index.html) ｜ 返回總檔：[spec.md](spec.md)
> 2026-07-02 變更：管理者單一「**系統設定**」畫面分 3 分頁（比照 DM09）：**參數設定**（受訓單位標籤庫維護）、**權限管理**（角色與標籤指派）、**通知範本**（US15）。本 US 對應「參數設定」與「權限管理」兩分頁。功能作業代碼 ET07 沿用。

管理者於「系統設定」之「**權限管理**」分頁設定其他帳號於 ET 之角色（管理者 / 教師 / 學員，三角色可重複指派）與**受訓單位標籤指派**（多對多，可複選、可全不勾即「未指派」，用於課程發布時之標籤自動邀請）；於「**參數設定**」分頁維護**受訓單位標籤庫**（新增 / 修改 / 停用 / 啟用；內建種子 5 筆：全體 / 護理師 / 行政人員 / 軍人 / 醫檢師；「全體」為系統內建特殊標籤（IS_ALL），代表所有具「學員」角色之使用者，不可停用、不可刪除）。角色變更後使用者下次登入即依新角色導向預設首頁。新使用者首次登入後系統自動授予「學員」角色，受訓單位標籤預設「未指派」。系統初始化之第一個管理者由 IT 透過 DB 直接寫入；不提供「最後一個管理者」之保護檢核（若意外造成 0 管理者狀態，由 IT 透過 DB 寫入恢復）。

**Priority**: P1

**Why this priority**: 權限管理為其他角色（教師 / 學員）取得 ET 存取之入口；標籤庫與人×標籤對應為課程發布時「標籤自動邀請」之資料基礎。無管理者完成角色指派與標籤維護，教師無法建立課程、學員無法被自動邀請，整個系統運作停擺。

**Independent Test**: 由管理者登入後進入 ET07，對特定使用者勾選 / 取消角色與受訓單位標籤，儲存後該使用者下次登入即依新角色導向預設首頁；於標籤庫新增一筆標籤後可於指派與課程設定選單選取；新註冊使用者自動以「學員」角色、標籤「未指派」存在於清單中。

**Acceptance Scenarios**:

### 角色與受訓單位標籤指派

1. **Given** 管理者已登入，**When** 進入 ET07 權限管理頁，**Then** 系統列出所有使用者，每筆顯示三角色（管理者 / 教師 / 學員）核取方塊與受訓單位標籤
2. **Given** 管理者於使用者列勾選「教師」角色並儲存，**When** 該使用者下次登入，**Then** 系統依新角色導向預設首頁（教師 → 課程列表）
3. **Given** 管理者於使用者列開啟「受訓單位標籤」編輯 modal 並勾選「護理師」、「醫檢師」，**When** 儲存，**Then** 該對應寫入 ET_USER_TAG（多對多，可複選、可全不勾即「未指派」）；之後新發布之掛有該標籤課程，會自動邀請該使用者
4. **Given** 當前登入之管理者嘗試**取消自己**之「管理者」角色核取方塊，**When** 點儲存，**Then** 系統阻擋並提示「無法停用自己之管理者角色」
5. **Given** 管理者 A 取消管理者 B 之「管理者」角色，**When** 儲存，**Then** 系統允許（不檢核「停用後系統是否仍有啟用中管理者」）
6. **Given** 一名新註冊使用者，**When** 管理者進入 ET07，**Then** 該使用者出現於清單中，預設角色「學員」、受訓單位標籤「未指派」
7. **Given** 管理者於使用者列變更角色或受訓單位標籤，**When** 儲存，**Then** 系統記錄修改者與時間於「最後修改」欄；完整變更歷史寫入平台共用 `DP_AUDIT_LOG`（資安稽核）

### 貼標追溯

8. **Given** 管理者於受訓單位標籤指派**新增**某使用者 × 護理師，**When** 儲存，**Then** 系統自動將該使用者補加入「護理師」標籤所有「**已發布且未關閉**」課程（加入來源「標籤帶入」），並寄**彙整一封**通知信（統一範本，列出所有新加入課程）
9. **Given** 管理者於受訓單位標籤指派**移除**某使用者 × 護理師，**When** 儲存，**Then** 既有課程之學員名單**不變動**（已加入者可繼續學習）；之後新發布之掛有該標籤課程，**不會**自動邀請該使用者

### 標籤庫維護（系統設定「參數設定」分頁）

10. **Given** 管理者於系統設定「**參數設定**」分頁進入標籤庫維護，**When** 系統載入，**Then** 列出所有受訓單位標籤（標籤名稱、啟用狀態、顯示順序）；內建種子 5 筆：全體 / 護理師 / 行政人員 / 軍人 / 醫檢師
11. **Given** 管理者新增一筆標籤「志工」，**When** 儲存，**Then** 系統檢核標籤名稱唯一後寫入 ET_TAG；該標籤即可於使用者標籤指派與課程受訓單位標籤選單中選取
12. **Given** 管理者修改某標籤之名稱，**When** 儲存，**Then** 既有之使用者標籤對應與課程標籤同步顯示新名稱（對應關係不變）
13. **Given** 管理者**停用**某標籤，**When** 儲存，**Then** 該標籤不可再掛至新課程（課程標籤選單不出現）；已掛之既有課程不受影響
14. **Given** 管理者對已停用之標籤點「啟用」，**When** 儲存，**Then** 該標籤恢復可掛至新課程
15. **Given** 管理者嘗試對「全體」標籤執行停用或刪除，**When** 點擊，**Then** 系統阻擋（「全體」為 IS_ALL 特殊標籤，不可停用、不可刪除）

## Functional Requirements

- **FR-ET-US1-01**: 系統 MUST 於「權限管理」分頁列出所有共用 `DP_USER`（平台模組 DP 定義）之使用者，對每人提供三角色（管理者 / 教師 / 學員）核取方塊（三角色可重複指派、互不互斥）與受訓單位標籤指派入口。
- **FR-ET-US1-02**: 系統 MUST 於角色或受訓單位標籤變更儲存後，記錄修改者與時間於「最後修改」欄，並將完整變更歷史寫入**平台共用 `DP_AUDIT_LOG`**（角色 / 標籤變更屬資安類稽核，由平台模組 DP 統一保存）。
- **FR-ET-US1-03**: 系統 MUST 使角色變更於使用者下次登入時生效，並依新角色導向預設首頁（如教師 → 課程列表）。
- **FR-ET-US1-04**: 系統 MUST 阻擋當前登入之管理者取消自己之「管理者」角色。
- **FR-ET-US1-05**: 系統 MUST NOT 檢核「停用後系統是否仍有啟用中管理者」；管理者間可互相取消管理者角色（若意外造成 0 管理者，由 IT 透過 DB 寫入恢復）。
- **FR-ET-US1-06**: 系統 MUST 於新註冊使用者首次登入後自動授予「學員」角色、受訓單位標籤預設「未指派」，並使其出現於權限管理清單。
- **FR-ET-US1-07**: 系統 MUST 將受訓單位標籤指派以多對多方式寫入 ET_USER_TAG，允許複選、允許全不勾即「未指派」，作為課程發布時標籤自動邀請之資料基礎。
- **FR-ET-US1-08**: 系統 MUST 於新增某使用者 × 標籤時，自動將該使用者補加入該標籤所有「已發布且未關閉」課程（加入來源「標籤帶入」），並寄送彙整一封通知信（統一範本，列出所有新加入課程）。
- **FR-ET-US1-09**: 系統 MUST 於移除某使用者 × 標籤時維持既有課程學員名單不變動（已加入者可繼續學習）；系統 MUST NOT 對之後新發布之掛該標籤課程自動邀請該使用者。
- **FR-ET-US1-10**: 系統 MUST 於「參數設定」分頁列出所有受訓單位標籤（標籤名稱、啟用狀態、顯示順序），並含內建種子 5 筆（全體 / 護理師 / 行政人員 / 軍人 / 醫檢師）。
- **FR-ET-US1-11**: 系統 MUST 於新增標籤時檢核標籤名稱唯一後寫入 ET_TAG，使其可於使用者標籤指派與課程受訓單位標籤選單中選取。
- **FR-ET-US1-12**: 系統 MUST 於修改標籤名稱時，使既有使用者標籤對應與課程標籤同步顯示新名稱，對應關係不變。
- **FR-ET-US1-13**: 系統 MUST 支援標籤之停用與啟用；停用後該標籤不可掛至新課程（課程標籤選單不顯示），已掛之既有課程不受影響；啟用後恢復可掛至新課程。
- **FR-ET-US1-14**: 系統 MUST NOT 允許對「全體」標籤（IS_ALL 特殊標籤）執行停用或刪除；「全體」代表所有具「學員」角色之使用者，於查詢時展開、不需逐人貼標。

## 相關 Clarifications 摘錄

- ET 與 DM 共用平台 DP 之 `DP_USER` 但**各自管理自己的角色**：管理者於 ET 之角色指派僅影響 ET，不影響 DM
- **標籤不與 DM 共用**：DM 亦有「可見對象/單位」標籤（名目雷同），但兩系統使用者群相交而不相同，ET 自持 ET_TAG / ET_USER_TAG，各自維護
- 「全體」標籤代表**所有具「學員」角色之使用者**，不需逐人貼標（IS_ALL 於查詢時展開）
- 標籤與角色為**獨立兩個維度**：角色決定能做什麼、標籤決定隸屬哪個受訓單位
- 系統初始化第一個管理者由 IT 透過 DB 直接寫入；不提供應用層初始化介面
- 不設「至少保留 1 個啟用中管理者」之系統檢核
