# User Story 1 — UCET010 權限管理（ET07）

> 返回總檔：[spec.md](spec.md) | 模組：教育訓練文件管理（ET）

管理者透過 ET07 權限管理畫面設定其他帳號於 ET 之角色（管理者 / 教師 / 學員，三角色可重複指派）與業務模組對應（多對多，用於課程預設帶入）。角色變更後使用者下次登入即依新角色導向預設首頁。新使用者首次登入後系統自動授予「學員」角色，業務模組對應預設「未指派」。系統初始化之第一個管理者由 IT 透過 DB 直接寫入；不提供「最後一個管理者」之保護檢核（若意外造成 0 管理者狀態，由 IT 透過 DB 寫入恢復）。

**Priority**: P1

**Why this priority**: 權限管理為其他角色（教師 / 學員）取得 ET 存取之入口；無管理者完成角色指派，教師無法建立課程、學員無法被歸類至業務模組，整個系統運作停擺。

**Independent Test**: 由管理者登入後進入 ET07，對特定使用者勾選 / 取消角色與業務模組對應，儲存後該使用者下次登入即依新角色導向預設首頁；新註冊使用者自動以「學員」角色存在於清單中。

**Acceptance Scenarios**:

1. **Given** 管理者已登入，**When** 進入 ET07 權限管理頁，**Then** 系統列出所有使用者，每筆顯示三角色（管理者 / 教師 / 學員）核取方塊與業務模組對應
2. **Given** 管理者於使用者列勾選「教師」角色並儲存，**When** 該使用者下次登入，**Then** 系統依新角色導向預設首頁（教師 → 課程列表）
3. **Given** 管理者於使用者列展開「業務模組」設定並勾選「採血」、「成分」，**When** 儲存，**Then** 該對應寫入 ET_USER_MODULE；之後新建之該模組課程，會自動邀請該使用者
4. **Given** 當前登入之管理者嘗試**取消自己**之「管理者」角色核取方塊，**When** 點儲存，**Then** 系統阻擋並提示「無法停用自己之管理者角色」
5. **Given** 管理者 A 取消管理者 B 之「管理者」角色，**When** 儲存，**Then** 系統允許（不檢核「停用後系統是否仍有啟用中管理者」）
6. **Given** 一名新註冊使用者，**When** 管理者進入 ET07，**Then** 該使用者出現於清單中，預設角色「學員」、業務模組「未指派」
7. **Given** 管理者於使用者列變更角色或業務模組對應，**When** 儲存，**Then** 系統記錄修改者與時間於「最後修改」欄；完整變更歷史寫入稽核 log
8. **Given** 管理者於業務模組對應**新增**某使用者 × 採血，**When** 儲存，**Then** 系統自動將該使用者加入過去所有採血模組之已發布課程（業務模組變更規則）
9. **Given** 管理者於業務模組對應**移除**某使用者 × 採血，**When** 儲存，**Then** 既有採血課程之學員名單**不變動**；之後新建之採血課程，**不會**自動邀請該使用者

---

## 相關 Clarifications 摘錄

- ET 與 DM 共用 user table 但**各自管理自己的角色**：管理者於 ET 之角色指派僅影響 ET，不影響 DM
- 系統初始化第一個管理者由 IT 透過 DB 直接寫入；不提供應用層初始化介面
- 不設「至少保留 1 個啟用中管理者」之系統檢核

- 使用者主檔（USER）由 [spec_us2.md](spec_us2.md) US2 註冊或系統初始化寫入
