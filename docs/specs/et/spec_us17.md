# User Story 17 — UCET017 核可查詢（ET10）

> 對應 UC：UCET017 ｜ 功能選項：ET10（核可查詢）｜ Priority：P2 ｜ Wireframe：[核可查詢](../../wireframes/et/index.html) ｜ 返回總檔：[spec.md](spec.md)
> 2026-07-17 新增（客戶需求：線下核可查詢）。核可資料由 [spec_us16.md](spec_us16.md) US16 寫入。

提供**核可紀錄查詢**，依角色分兩種視角：

- **教師 / 管理者視角**：於「核可查詢」畫面**依學員姓名**查詢**全部學員**之核可課程，結果含課程名稱、核可結果（通過 / 不通過）、核可時間、核可人，以及是否已撤銷與撤銷原因。
- **學員視角**：學員登入後僅能查看**自己「已通過核可」**的課程清單（不顯示不通過 / 撤銷紀錄、不得查他人）。

本 US **不提供**核可證明 / 結業證書輸出（2026-07-17 客戶確認）。查詢屬唯讀，不含任何核可 / 撤銷動作（動作於 [spec_us16.md](spec_us16.md) US16）。

**Priority**: P2

**Why this priority**: 為客戶明確提出之「用姓名查學員通過課程」需求，與 US16 核可作業成對交付；屬查詢類唯讀作業。

**Independent Test**: 教師 / 管理者於核可查詢輸入學員姓名可查得該學員之核可課程（含通過 / 不通過 / 撤銷）；學員登入僅能查得自己「已通過」之課程，查不到不通過 / 撤銷紀錄、亦查不到他人；查無資料時顯示對應空狀態。

**Acceptance Scenarios**:

### 教師 / 管理者視角

1. **Given** 教師 / 管理者進入核可查詢畫面，**When** 輸入學員姓名（可模糊）查詢，**Then** 系統列出符合學員之核可課程：課程名稱、核可結果（通過 / 不通過）、核可時間、核可人
2. **Given** 某學員某課程之核可已被撤銷，**When** 教師 / 管理者查詢，**Then** 該筆顯示「已撤銷」狀態與撤銷原因、撤銷人 / 時間
3. **Given** 管理者查詢，**When** 檢視結果，**Then** 可查得**全部課程**之全部學員核可紀錄（不限自己建立之課程）
4. **Given** 教師查詢，**When** 檢視結果，**Then** 依權限可查得核可紀錄（教師可查範圍由 plan 階段確認：至少自己 owner 之課程；是否含全部由設計決策，預設可查全部學員之核可結果）
5. **Given** 查詢條件查無核可紀錄，**When** 系統回應，**Then** 顯示空狀態提示

### 學員視角

6. **Given** 學員登入後進入「我的核可 / 已通過課程」，**When** 系統載入，**Then** 僅顯示該學員**自己「已通過」（有效未撤銷）**之課程清單（課程名稱、核可時間）
7. **Given** 學員有「不通過」或「已撤銷」之核可紀錄，**When** 學員檢視，**Then** 該些紀錄**不顯示**（學員端僅見已通過）
8. **Given** 學員嘗試查詢他人之核可紀錄（竄改參數 / 直呼 API），**When** 系統處理，**Then** **拒絕**並不回傳他人資料
9. **Given** 學員目前尚無已通過核可之課程，**When** 進入畫面，**Then** 顯示空狀態提示

### 不提供證書

10. **Given** 任一角色檢視核可查詢結果，**When** 檢視畫面，**Then** **不提供**核可證明 / 結業證書之下載 / 列印功能

## Functional Requirements

- **FR-ET-US17-01**: 系統 MUST 提供核可查詢畫面（ET10）供教師 / 管理者依學員姓名（可模糊比對）查詢核可紀錄，結果 MUST 含課程名稱、核可結果（通過 / 不通過）、核可時間、核可人，撤銷者 MUST 標示「已撤銷」並顯示撤銷原因 / 撤銷人 / 時間
- **FR-ET-US17-02**: 管理者 MUST 可查得全部課程之全部學員核可紀錄；教師之可查範圍 MUST 至少涵蓋自己 owner 之課程（是否擴及全部學員於 plan 階段依權限設計最終確認）
- **FR-ET-US17-03**: 學員視角 MUST 僅顯示該登入學員本人**已通過（有效未撤銷）**之課程清單，MUST NOT 顯示不通過 / 已撤銷之紀錄，MUST NOT 顯示他人之任何核可資料
- **FR-ET-US17-04**: 系統 MUST 於後端以登入者身分控管查詢範圍，對學員存取教師 / 管理者查詢或查詢他人資料之請求 MUST 拒絕（不以前端隱藏為唯一防線）
- **FR-ET-US17-05**: 本 US MUST NOT 提供核可證明 / 結業證書之產生 / 下載 / 列印功能
- **FR-ET-US17-06**: 查無符合條件之核可紀錄時系統 MUST 顯示對應空狀態提示

## 相關 Clarifications 摘錄

- 教師 / 管理者可查**全部學員**；學員只能查**自己已通過**的課程（2026-07-17 客戶確認）
- **不需**核可證明 / 結業證書（2026-07-17 客戶確認）
- 學員端僅見「已通過」，不見不通過 / 撤銷紀錄（隱私與體驗考量）
- 核可資料來源為 ET_APPROVAL（[spec_us16.md](spec_us16.md) US16 寫入）；查詢唯讀
