# UI Design Guide — 頁面級 UI 規範

> 來源：`docs/todo/prototype.html`（Bootstrap 原型），實作時以 **MUI** 元件對應。
> 適用範圍：Phase 2（#95）逐頁遷移時，依此規範調整各功能頁面的交互與佈局。

---

## 1. 頁面整體結構

每個功能頁面由以下區塊組成：

```
┌─────────────────────────────────────────────┐
│ Screen Header（頁面標題列）                    │
│  左：Icon + 功能名稱（粗體 1.3rem）             │
│  右：頁面代碼 / meta 資訊（灰色 0.75rem）        │
│  底線：綠色 2px（戰時紅色）                      │
├─────────────────────────────────────────────┤
│ 功能區（依頁面類型不同，見下方分類）               │
└─────────────────────────────────────────────┘
```

### 頁面類型與區塊排列

| 頁面類型 | 區塊排列（由上到下） |
|---------|-------------------|
| **CRUD 列表頁** | 篩選/搜尋列 → 表格 → 分頁 |
| **掃碼作業頁** | 掃碼輸入區 → 資訊卡片 → 狀態訊息 → 動作區 → 紀錄表格 |
| **訂單/多步驟頁** | Tab 頁籤 or 步驟指示 → 各步驟面板 |
| **報表頁** | 篩選列 → 圖表區 → 數據表格 |
| **明細頁** | 資訊摘要卡片（多欄）→ 明細表格 → 狀態時間軸 → 返回按鈕 |

## 2. 卡片容器（`.wf-card` → MUI `Card` / `Paper`）

所有內容區塊以卡片包裹：

| 屬性 | 值 |
|------|-----|
| 背景 | 白色 |
| 邊框 | `1px solid #dee2e6` |
| 圓角 | `8px` |
| 內距 | `1rem`（Mobile: `0.75rem`） |
| 下距 | `1rem` |

**特殊卡片變體：**

| 變體 | 樣式 | 用途 |
|------|------|------|
| 表單卡片 | 綠色 2px 實線邊框，`max-width: 500~600px` | 新增/編輯表單 |
| 結案/取消卡片 | 黃色或紅色 2px 邊框 | 結案原因、取消確認 |
| 摘要卡片 | `bg-light` 淺灰背景 | 上方摘要資訊列 |

## 3. 篩選列（Filter Bar）

### 位置
表格上方，與「新增」按鈕同列或獨立一行。

### 佈局

```
┌──────────────────────────────────────────────────┐
│ 左側：搜尋框 / 下拉選單 / 日期範圍               右側：新增按鈕 │
└──────────────────────────────────────────────────┘
```

**兩種常見排列：**

1. **簡單搜尋 + 新增按鈕**（如層架管理）：
   - 左上：搜尋框（240px，帶搜尋 icon）
   - 右上：「新增」按鈕（主色）

2. **多條件篩選**（如訂單管理、召回管理）：
   - 水平排列：狀態下拉（140px）+ 日期起（150px）+ `~` + 日期迄（150px）+ 查詢按鈕
   - 使用 `flex-wrap`，Mobile 時垂直堆疊

### 搜尋框樣式
- 左側帶搜尋 icon（`InputAdornment`）
- 寬度：`240px`
- placeholder 具體描述：「搜尋層架編號或名稱...」

### 日期範圍
- 兩個 date input 中間用 `~` 文字分隔
- 各 `150px` 寬

### RWD
- Mobile（≤767px）：篩選列垂直堆疊，每個元素 `width: 100%`
- `~` 分隔符在 Mobile 時隱藏

## 4. 表格（Table）

### 基本樣式

| 屬性 | 值 |
|------|-----|
| 字體大小 | `0.88rem`（Mobile: `0.8rem`） |
| 表頭背景 | `#f8f9fa` |
| 表頭文字 | `white-space: nowrap` |
| Hover | 啟用列 hover 高亮 |
| 響應式 | 外層包 `table-responsive`（MUI: `TableContainer`） |

### 欄位模式

| 欄位類型 | 說明 |
|---------|------|
| `#` 序號欄 | 第一欄，純數字 |
| 名稱/編號欄 | 文字，可點擊進入明細的用 `<a>` 連結樣式 |
| 狀態欄 | Badge/Chip（見第 7 節） |
| 日期欄 | `YYYY-MM-DD` 格式 |
| 進度欄 | Progress Bar（`width:100px, height:18px`），顯示如 `3/5` |
| 操作欄 | 最右欄，放操作按鈕 |

### 列狀態高亮

整列依狀態上不同背景色：

| 背景色 | 用途 |
|--------|------|
| `table-warning`（淡黃） | 待處理、需注意 |
| `table-danger`（淡紅） | 逾期、錯誤 |
| `table-success`（淡綠） | 已完成、已掃碼 |
| `table-secondary`（淡灰） | 合計列 |

### 操作欄按鈕

```
[編輯] [停用]          ← 一般 CRUD
[明細] [取消]          ← 訂單類
[處理] / [查看]        ← 狀態依賴（待處理 vs 已完成）
```

- 按鈕大小：`btn-sm`（MUI: `size="small"`）
- 按鈕間距：`me-1`（`marginRight: 4px`）
- 「停用」已停用時變為「啟用」按鈕（顏色也換）

## 5. 新增/編輯表單

CRUD 頁面有兩種新增/編輯模式，依資料複雜度選擇：

| 模式 | 說明 | 適用場景 | 範例頁面 |
|------|------|---------|---------|
| **FormCard（卡片展開式）** | 表格下方展開綠色邊框表單卡片 | 欄位多、結構複雜 | 系統參數管理、選單管理 |
| **Inline Row Edit（行內編輯）** | 表格行內直接新增/編輯 | 欄位少（3~5 個）、設定類資料 | 層架管理、用血單位管理 |

### 模式 A：FormCard 卡片展開（預設）

```
表格卡片
─────────────
↓ 點擊「新增」或「編輯」
─────────────
表單卡片（綠色邊框，初始隱藏）
  標題：icon + 「新增 XXX」/「編輯 XXX」
  欄位...
  [儲存] [取消]
```

**關鍵行為：**
- 表單預設 `display: none`，點擊按鈕後顯示
- 綠色 2px 實線邊框
- `max-width: 500~600px`（不撐滿整頁）
- 新增/編輯共用同一個表單區塊，標題動態切換
- 「取消」隱藏表單

### 模式 B：Inline Row Edit 行內編輯

```
表格
─────────────
↓ 點擊「新增」
─────────────
[NEW] [input] [input] [select] [儲存] [取消]  ← 淺綠底 (#f0faf0) 新增列
  行1   資料    資料    資料    [編輯] [停用]
  行2   資料    資料    資料    [編輯] [停用]
```

**新增行為：**
- 點擊「新增」→ 表格 `tbody` 頂部插入一列
- 背景色：`#f0faf0`（淺綠），第一欄標記 `NEW`（綠色粗體）
- 各欄位直接是 `<input>` / `<select>`
- 操作欄：「儲存」「取消」

**編輯行為：**
- 點擊「編輯」→ 該列文字就地變成 `<input>` / `<select>`（保留原值）
- 操作欄從「編輯/停用」變為「儲存/取消」
- 「儲存」後寫回、「取消」還原

**驗證失敗：**
- 下方插入紅色錯誤提示列

### 表單欄位排版（適用 FormCard 模式）

**單欄直排**（簡單表單，3~4 個欄位）：
```
分類代碼 *    [___________]
顯示名稱 *    [___________]
排序 *        [___________]
[儲存] [取消]
```

**多欄橫排**（複雜表單，5+ 個欄位）：
```
所屬醫院 *  [▼____]     角色 *      [▼____]
帳號 *      [________]  姓名 *      [________]
Email *     [________]  聯絡電話    [________]
[儲存] [取消]
```

- 使用 `Grid`（MUI）實現多欄：2 欄 = `col-md-6`，3 欄 = `col-md-4`，4 欄 = `col-md-3`
- Label：小字（`form-label small`）
- 必填標記：Label 後方紅色 `*`（`<span style="color: error.main">*</span>`）
- 輸入框：小尺寸（MUI: `size="small"`）

### 按鈕排列
- 水平排列，主要在左、取消在右
- 「儲存」= 主色按鈕，「取消」= 灰色外框按鈕

## 6. 列表→明細→返回 流程

### Panel 切換（非跳頁）

```
列表面板（預設顯示）
  ├── 篩選列
  ├── 表格（含「明細」按鈕）
  └── 分頁

  ↓ 點擊「明細」

明細面板（隱藏列表，顯示明細）
  ├── 摘要資訊（多欄橫排）
  ├── 明細表格
  ├── 相關記錄
  ├── 狀態時間軸
  └── [← 返回列表] [其他操作按鈕]
```

**實作方式：**
- 兩個 `<div>` 互斥顯示（`display: none` 切換）
- 或使用 React state 控制：`view: 'list' | 'detail'`
- 「返回列表」按鈕在面板底部左側

### 明細面板摘要區

多欄橫排，每欄結構：
```
[灰色小字 Label]
[粗體值 / Badge]
```
- 通常 4 欄（`col-md-3`）

## 7. 狀態 Badge / Chip

| 狀態 | 顏色 | MUI 對應 |
|------|------|----------|
| 啟用 / 成功 / 庫存中 | `bg-success`（綠） | `color="success"` |
| 停用 / 已取消 | `bg-secondary`（灰） | `color="default"` |
| 訂單成立 / 院內 | `bg-primary`（藍） | `color="primary"` |
| 已通知 / 警告 | `bg-warning`（黃） | `color="warning"` |
| 逾期 / 錯誤 / 召回 | `bg-danger`（紅） | `color="error"` |
| 已領用 / 跨院 | `bg-info`（淺藍） | `color="info"` |
| 待回覆 | `bg-light` + 灰框 | `variant="outlined"` |
| 已取消（深色） | `bg-dark` | 自訂 |

**血型 Badge（大）：** `bg-danger` + 較大字體（`fs-6`），如 `A+`

| 類型 | 背景 | 邊框 | Icon |
|------|------|------|------|
| 成功 | `#e8f5e9` 淺綠 | `#a5d6a7` | `CheckCircle` |

| 警告 | `#fff3e0` 淺橘 | `#ffcc80` | `Warning` |

格式：`[Icon] 訊息文字`，圓角 `6px`，`padding: 0.5rem 1rem`

## 10. Tab 頁籤

用於同一頁面內的功能分組（如訂血作業：新增訂血 / 我的訂單）：

- MUI: `Tabs` + `Tab` 元件
- 位於頁面標題下方
- 切換時 Panel 互斥顯示
- 不是路由跳轉，是同頁面內的面板切換

---

## 11. 步驟指示列（Stepper）

用於多步驟流程（如出庫作業：選擇訂單 → 掃碼出庫 → 出庫完成）：

```
[1 選擇訂單] → [2 掃碼出庫] → [3 出庫完成]
```

- 當前步驟：綠色 Badge + 粗體文字
- 未達步驟：灰色 Badge + 灰色文字
- 步驟間用 `→` 箭頭連接
- MUI: `Stepper` + `Step` + `StepLabel`

---

## 12. 狀態時間軸

顯示一筆資料的狀態變更歷程：

```
[庫存中]  →  [已供出]  →  [已領用]  →  [已使用]
 03-12       03-17        03-17       待登記
```

- 水平排列，每個節點：Badge + 下方時間
- 已完成：彩色 Badge
- 未完成：灰色外框 Badge
- 節點間用 `→` 圖示分隔
- 使用 `flex-wrap`，小螢幕時自動換行

## 13. 摘要統計卡片

用於頁面頂部快速瀏覽（如召回管理、API 紀錄）：

```
┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐
│  3  │  │  5  │  │  2  │  │ 12  │
│待通知│  │已通知│  │逾期  │  │已完成│
└─────┘  └─────┘  └─────┘  └─────┘
```

- 4 欄橫排（`col-md-3`），Mobile 時 2 欄（`col-6`）
- 大字數字 + 小字說明
- 數字可帶顏色（紅=危急、黃=注意、綠=正常）

## 14. 提示資訊框

### Alert 提示
- MUI: `Alert` 元件
- 小字體
- 常見：`alert-info`（藍）、`alert-danger`（紅）、`alert-warning`（黃）
- 格式：`[Icon] 提示文字`

### 左邊色條提示框
- 左邊 `3px` 黃色 border
- 淺灰背景
- 用於條件說明（如「若選擇已使用，需登記用血人資訊」）

## 15. 按鈕規範

### 顏色對照

| 用途 | Prototype class | MUI 對應 |
|------|----------------|----------|
| 主要動作（儲存、確認、查詢） | `.btn-tsbms`（綠色） | `variant="contained" color="primary"` |
| 次要動作（取消、返回） | `.btn-outline-secondary` | `variant="outlined"` |
| 危險動作（刪除、停用、取消訂單） | `.btn-outline-danger` | `variant="outlined" color="error"` |
| 警告動作（結案、重設密碼） | `.btn-outline-warning` | `variant="outlined" color="warning"` |
| 資訊動作（查看帳號） | `.btn-outline-info` | `variant="outlined" color="info"` |
| 啟用（反向操作） | `.btn-outline-success` | `variant="outlined" color="success"` |
| 確認性危險動作（確認取消、確認銷毀） | `.btn-danger` | `variant="contained" color="error"` |

### 按鈕格式
- Icon + 文字：`<Icon sx={{ mr: 0.5 }} /> 文字`
- 大小：一般 `size="small"`，掃碼確認可用預設大小
- 不可用動作顯示灰色文字替代按鈕（如「不可取消」）

### 按鈕位置

| 位置 | 說明 |
|------|------|
| 表格上方右側 | 「新增」按鈕 |
| 表格列操作欄 | 「編輯」「停用」「明細」等 |
| 表單下方左對齊 | 「儲存」「取消」 |
| 明細頁底部 | 「返回列表」+ 其他操作 |
| 完成畫面居中 | 「列印」「返回」 |

## 21. 確認性操作（非 Modal）

刪除/取消/結案等危險操作：

**模式 A：直接操作**（停用/啟用）
- 按鈕直接切換狀態，無確認步驟（prototype 設計）
- 實作時可加 `confirm` 對話框（MUI Dialog）

**模式 B：展開確認卡片**（結案、取消出庫）
- 點擊按鈕後，在下方展開一個帶顏色邊框的卡片
- 包含原因 textarea + 確認/取消按鈕
- 如：結案原因卡片（黃色邊框）、取消確認卡片（紅色邊框）

## 22. 響應式設計

### 斷點

| 斷點 | 寬度 | 變化 |
|------|------|------|
| Desktop | > 992px | 完整佈局 |
| Tablet | ≤ 991px | Sidebar → overlay drawer |
| Mobile | ≤ 767px | 篩選列垂直堆疊、表格縮小、內距減少 |
| XS | ≤ 575px | 頁面標題縮小、最小內距 |

### Mobile 調整

| 元素 | 調整 |
|------|------|
| 篩選列 | 垂直堆疊，全寬 |
| 表格字體 | `0.8rem` |
| 表格 cell padding | 減少 |
| 卡片內距 | `0.75rem` |
| 掃碼區 | 輸入框 `max-width: 100%` |
| 統計卡片 | 4 欄 → 2 欄 |
| `hide-sm` 元素 | 隱藏 |
