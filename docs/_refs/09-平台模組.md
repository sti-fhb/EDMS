# 09 — 平台模組（DP）

**文件狀態：** 草稿（Draft）
**建立日期：** 2026-07-06
**來源：** 裁剪自 TBMS `_refs/08-資訊模組.md`，依 EDMS 2026-07-06 決策精簡；2026-07-08 依「參數/範本/排程/設定集中、統一管理後台」再修（見文末「決策紀錄」）

> 本模組為 **EDMS 的共用平台層 + 統一管理後台**，非業務模組。ET（教育訓練）與 DM（文件管理）共用其帳號、認證、稽核，以及系統參數 / 清單定義 / 通知範本 / 排程之**儲存與引擎**；各模組的業務**判定與關聯資料**（角色能做什麼、誰配哪個標籤/角色/分類）仍留各模組。表名沿用 `DP_` 前綴。

## 1. 模組概述

平台模組（DP）提供 EDMS 全系統共用的基礎功能：**使用者帳號與登入認證、忘記密碼、密碼變更、使用者管理、帳號鎖定、系統參數與清單定義（全平台集中，前綴分模組）、通知範本與發信引擎、排程引擎、操作記錄（稽核）**，並作為**統一管理後台**——所有管理畫面（參數、分類/標籤/func_name 等清單、通知範本、角色指派）集中於此、按模組過濾。它不處理業務邏輯（課程、文件皆屬 ET / DM），也不做業務判定（角色能用哪些功能由各模組 enforce）。

### 1.1 定位與邊界

- **共用帳號、權限分開（SSO）**：ET 與 DM 共用同一份使用者帳號（`DP_USER`），使用者註冊 / 登入一次即可存取兩系統（SSO 指認證面：一次登入通用）。角色**種類**（ET 管理者/教師/學員、DM 管理者/編輯者/審核者/閱覽者）與**權限判定**（某角色能做什麼）由各模組自理、各模組 enforce——**平台不做全域 RBAC**。
- **集中儲存與引擎、統一後台維護**：系統參數、清單定義（分類/標籤/func_name）、通知範本、排程，其**資料儲存與執行引擎集中於平台**；**維護畫面統一於 DP 系統管理後台**，按模組過濾（ET 管理者只見 ET 的、DM 管理者只見 DM 的）。集中的是「儲存 + 引擎 + 操作畫面」。
- **定義 vs 關聯 分層（重要）**：「**有哪些**」的清單定義（分類清單、標籤名稱、func_name、參數值）存平台 `DP_PARAM`；「**誰配誰**」的關聯 / 指派（課程×標籤、使用者×標籤、使用者×角色、使用者×可見對象、文件屬哪分類）與**角色判定**留各模組業務表，引用 DP_PARAM 的碼。

  | 有哪些（定義）→ 平台 DP_PARAM | 誰配誰（關聯/判定）→ 模組 |
  |------|------|
  | 分類清單、func_name、可見對象清單、標籤名稱 | 文件屬哪分類、閱覽者被授權哪些對象、課程×標籤、使用者×標籤 |
  | 角色「種類」（固定 enum）| 使用者×角色指派（`ET_USER_ROLE` / `DM_USER_ROLE`）、角色能做什麼之判定 |
  | 各類參數值、密碼策略、排程時間 | — |

  > 口訣：**「有哪些」搬 DP、「誰配誰」留模組；畫面全在 DP 後台、判定在模組。**

- **稽核集中（限資安類）**：登入登出、帳號建立/停用、角色/權限異動、系統參數/清單異動等**資安稽核事件**，ET / DM 統一寫入 `DP_AUDIT_LOG`（平台共用一張）；各模組之**業務歷程**（DM 文件變更歷程、閱讀紀錄、ET 學習/作答紀錄）仍留各模組、面向使用者查詢，不併入本表。
- **通知範本與發信引擎集中**：範本統一存 `DP_NOTIFY_TEMPLATE`（`MODULE` 欄分 `DP`/`ET`/`DM`），平台提供**唯一發信服務**與寄件 outbox（`DP_EMAIL_LOG`）；各模組傳 `template_code` 呼叫，不自持範本表 / 不自建 outbox。
- **排程集中：引擎 + DB 註冊表**：平台提供排程引擎（APScheduler）＋ `DP_SCHEDULE` 註冊表 ＋ `DP_SCHEDULE_LOG` 執行歷程；各模組 job handler 向引擎註冊（ET SCHET001/002、DM SCHDM001），需要業務資料時反向 import 模組 service。

### 1.2 操作角色

平台**不另設**「平台管理員」角色；DP 系統管理後台由 **ET 管理者 / DM 管理者**進入，靠模組過濾決定可見與可改內容。

| 角色 | 說明 |
|------|------|
| ET 管理者 | ET 業務角色（`ET_USER_ROLE`）。於 DP 後台維護 **ET 相關**（前綴 `ET_`）之參數 / 清單 / 範本 / ET 角色指派；並可存取**共用項**（見下） |
| DM 管理者 | DM 業務角色（`DM_USER_ROLE`）。於 DP 後台維護 **DM 相關**（前綴 `DM_`）之參數 / 清單 / 範本 / DM 角色指派；並可存取**共用項** |
| 一般使用者 | 登入、登出、忘記密碼、變更密碼、維護自己的基本資料 |
| eMail Server | 外部郵件系統，平台發信引擎透過它寄發所有信件（密碼重設 / 帳號驗證 / 各模組業務通知）|

**可見與存取規則：**

- **模組項（按模組隔離）**：前綴 `ET_` / `DM_` 的參數與清單、`MODULE=ET`/`DM` 的範本、各模組角色指派——ET 管理者只見 ET 的、DM 管理者只見 DM 的，**互不可見**。
- **共用項（兩管理者皆可見/可改）**：**帳號管理**（`DP_USER` 為兩模組共用）、**平台級參數**（無前綴 `DP_`：密碼策略、JWT、鎖定、發信調校）、**操作記錄（稽核）**。此三者不分模組，任一管理者皆可存取（2026-07-08 決策）。

> 注意：帳號為共用，故 ET 管理者停用一個帳號會同時影響 DM（反之亦然）；平台級參數（如密碼策略）任一管理者皆可改、影響全平台。此為「兩管理者共管平台」之取捨。

### 1.3 登入機制（簡單 JWT）

- 使用者於登入頁輸入帳號（Email）+ 密碼；驗證通過後系統核發一顆 **JWT access token**（TTL 由參數控制），前端後續請求以 `Authorization: Bearer <token>` 認證。
- **不採 MFA**（無二次驗證碼）、**不採 Refresh Token / Session Rotation**（無伺服器端 session 表）；登出＝前端丟棄 token。
- **無站點/工作站選擇、無 IP 設備識別**（EDMS 為單一組織，無多站點概念）。
- 帳號共用：同一帳號可登入 ET 與 DM；登入後由各模組依自己的角色決定可用功能與預設首頁。
- 新使用者**自助註冊即可用**（對齊 ET：註冊後自動取得該模組預設角色，如 ET 的「學員」）；管理者亦可代為建立帳號。

### 1.4 標準稽核欄位（全表適用）

EDMS 為單一組織，**不含站點/院區維度**。各表標準欄位統一為：

`CREATED_USER`、`CREATED_DATE`、`UPDATED_USER`、`UPDATED_DATE`、`RES_ID`、`DELETED`

- 預設**軟刪除**（`DELETED = 1`），查詢預設加 `WHERE DELETED = 0`
- append-only 記錄表（如 `DP_AUDIT_LOG`、`DP_PWD_HIST`）僅含 `CREATED_USER` / `CREATED_DATE`，無 `UPDATED_*` / `DELETED`

## 2. 功能需求

### 2.1 使用者登入

- 登入頁欄位：帳號（Email）、密碼。
- 驗證流程：
  1. 查 `DP_USER` 驗證帳密
     - 帳號不存在 → 提示「查無此帳號，請先註冊」
     - 密碼錯誤 → 累計失敗次數；達上限自動鎖定（見 §2.5）
     - 通過 → 核發 JWT，寫入登入操作記錄（`DP_AUDIT_LOG`，類別 LOGIN）
  2. 登入後由各模組依角色導向預設首頁（ET：管理者→ET07 / 教師→ET01 / 學員→ET04；DM 由 DM 決定）
- 登出：前端丟棄 token；寫登出記錄（類別 LOGOUT）。

### 2.2 忘記密碼

- 使用者於登入頁點「忘記密碼」，輸入帳號（Email）。
- 系統驗證帳號存在後，產生密碼重設 token（含有效期限，TTL 由參數控制，預設 30 分鐘），寄重設信至該 Email。
- **防帳號列舉**：不論帳號是否存在，皆回覆相同訊息（「若該 Email 已註冊，密碼重設信將寄至信箱」）。
- 使用者點信中連結 → 驗證 token 有效性與期限 → 輸入新密碼（驗證複雜度與重複性）→ 更新密碼、記錄操作歷程。

### 2.3 密碼變更

- 已登入使用者主動變更密碼：輸入舊密碼 + 新密碼 + 確認新密碼。
- 系統驗證舊密碼正確、新密碼兩次一致、符合複雜度與重複性規則後更新。
- 密碼一律以雜湊儲存（bcrypt 或等效），不可逆。

### 2.4 使用者 / 人員管理（共用項）

- **自助註冊**：新使用者於登入頁註冊（帳號 Email、姓名、密碼），成功後自動取得所屬模組預設角色（如 ET 學員），即可登入。
- **管理者維護**：ET / DM 管理者（任一）於 DP 後台可建立帳號、停用（Disable）/ 啟用（Enable）帳號、維護基本資料（姓名、Email）、手動解鎖。帳號為兩模組共用項，**任一管理者皆可操作**。
- **角色指派不在此頁**：帳號管理只處理帳號本身；ET / DM 之角色指派於 DP 後台「權限管理」畫面另行處理（見 §2.8），寫入各模組角色表。
- 帳號（Email）變更採「新 Email 驗證後切換」延遲生效（對齊 ET US10：寄驗證信至新 Email，點連結後生效；期間舊 Email 仍可登入）。
- 帳號停用後於 ET / DM 兩端同步失效；所有帳號異動寫入操作記錄。

> **已確認（2026-07-06）**：帳號啟用採「自助註冊即用」（對齊 ET US2）——註冊後即可登入、自動取得所屬模組預設角色；管理者代建之帳號亦**直接啟用可用**，不另寄「帳號開通確認信」。

### 2.5 帳號鎖定管理

- 連續登入失敗達設定次數（參數，預設 5）→ 自動鎖定帳號。
- 解鎖方式：**管理者手動解鎖** 或 **逾時自動解鎖**（等待時間為參數，預設 30 分鐘）。
- 鎖定 / 解鎖寫入操作記錄。

### 2.6 系統管理後台（統一）

DP 提供單一「系統管理」後台，集中所有管理畫面；操作者為 ET / DM 管理者，畫面內容按模組過濾（見 §1.2）。後台涵蓋：

| 管理畫面 | 資料落點 | 過濾 |
|---------|---------|------|
| 系統參數（平台級 + 模組級）| `DP_PARAM`（平台級服務讀寫）| 平台級共用；`ET_`/`DM_` 按模組 |
| 清單定義（分類 / func_name / 標籤名稱 / 可見對象值 / DM 檢索標籤）| `DP_PARAM`（前綴分模組）| 按模組 |
| 通知範本 | `DP_NOTIFY_TEMPLATE`（`MODULE` 分模組）| 按模組（`DP` 系統信兩者皆見、僅可編輯不可停用）|
| 權限管理（角色 + 標籤/可見對象指派）| 呼叫模組 service 寫 `ET_USER_ROLE` / `DM_USER_ROLE` + `ET_USER_TAG` / DM 可見對象授權（見 §2.8）| 按模組 |
| 帳號管理 / 操作記錄 | `DP_USER` / `DP_AUDIT_LOG`（共用）| 共用 |

**系統參數與清單定義（存 `DP_PARAM`）：**

- **儲存集中**：全平台參數值與清單定義統一存 `DP_PARAM_M/D`，以 `PARAM_ID` 前綴（無前綴 `DP_` 平台級 / `ET_` / `DM_`）區分歸屬。清單型（如分類、標籤名稱）以一個 `PARAM_ID` 下多筆 `PARAM_KEY` 表達；需鎖定之碼（如文件分類碼建立後不可改）以 `DP_PARAM_M.DETAIL_LOCK` 標記。
- **唯讀服務**：平台提供參數查詢服務（`get_param_values(param_id, key)`），供各模組讀取；模組讀自己前綴之定義（如 DM 讀 `DM_DOC_CATEGORY` 組出分類下拉、產生 `DOC_ID`）。
- **維護**：於 DP 後台，平台級（`DP_`）兩管理者共用可改；模組級（`ET_`/`DM_`）僅該模組管理者可見可改。
- **業務規則歸模組**：清單「定義」在 DP，但套用清單的**業務規則**（分類碼嵌入 `DOC_ID`、`func_name` 唯一手冊檢核、標籤「全體」`IS_ALL` 展開與自動邀請）仍由各模組於讀取後執行。
- 參數與清單清單見 §5。

### 2.7 通知範本與發信引擎（集中）

- **範本集中**：所有通知範本存 `DP_NOTIFY_TEMPLATE`，`MODULE` 欄（`DP` / `ET` / `DM`）+ `TEMPLATE_CODE` 區分；欄位含主旨、內文、可用變數、管道（Email / 站內 / 兩者）、啟用停用、版本（樂觀鎖）。內容不同的通知＝同表不同列。
- **發信引擎集中**：平台提供唯一發信服務 `send_email(recipients, template_code, params)` 與寄件 outbox `DP_EMAIL_LOG`（非同步批次、狀態、重試、`CALLER_MODULE`）；各模組只呼叫發信服務並傳 `template_code`，不自持範本表、不自建 outbox。
- **編輯 UI 於 DP 後台、按 MODULE 過濾**：ET 管理者編輯 `MODULE=ET` 範本、DM 管理者編輯 `MODULE=DM`；DP 自身系統信（密碼重設 / 帳號變更驗證，`MODULE=DP`）兩管理者皆可見、**可編輯主旨 / 內文但不可停用 / 刪除**（帳號安全信）。
- 發信引擎之調校參數（重試上限、每分鐘速率、失敗告警比例）屬平台級 `DP_` 參數（見 §5.1）。

### 2.8 權限管理（角色 + 標籤 / 可見對象指派）

- **畫面在 DP 後台**：ET / DM 管理者於後台「權限管理」畫面**同時**指派使用者角色與標籤（對齊 ET / DM 現行權限管理），按模組過濾（ET 管理者只見/只改 ET）：
  - **ET**：角色（管理者/教師/學員）＋**受訓單位標籤**（用於課程發布自動邀請）。
  - **DM**：角色（管理者/編輯者/審核者/閱覽者）＋**可見對象/單位授權**（主要對「閱覽者」有意義，決定其 DM01 可見範圍）。
  - **角色與標籤為獨立維度**，可各自獨立設定（可只給角色不給標籤、反之亦然）。
- **資料與判定留模組**：角色寫 `ET_USER_ROLE` / `DM_USER_ROLE`；標籤指派寫 `ET_USER_TAG`、可見對象授權寫 DM 授權表。「某角色能做什麼」之判定與 enforce 由各模組負責；平台**不做全域 RBAC**、不定義業務角色能力。標籤/可見對象之**定義清單**在 `DP_PARAM`（此頁只做「指派」＝誰配誰）。
- 角色種類為固定 enum（無「新增角色種類」功能）；同一使用者可多角色，權限取聯集。
- 角色 / 標籤指派異動屬資安事件，寫入 `DP_AUDIT_LOG`。

### 2.9 操作記錄（稽核，共用項）

- 系統記錄帳號建立、密碼重置、角色/資料異動、登入登出等操作歷程，供資安稽核。
- **平台共用一張** `DP_AUDIT_LOG`：ET / DM 的**資安稽核事件**（帳號 / 角色權限 / 系統操作）亦寫入本表統一查詢；業務歷程（DM 文件變更歷程等）不在此、留各模組。
- **可見範圍**：稽核為共用項，**ET / DM 管理者皆可查全部**（含登入等不分模組之事件；資安監督需要全視野）。
- 記錄內容：序號（PK）、操作者、功能/模組代碼、操作類別（LOGIN / LOGOUT / CREATE / UPDATE / DELETE）、來源 IP、時間、異動對象代碼、異動前後值（JSONB）。
- 稽核日誌 **append-only**，不可竄改、不可刪除。提供多條件查詢與匯出（CSV）。

### 2.10 排程執行基礎建設（引擎 + DB 註冊表，集中）

- 平台提供**單一排程執行引擎**（如 APScheduler）＋ **`DP_SCHEDULE` 註冊表**（`JOB_ID`、cron 表達式、執行程式參照、啟停、上次執行時間 / 結果）；多實例部署時以 leader 選舉確保只有一個實例觸發（單一實例可簡化）。
- 排程作業：ET 週報 / 未看提醒（SCHET001）、ET 到期關閉 + 加急提醒（SCHET002）、DM KPI 週報 + 未讀提醒（SCHDM001）等，於註冊表登錄 job 定義；job handler 由各模組提供並向引擎註冊。
- 平台記錄 **`DP_SCHEDULE_LOG` 排程執行歷程**（起訖時間、成功/失敗、錯誤訊息）供查詢與補跑判斷。
- 排程時間等參數存 `DP_PARAM`（前綴分模組，如 `ET_WEEKLY_STAT_DAY_TIME`、`DM_WEEKLY_SCHED_DAY_TIME`），於 DP 後台由各模組管理者維護。
- **依賴方向**：job 需要業務資料時反向 import 各模組 service（平台 job → 模組 service）；實作層 job 程式碼落點於 `docs/specs/dp/` 定案（可比照 TBMS 集中於平台命名空間 + 白名單）。

| 使用者主檔 | DP_USER | 主檔 | USER_ID | 帳號（Email）/ 密碼雜湊 / 姓名 / 狀態（啟用/停用）/ 登入失敗計數 / 鎖定狀態 / 最後登入；Email 變更之 PENDING 欄位（對齊 ET US10）。**無平台管理員旗標**（不另設平台管理員）|

| 密碼歷程 | DP_PWD_HIST | 明細（append-only）| USER_ID + SEQ_NO | 前 N 次密碼雜湊，供密碼重複性檢核（精簡版）|
| 操作歷程 LOG | DP_AUDIT_LOG | 明細（append-only）| LOG_ID | 平台共用稽核日誌；ET / DM 皆寫入、皆可查 |
| 功能參數主檔 | DP_PARAM_M | 主檔 | PARAM_ID | 全平台參數 / 清單定義分類（前綴無=平台級 / `ET_` / `DM_`）；`DETAIL_LOCK` 表達鎖定碼 |
| 功能參數明細 | DP_PARAM_D | 明細 | PARAM_ID + PARAM_KEY | 全平台參數值與清單項（分類 / func_name / 標籤名稱 / 可見對象值 皆為此表之列）|
| 通知範本 | DP_NOTIFY_TEMPLATE | 主檔 | MODULE + TEMPLATE_CODE | 全平台通知範本（`MODULE` 分 `DP`/`ET`/`DM`）；主旨 / 內文 / 變數 / 管道 / 啟停 / 版本 |
| 寄件 outbox / 發信 LOG | DP_EMAIL_LOG | 明細（append-only）| MESSAGE_ID | 平台發信引擎之寄件佇列與結果（狀態 / 重試 / 錯誤 / `CALLER_MODULE`）；取代 DM_NOTIFY_QUEUE |
| 排程註冊表 | DP_SCHEDULE | 主檔 | JOB_ID | 排程定義（cron / 執行程式 / 啟停 / 上次執行）；各模組 job 於此登錄 |

> **關聯 / 指派資料不在 DP**：課程×標籤、使用者×標籤、使用者×角色（`ET_USER_ROLE`/`DM_USER_ROLE`）、使用者×可見對象授權、文件屬哪分類等，皆為 FK 到業務記錄之關聯，**留各模組**、引用 DP_PARAM 之碼（見 §1.1 定義 vs 關聯）。

**明確不納入**（相對 TBMS 08 已裁剪）：

- ❌ `DP_SITE`（站點）/ 多站點架構 — EDMS 單一組織
- ❌ `DP_ROLE` / `DP_USER_ROLE` / `DP_MENU` / `DP_ROLE_MENUS`（全域 RBAC 與選單、業務角色能力定義）— 權限完全分開做，角色能力定義 / 指派資料 / 判定皆由各模組（ET/DM）自理，平台不碰（角色**指派畫面**在 DP 後台，但寫入模組角色表）
- ❌ `DP_SESSION`（伺服器端 session / Refresh Token）— 採無狀態簡單 JWT
- ❌ `DP_COMPDEVICE`（資訊設備 / IP 對照）、標籤印表機對照 — EDMS 不需要
- ❌ MFA 相關（無二次驗證）
- ❌ 外部系統 API（APIDP001/002）— 目前無對外 API 需求
- ❌ **平台管理員旗標**（`IS_PLATFORM_ADMIN`）— 不另設平台管理員；DP 後台由 ET / DM 管理者進入（2026-07-08 決策）

> **2026-07-08 納入調整**：`DP_NOTIFY_TEMPLATE`、`DP_EMAIL_LOG`、`DP_SCHEDULE`（DB 註冊表）由「原不納入 / 各模組自持」改為**納入平台集中**；`DP_PARAM` 範圍擴及**清單定義**（分類 / func_name / 標籤名稱 / 可見對象值）。連帶廢除各模組 `ET_PARAM` / `DM_PARAM` / `ET_NOTIFY_TEMPLATE` / `DM_NOTIFY_TEMPLATE` / `DM_NOTIFY_QUEUE` 及 `ET_TAG` / DM 分類等**定義表**（改為 DP_PARAM 清單；M2M / 指派關聯表仍留模組）。

## 4. 跨模組介接

| 介接對象 | 介接方式 | 說明 |
|---------|---------|------|
| ET / DM | 共用帳號 | 兩模組共用 `DP_USER`；登入後各自依角色載入功能；帳號管理為共用項（任一管理者可操作）|
| ET / DM | 資安稽核 | 資安事件統一寫入 `DP_AUDIT_LOG`，兩管理者皆可查；業務歷程留各模組 |
| ET / DM | 系統參數 + 清單定義 | 值與清單（分類/標籤/func_name/可見對象）存 `DP_PARAM`（前綴分模組）；平台提供唯讀查詢服務，各模組讀自己前綴、套用業務規則；維護畫面在 DP 後台按模組過濾 |
| ET / DM | 通知範本 + 發信 | 範本存 `DP_NOTIFY_TEMPLATE`（`MODULE` 分模組）；各模組呼叫平台唯一發信服務並傳 `template_code`，經 `DP_EMAIL_LOG` outbox 非同步寄送 |
| ET / DM | 角色 + 標籤指派 | 指派畫面在 DP 後台（按模組過濾），角色寫 `ET_USER_ROLE` / `DM_USER_ROLE`、標籤/可見對象寫 `ET_USER_TAG` / DM 可見對象授權；判定與 enforce 在模組 |
| ET / DM | 排程 | 各模組 job 於 `DP_SCHEDULE` 註冊、由平台引擎執行，`DP_SCHEDULE_LOG` 記錄；job 需要業務資料時反向 import 模組 service |
| 外部 eMail 系統 | SMTP | 平台發信引擎透過它寄發**所有**信件（密碼重設 / 帳號驗證 / ET / DM 業務通知），統一走 `DP_NOTIFY_TEMPLATE` 範本 |

> **對齊事項（ET / DM data-model 需配合）**：
> - 共用使用者表統一為 `DP_USER`；DM 資安類 `DM_USER_ROLE_LOG`（角色異動）對齊 `DP_AUDIT_LOG`；業務歷程 `DM_CHANGE_LOG` / `DM_DOC_READ` 留 DM。
> - **移除定義表改引用 DP_PARAM**：`ET_PARAM` / `DM_PARAM`（→ `DP_PARAM` 前綴 `ET_`/`DM_`）、`ET_NOTIFY_TEMPLATE` / `DM_NOTIFY_TEMPLATE`（→ `DP_NOTIFY_TEMPLATE`）、`DM_NOTIFY_QUEUE`（→ `DP_EMAIL_LOG`）、`ET_TAG` 標籤名稱 / DM 文件分類 / func_name / 可見對象值等**清單定義**（→ `DP_PARAM` 清單）。
> - **保留於模組**：`ET_USER_TAG` / `ET_COURSE_TAG`（課程/使用者×標籤）、`ET_USER_ROLE` / `DM_USER_ROLE`（角色指派）、DM 使用者×可見對象授權、文件之分類歸屬——引用 DP_PARAM 之碼。密碼重設 TTL 等認證參數改列平台級 `DP_`。

## 5. 可配置參數與清單

> 全平台參數與清單統一存 `DP_PARAM_M/D`，以 `PARAM_ID` 前綴分模組。§5.1 平台級（無前綴，兩管理者共用）；§5.2 模組級（`ET_`/`DM_`，按模組過濾）。帳號類信件已改為範本（`DP_NOTIFY_TEMPLATE`），不列為參數。

### 5.1 平台級參數（無前綴 `DP_`，兩管理者共用）

沿用 TBMS 命名慣例（如 `LOGIN`、`PWD_POLICY`、`ACTION_TYPE`）：

| PARAM_ID（示意）| 說明 | 預設值建議 |
|------|------|-----------|
| `JWT` | access token TTL、閒置自動登出（含敏感資料系統 ≤ 15 分）| TTL 8 小時 / 閒置 15 分 |
| `PWD_POLICY` | 密碼最小長度（一般 8 / 特權 12）、字元組合（≥3 種）、歷史記憶次數（3）、效期（90 天）| — |
| `LOGIN` | 登入失敗鎖定次數（5）、帳號鎖定時間（30 分）、閒置帳號禁用天數（90）、密碼重設連結有效時間（30 分）| — |
| `MAIL` | 發信重試上限、每分鐘速率、失敗告警比例 | （待定）|
| `ACTION_TYPE` | 操作類別代碼表（LOGIN/LOGOUT/CREATE/UPDATE/DELETE）| — |

### 5.2 模組級參數與清單（`ET_` / `DM_` 前綴，按模組過濾）

| PARAM_ID | 模組 | 型態 | 說明 |
|---------|------|------|------|
| `ET_VIDEO_ALLOWED_FORMATS` | ET | 參數 | 允許之影片格式 |
| `ET_VIDEO_MAX_SIZE_MB` | ET | 參數 | 影片大小上限（MB）|
| `ET_WEEKLY_STAT_DAY_TIME` | ET | 參數 | SCHET001 每週統計 / 週報執行時間 |
| `ET_URGENT_REMIND_DAYS` | ET | 參數 | SCHET002 截止前加急提醒天數 |
| `ET_TRAINING_UNIT` | ET | **清單** | 受訓單位標籤**名稱清單**（全體 / 護理師 / 行政人員 / 軍人 / 醫檢師…；「全體」為 `IS_ALL` 特殊項）|
| `DM_DOC_CATEGORY` | DM | **清單** | 文件分類**清單**（分類碼 + 名稱；分類碼建立後鎖定＝`DETAIL_LOCK`）|
| `DM_FUNC_NAME` | DM | **清單** | 關聯作業項目 func_name **清單** |
| `DM_AUDIENCE` | DM | **清單** | 可見對象/單位**清單**（權限標籤；護理師 / 軍人 / 醫檢師 / 行政人員…；「全體」通用值）|
| `DM_TAG_MODULE` / `DM_TAG_NATURE` / `DM_TAG_LEGAL` | DM | **清單** | 檢索標籤（檢視用，不影響可見性）：適用模組（採血/成分/檢驗/供應/醫務）/ 文件性質（戰時/緊急/平時/訓練）/ 法規關聯（衛福部）|
| `DM_REMIND_THRESHOLD` | DM | 參數 | 自動催辦停留門檻（天，1–30）|
| `DM_FILE_MAX_MB` / `DM_FILE_TYPES` | DM | 參數 | 文件上傳大小上限 / 允許類型 |
| `DM_WEEKLY_SCHED_DAY_TIME` | DM | 參數 | SCHDM001 KPI 週報 / 未讀提醒每週執行時間 |

> **清單型（定義）在 DP，關聯在模組**：如 `ET_TRAINING_UNIT` 只存「標籤名稱清單」；「王小明屬護理師」「課程 A 掛護理師」存 ET（`ET_USER_TAG` / `ET_COURSE_TAG`）。`DM_DOC_CATEGORY` 只存分類清單；「某文件屬 SOP」在文件上（DOC_ID）。套用清單之業務規則（DOC_ID 產碼、`IS_ALL` 展開、自動邀請、唯一手冊檢核）由各模組執行。
> 原 `ET_PARAM.PASSWORD_RESET_TTL_MIN` 改列平台級 `LOGIN`；原 `DM_MAIL_*` 發信調校改列平台級 `MAIL`。

## 決策紀錄（2026-07-06）

| 項目 | 決定 | 說明 |
|------|------|------|
| 模組定位 | 平台共用層，非業務模組 | ET / DM 共用帳號/認證/稽核 |

## 決策紀錄（2026-07-08 — 集中化與統一管理後台）

> **背景**：查證 TBMS 分工後確認——核心業務模組（BS/BC/CP/MA/TL）之參數 / code-list / 範本 / 排程本就集中於 DP（`DP_PARAM` 以 `PARAM_ID` 前綴分模組），僅後期獨立子系統 ET / DM 為避免耦合外部大平台而自持。EDMS 的 DP 是**自家平台**、ET / DM 已共用 DP_USER / DP_AUDIT_LOG / DP 認證，自持理由不成立，故**改為集中**、對齊 TBMS 核心模組。

| 項目 | 決定 | 說明 |
|------|------|------|
| 系統參數 + 清單定義 | **集中 `DP_PARAM_M/D`，前綴分模組** | 廢除 `ET_PARAM` / `DM_PARAM` 與 `ET_TAG`/DM 分類/func_name/可見對象等**定義表**；分類 / 標籤名稱 / func_name / 可見對象值皆為 `DP_PARAM` 清單；鎖定碼用 `DETAIL_LOCK`。密碼重設 TTL、發信調校改列平台級 |
| 通知範本 | **集中 `DP_NOTIFY_TEMPLATE`，`MODULE` 分模組** | 廢除 `ET_NOTIFY_TEMPLATE` / `DM_NOTIFY_TEMPLATE` |
| 發信引擎 | **集中：平台唯一發信服務 + `DP_EMAIL_LOG` outbox** | 廢除 `DM_NOTIFY_QUEUE`；含 DP 系統信（`MODULE=DP`）|
| 排程 | **`DP_SCHEDULE` DB 註冊表 + 引擎 + `DP_SCHEDULE_LOG`** | 由方案 B 升級；job 反向 import 模組 service |
| 統一管理後台 | **所有管理畫面集中於 DP 後台、按模組過濾** | 參數 / 清單 / 範本 / 角色指派畫面都在 DP；各模組只留操作型畫面 |
| 定義 vs 關聯 分層 | **「有哪些」搬 DP_PARAM、「誰配誰」留模組** | 分類/標籤/func_name/角色種類等定義在 DP；M2M / 指派 / 角色判定在模組，引用 DP 碼 |
| 管理者模型 | **只有 ET 管理者 / DM 管理者，拿掉平台管理員** | 模組項按前綴 / MODULE 過濾（互不可見）；帳號管理 / 平台級 `DP_` 參數 / 稽核為共用項，兩者皆可存取 |

> **取捨**：拿掉平台管理員後，帳號管理與平台級參數由兩模組管理者共管——ET 管理者可停用帳號（影響 DM）、可改密碼策略（影響全平台）；稽核兩者皆可查全部。為換取「單一管理後台、無獨立 IT 角色」之簡化，接受此共管範圍。
