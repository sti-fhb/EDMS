# 10 — 教育訓練模組

## 1. 模組概述

教育訓練模組（ET）提供內部人員的線上教育訓練與知識管理功能。由教師建立課程、上傳教材、安排測驗，學員透過線上方式完成課程學習與考核；管理者僅負責權限管理。

### 1.1 角色定義

| 角色 | 說明 |
|------|------|
| 管理者 | 權限管理 — 設定其他帳號於 ET 的角色（包含指派其他管理者、教師、學員） |
| 教師 | 課程安排 — 建立 / 編輯課程；編排章節、教材、測驗；邀請學員；查看學員學習進度與成績 |
| 學員 | 加入課程、觀看教材、進行線上測驗、查看個人學習進度 （首次登入自動授予此權限）|

> **角色可重複**：同一帳號可同時擁有多個角色（例：「管理者 + 教師」兼任、「教師 + 學員」教師也想上別人的課）；權限取聯集。

### 1.2 模組人員對應

ET 額外維護「**使用者 × 業務模組**」對應表（`ET_USER_MODULE`），用途為**課程預設帶入**：

- 管理者於權限管理頁面設定每個帳號隸屬之模組（如「陳大華 → 採血 BC」、「李主任 → 採血 BC + 供應 BS」）
- 教師建立課程時可指定「**關聯模組**」（如 BC 採血），系統自動將該模組所有人員加入本課程，**不需個別邀請**
- 一人可隸屬多個模組（多對多）；對應主系統業務模組代碼（採血 / 成分 / 檢驗 / 供應 / 醫務 / 報表與標籤 / 其他）
- 與 ET 角色（管理者/教師/學員）為**獨立兩個維度**：角色決定能做什麼；模組決定隸屬哪個業務領域

### 1.3 登入機制

ET 與 DM 共用同一個資料庫與 user table，使用者註冊一次即可登入兩系統。登入流程與 DM 完全一致。

#### 1.3.1 共用 user table 欄位

| 欄位 | 格式 | 變動性 | 用途 |
|------|------|--------|------|
| USER_ID | 系統自動產生流水號，系統內唯一 | 永久不變 | 內部使用者識別碼；`ET_USER_ROLE` / `ET_USER_MODULE` / 各 Table 的 audit `CREATED_USER` 等 FK 一律指向此 ID（不存帳號字串） |
| 帳號 | 標準 Email 格式，UNIQUE | 使用者可變更 | 使用者登入用、寄信收件人；換 Email 時更新此欄，USER_ID 不動 |
| 密碼 | hashed（不可逆儲存） | 使用者可變更 | 系統登入用 |
| 姓名 | 中文姓名 | 使用者可變更 | UI 顯示 |

> **USER_ID 為內部穩定識別碼**，帳號（Email）/ 密碼 / 姓名皆可由使用者於個人專區變更；歷史紀錄因 FK 指向 USER_ID 而不受影響。

#### 1.3.2 登入流程

ET 登入頁提供兩個動作：**登入** / **註冊**（與 DM 共用同一張登入頁設計樣式）。

**登入**：

1. 使用者於登入頁輸入帳號（Email）+ 密碼，點「登入」
2. 系統查共用 user table 驗證帳密：
   - **帳號不存在**：顯示錯誤訊息「**查無此帳號，請先註冊**」
   - **帳號存在但密碼錯誤**：顯示錯誤訊息「**密碼錯誤**」
   - **驗證通過**：取得該使用者之 ET 角色清單；若尚無紀錄（已透過 DM 註冊、首次進 ET），自動授予「**學員**」角色一筆
3. 登入完成 → 進入 ET 首頁（依角色不同：管理者 → 權限管理（ET07）；教師 → 課程列表（ET01）；學員 → 我的課程（ET04）；多重角色者由 sidebar 自行選擇進入哪個功能）

**註冊**：

1. 使用者於登入頁點「註冊」，跳出註冊表單請使用者填寫帳號（Email）/ 密碼 / 確認密碼 / 姓名
2. 確認後寫入共用 user table（DM 端同步可登入），**ET 端自動授予「學員」角色**
3. 顯示「註冊成功」提示，**跳回登入頁**請使用者以新建帳號重新登入

#### 1.3.3 忘記密碼

- 登入頁提供「忘記密碼」連結
- 使用者輸入帳號（Email）→ 系統寄出密碼重設連結至該 Email（30 分鐘內有效）
- 重設後即可使用新密碼登入 ET 與 DM

## 2. 功能需求

### 2.1 ET01 課程列表

**進入權限**：教師。

#### 2.1.1 用途

教師於此檢視所有教師建立之課程，提供以下作業：

- **搜尋 / 篩選**：以關鍵字、關聯模組、建立者等條件過濾課程（詳見 §2.1.2）
- **新增課程**：右上「新增課程」按鈕進入 ET02 新增模式
- **編輯課程**：點擊自己建立（`CREATED_USER` = 當前 USER_ID）之課程卡進入 ET02 編輯模式；點擊他人建立之課程進入檢視模式（唯讀）

#### 2.1.2 列表呈現

- 列表依「關聯模組」分組（採血 / 成分 / 檢驗 / 供應 / 醫務 / 報表與標籤 / 其他），每組以課程卡片網格呈現
- 上方 tab 篩選：
  - **我建立的**：僅顯示 `CREATED_USER` = 當前 USER_ID 之課程
  - **全部課程**：顯示所有教師建立之課程
- 卡片資訊：課程名稱、關聯模組 badge、章節數、發布狀態（草稿 / 已發布）、學員人數、**建立者姓名**
- 搜尋條件：關鍵字（課程名稱）、關聯模組；**「全部課程」分頁額外提供「建立者」篩選**（「我建立的」分頁無此欄位，因卡片均為當前使用者）
- 右上「新增課程」按鈕進入 ET02 新增模式

---

### 2.2 ET02 新增/編輯 課程

**進入權限**：教師。

#### 2.2.1 兩種模式

- **新增課程**：由 ET01「新增課程」進入，建立新課程
- **編輯課程**：由 ET01 點擊既有課程卡進入，調整既有課程

進入後可維護課程下之**章節**（chapter）、**教材**（material）、**測驗**（quiz）三類子物件。

> 教師只能編輯自己建立的課程；點擊他人建立之課程僅可檢視，詳見 §2.1.1 用途。

#### 2.2.2 課程基本欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 課程名稱 | Y | 顯示於列表卡片與學員端 |
| 關聯模組 | Y | 單選；對應 §1.2 業務模組代碼（採血 / 成分 / 檢驗 / 供應 / 醫務 / 報表與標籤 / 其他）；不可於編輯模式調整 |
| 描述 | N | 課程說明（純文字，至多 500 字） |

#### 2.2.3 章節編排

- 章節為課程下之**順序容器**，學員須依序解鎖
- 透過拖拉介面（drag-and-drop）調整章節、教材、測驗之先後順序
- 章節下可放置任意數量之教材與測驗
- 刪除章節時 modal 確認；章節本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員於本章節之學習紀錄與測驗成績連同移除（hard delete）；後續章節順序自動往前遞補

#### 2.2.4 教材管理

教材項目由三類媒材自由組合而成（同一項目可同時包含影片、多份 DM 文件、與說明文字）：

| 媒材類別 | 來源 |
|---------|------|
| 影片 | ET 本地上傳 | 
| 文件 | DM 模組引用 | 
| 說明文字 | ET 自有 WYSIWYG 編輯器 | 

教材項目須填寫**標題**（必填）；標題顯示於 ET05 學員端章節導覽列。

**影片（本地上傳）**：

- 單檔大小上限：500 MB（避免影響系統效能）
- 播放進度追蹤：透過 HTML5 video API（`timeupdate` 事件）追蹤實際觀看進度；跳轉播放不計入完成判定

**文件（DM 模組引用）**：

ET 不自行儲存文件檔案；訓練文件**統一由 DM 模組提供**（PDF / PPT / 圖片 / Word 等）。

- 新增方式：教材編輯介面以**下拉選單**從 DM「訓練教材」分類選取既有文件
- 自動帶最新版：DM 端發布新版本時，ET 教材自動取得最新版（FK 永遠對應 DM `CURRENT_VERSION_ID`），無需 ET 手動更新
- 新增文件需至 DM：若 DM「訓練教材」分類無所需文件，請至 DM 模組建立新文件並發布為「訓練教材」分類後，再回 ET 引用
- DM 文件廢止之處理：DM 廢止某文件後，ET 教材於 ET02 顯示「**該文件已廢止**」警告，學員端 ET05 仍可閱讀廢止前最後版本（避免突然斷檔），由教師決定是否移除引用

#### 2.2.5 測驗建立與題庫管理

- 測驗設定欄位：

| 欄位 | 必填 | 說明 |
|------|------|------|
| 測驗名稱 | Y | 顯示於 ET05 章節導覽與 ET06 線上測驗頁面 |
| 及格分數 | Y | 0–100 之整數；預設 80 |
| 作答時間限制 | Y | 分鐘；空白表示不限時 |
| 重考次數上限 | Y | 0–999；0 = 不允許重考；預設 3 |

> **題目順序皆為隨機**：系統內建機制，學員每次作答時題目順序自動洗牌，無需教師設定。

- 題型支援：**單選題**、**多選題**(下拉選單)
- 每題設定：
  - **題幹**：純文字（至多 500 字）
  - **選項**：至少 2 個、至多 6 個；每個選項至多 200 字
  - **配分**：同一測驗各題配分總和須等於 100
- 題目順序可拖拉調整
- 刪除題目時題目本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員於該題之作答紀錄與得分連同移除（hard delete）

#### 2.2.6 三種行為

| 按鈕 | 行為 |
|------|------|
| **取消編輯** | 放棄變更，回到 ET01 課程列表，不寫入任何資料 |
| **儲存草稿** | 建立課程並標示為 `drafting` 狀態，**草稿狀態下不對學員開放**；儲存後跳轉至建立完成提示頁，可繼續新增章節 / 教材 / 測驗 |
| **發布** | 建立課程並直接標示為 `published`；系統檢核「至少 1 個章節、至少 1 項教材」，不通過則阻擋並提示；通過則寫入 `PUBLISHED_AT` 時間戳（即開課日期）並開放學員加入 |

#### 2.2.7 停課

- **觸發按鈕**：ET02 頁面右上「**停課**」按鈕；**僅已發布狀態顯示**（草稿與已停課皆不顯示）
- 點擊跳出 confirm modal，提示「停課後此課程將自動下架、無法再編輯；已加入學員之歷史紀錄保留」，需使用者確認
- 確認後課程狀態由 `published` → **`closed`**（已停課），**不可逆**
- 停課後影響：

| 範圍 | 行為 |
|------|------|
| ET01 課程列表 | 卡片顯示「**已停課**」狀態 pill；點擊只可進入**檢視模式**（章節 / 教材 / 測驗皆唯讀），無法重新編輯或重新發布 |
| ET02 編輯頁 | 所有編輯控制項禁用；右上隱藏「停課」、「邀請學員」按鈕 |
| ET05 課程學習 | 學員嘗試開啟此課程連結時，顯示「**此課程已停止開放**」訊息頁，無法繼續學習 |
| ET03 學員（教師端） | 該課程仍可於課程下拉選單查看已加入學員之歷史完課狀態 / 平均成績，**但無法再邀請新學員** |

#### 2.2.8 邀請學員入口

- 已發布之課程於 ET02 頁面右上顯示「**邀請學員**」按鈕；**草稿與已停課狀態下隱藏**
- 可用 Email 邀請、邀請碼，個別邀請學員
- 預設學員：學員屬於關聯模組的人員

### 2.3 ET03 學員

**進入權限**：教師。

#### 2.3.1 用途

教師於此檢視已發布課程之**學員學習狀況**（完課率、進度、成績），並追蹤**已寄出但未加入**之邀請。

#### 2.3.2 課程選擇與篩選

頁面頂部提供：

- **課程下拉選單**：顯示單一課程的學員
  - 已加入**與**待加入**兩個 tab 同步以該課程過濾
- **關鍵字搜尋**：姓名或 Email
- **匯出 CSV**：依當前篩選條件匯出資料

#### 2.3.3 已加入學員（tab 1）

學員清單表格，欄位如下：

| 欄位 | 說明 |
|------|------|
| 學員 | 姓名 + 帳號（Email） |
| 加入日期 | 學員首次加入該課程之日期 |
| 完課狀態 | **已完成**/ **進行中**/ **未開始**|
| 學習進度 | 進度條 + 百分比 + 當前章節（如「75%（第 4 章）」） |
| 平均成績 | 所有測驗之平均得分；未答測驗者顯示「—」 |
| 最後活動 | 最後一次學習或作答之時間戳 |

教師可對個別學員執行以下操作（於學員列右側）：

- **重置重考次數**：當學員達測驗重考次數上限仍未及格時，恢復該學員之可用次數（稽核記錄修改者與時間）
- **移除學員**：將學員自課程移除，前台清單不再顯示；學習歷史紀錄（章節進度、測驗成績等）完整保留，**移除學員不計入該課程之完課率統計，但歷史成績仍存在於系統**

#### 2.3.4 待加入學員（tab 2）

顯示**已寄出邀請信但尚未加入**之學員清單（限 Email 邀請產生）。

| 欄位 | 說明 |
|------|------|
| 收件 Email | 學員 Email |
| 邀請寄出日 | 系統寄出邀請信之日期 |
| 動作 | 「**再次寄送**」/「**撤回邀請**」 |

### 2.4 ET04 我的課程

**進入權限**：學員。

#### 2.4.1 用途

學員登入後預設首頁，呈現自己已加入之所有課程與學習狀態，作為 ET05 課程學習之入口；亦提供「加入新課程」入口（邀請碼輸入）。

#### 2.4.2 課程清單

- 頁面上方依學習狀態分區呈現總數：**進行中課程**、**未開始課程**、**已完成課程**
- 下方依關聯模組分組呈現各課程卡片，卡片含**課程名稱、當前進度、章節**等資訊
- 點擊卡片進入該課程之 ET05 學習頁面

#### 2.4.3 加入新課程

- **邀請碼輸入**：於本頁「+ 加入新課程」modal 輸入 8 碼邀請碼 → 系統驗證 → 顯示課程資訊 → 「確認加入」即生效

例外處理：

- 邀請連結 / 邀請碼**無效**（不存在或已重置）：顯示「邀請碼無效，請聯絡管理者」
- 學員**已加入**該課程：直接跳至 ET05 課程學習頁，不重複加入
- 課程已**停課**：顯示「此課程已停止開放」

### 2.5 ET05 課程學習

**進入權限**：學員。

#### 2.5.1 用途

學員觀看課程內容、追蹤章節進度。

#### 2.5.2 畫面結構

- 左側：**章節導覽列**，列出所有章節與其下之教材 / 測驗
- 中間：**內容區**，依當前選取項目顯示影片播放器 / DM 文件 / 說明文字
  - **影片**：HTML5 video player；含播放 / 暫停、調速、音量、全螢幕控制
  - **DM 文件**：PDF 於頁內嵌入預覽；非 PDF 格式顯示「**下載原檔以本機應用程式開啟**」連結
  - **說明文字**：直接渲染為 HTML 顯示
  - 教材內容跨章節保持上次觀看位置，學員返回時自動恢復

#### 2.5.3 章節解鎖邏輯

依系統內建之進度條件規則判定：

- 章節「**強制觀看完成**」 → 上一章節所有影片皆須播放超過 80%
- 章節「**測驗通過**」 → 上一章節之測驗成績達及格分數
- 內容可以重複觀看

#### 2.5.4 章節內測驗入口

- 章節中之測驗項目以「**📝 測驗：xxx**」呈現；前置條件滿足後解鎖
- 點擊測驗 → 進入 ET06 線上測驗（詳見 §2.6）
- 測驗作答完成返回 ET05 後，章節導覽列即時更新該測驗之完成 / 成績狀態

### 2.6 ET06 線上測驗

**進入權限**：學員。

#### 2.6.1 用途

學員作答課程章節內之線上測驗，系統自動閱卷並回傳成績；提供複習檢討與重考機制。測驗之建立與題庫管理屬 ET02，詳見 §2.2.5。

#### 2.6.2 測驗開始引導

- 學員自 ET05 點擊章節內測驗項目時，先進入引導頁
- 顯示資訊：測驗名稱、題目數量、及格分數、作答時間限制、剩餘可重考次數、上次成績（若有）
- 「**開始作答**」按鈕 → 進入答題介面；按下後系統開始計時（若有限時）
- 「**取消**」回到 ET05 課程學習頁

#### 2.6.3 答題介面

- 上方狀態列：題號進度（如 3 / 10）、倒數計時（若有限時）、「**提交**」按鈕
- 左側題目導覽列：列出所有題號，未答 / 已答以不同顏色標示，可任意跳題
- 答題操作：單選用 radio、多選用 checkbox；切換題目自動暫存當前作答
- **時間到自動提交**：若啟用時間限制，倒數歸零時系統自動以當前作答狀態提交
- **離開頁面警示**：作答中切換頁面或關閉視窗時，瀏覽器提示確認

#### 2.6.4 自動閱卷與成績顯示

- 學員按「提交」或時間到自動提交後，系統**即時閱卷**並回傳結果
- 顯示資訊：
  - 總分（0–100）
  - **是否及格**
  - **答題明細**：題號、題型、自己選的答案、正確答案、結果、得分；點擊該題答題明細，可看到題目更多資訊，讓學員複習內容
- **多選題判定（部分計分）**：每答對一個正確選項加分、每選錯一個錯誤選項扣分，最低為 0 分。
  - 計算公式：`得分 = max(0, (答對正確選項數 − 誤選數) ÷ 應選正確數 × 該題配分)`

#### 2.6.5 重考機制

- 學員未達及格分數時，系統顯示「**重新作答**」按鈕（前提：剩餘重考次數 > 0）
- 點擊後返回引導頁，剩餘次數扣 1，重新計時、重新發題（題目順序重新洗牌）
- **成績紀錄**：每次重考之成績皆獨立記錄；於各處顯示成績時以**最高分**為準（如 ET03 學員清單之「平均成績」欄、學員端結業成績）
- 達及格分數後不再允許作答（避免重考刷高分）；唯讀模式查看成績明細
- 達重考次數上限仍未及格者：不再允許作答；教師可於 ET03 學員手動「**重置重考次數**」（詳見 §2.3.3）

### 2.7 ET07 權限管理

**進入權限**：僅管理者。

#### 2.7.1 用途

管理者於此設定每個帳號在 ET 中之**角色**與**隸屬之業務模組**（詳見 §1.2 模組人員對應）。

#### 2.7.2 使用者清單

- 列出**所有共用 user table 中之使用者**（ET 與 DM 共用，故包含 DM 註冊但未進過 ET 之使用者）
- 顯示欄位：姓名、帳號（Email）、角色(管理者 / 教師 / 學員)、業務模組（採血 / 成分 / 檢驗 / 供應 / 醫務 / 報表與標籤 / 其他）、最後修改（時間 + 修改者）
- 搜尋與篩選：角色篩選、模組篩選、關鍵字搜尋（姓名 / 帳號）

#### 2.7.3 角色多重指派

- 三種角色（管理者 / 教師 / 學員）以**核取方塊**呈現，可任意組合（聯集生效）
- 角色職責定義詳見 §1.1

#### 2.7.4 模組對應設定

- 點欄內「**編輯**」按鈕開啟 modal，以核取方塊勾選六模組可全不勾（即「未指派」）
- 模組對應與角色為**獨立兩個維度**：角色決定能做什麼、模組決定隸屬哪個業務領域
- 用途為「課程預設帶入」

#### 2.7.5 預設行為（新使用者首次登入）

- 新使用者於 ET 首次登入時：
  - **角色**：自動授予「**學員**」一筆（寫入 `ET_USER_ROLE`）
  - **模組對應**：預設「**未指派**」，由管理者後續指派

#### 2.7.6 稽核紀錄

- 每筆角色 / 模組變更記錄修改者與時間於「最後修改」欄
- 完整變更歷史寫入系統稽核 log（含舊值 / 新值 / 修改者 / 修改時間 / IP），不於 UI 顯示，可由 DBA 查詢

### 2.8 ET08 個人資料維護

**進入權限**：所有角色（管理者 / 教師 / 學員）。

#### 2.8.1 用途

使用者編輯自己的帳號 / 姓名 / 密碼。變更同步寫入共用 user table，ET 與 DM 兩端同步生效。

#### 2.8.2 欄位編輯行為

| 欄位 | 編輯行為 |
|------|---------|
| 帳號（Email）| 變更時系統寄驗證信至**新 Email**，點連結後新帳號才生效（避免誤輸入錯誤 Email 把帳號鎖死） |
| 姓名 | 直接編輯儲存 |
| 密碼 | 點「變更密碼」按鈕跳 modal，須填**舊密碼 + 新密碼 + 確認新密碼**（驗證舊密碼避免他人冒用 session 改密碼）|

> 忘記密碼請改走登入頁「**忘記密碼**」連結（詳見 §1.3.3）；本頁僅供已登入使用者主動變更。
