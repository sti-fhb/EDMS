# 10 — 教育訓練模組

## 1. 模組概述

教育訓練模組（ET）提供內部人員的線上教育訓練與知識管理功能。由教師建立課程、上傳教材、安排測驗與課後問卷，學員透過線上方式完成課程學習與考核；管理者負責權限管理、受訓單位標籤維護與通知範本維護。

### 1.1 角色定義

| 角色 | 說明 |
|------|------|
| 管理者 | 權限管理 — 設定其他帳號於 ET 的角色（包含指派其他管理者、教師、學員）；維護**受訓單位標籤庫**與**使用者×標籤**對應；維護**通知信範本**與排程參數 |
| 教師 | 課程安排 — 建立 / 編輯課程；編排章節、教材、測驗、課後問卷；設定課程起訖時間；邀請學員；查看學員學習進度、成績、**每次作答明細**與**問卷結果** |
| 學員 | 加入課程、觀看教材、進行線上測驗、填寫課後問卷、查看個人學習進度 （首次登入自動授予此權限）|
| 系統自動 | 發布時依標籤自動邀請＋寄信、到期自動關閉課程、每週統計與提醒排程、截止前加急提醒 |

> **角色可重複**：同一帳號可同時擁有多個角色（例：「管理者 + 教師」兼任、「教師 + 學員」教師也想上別人的課）；權限取聯集。

### 1.2 受訓單位標籤（原「模組人員對應」，2026-07-02 變更）

原「使用者 × 業務模組」對應（`ET_USER_MODULE`）與課程「關聯模組」**移除**，改為**受訓單位標籤**機制：

- **標籤庫（`ET_TAG`）**：由管理者於 ET07 維護，可**新增 / 修改 / 停用 / 啟用**；系統內建種子 5 筆：**全體 / 護理師 / 行政人員 / 軍人 / 醫檢師**
  - 「**全體**」為特殊標籤（`IS_ALL`），代表**所有具「學員」角色之使用者**，不需逐人貼標
  - **停用**之標籤：不可再掛至新課程；已掛之既有課程不受影響
- **使用者 × 標籤（`ET_USER_TAG`）**：多對多；由管理者於 ET07 指派（一人可屬多個受訓單位）
- **課程 × 標籤（`ET_COURSE_TAG`）**：多對多；一課程可掛多個標籤；**發布前至少須掛 1 個標籤**
- **自動邀請**：課程發布時，系統取課程所掛標籤對應人員之**聯集（去重）**，自動加入課程並各寄一封通知信（詳見 §2.2.8）
- **事後貼標追溯**：管理者事後將某使用者貼上標籤時，系統自動將其**補加入**該標籤對應之所有「已發布且未關閉」課程，並寄**彙整一封**通知信（列出所有新加入課程）
- **已發布課程之標籤調整**：已發布課程可**新增**標籤（觸發該標籤人員之補邀請＋寄信）、**不可移除**既有標籤（已加入學員不受影響）；草稿狀態可自由增刪

> **不與 DM 共用標籤**：DM 亦有「可見對象/單位」標籤（`DM_TAG` AUDIENCE 組），名目雷同，但**兩系統使用者群相交而不相同**，且後續可能各自新增對方沒有的單位，故 ET 自持 `ET_TAG` / `ET_USER_TAG`，與 DM 各自維護。

### 1.3 登入機制

ET 與 DM 共用同一個資料庫與 `DP_USER`，使用者註冊一次即可登入兩系統。登入流程與 DM 完全一致。

#### 1.3.1 共用 `DP_USER` 欄位（由平台模組 DP 定義，詳見 [09-平台模組](09-平台模組.md)）

| 欄位 | 格式 | 變動性 | 用途 |
|------|------|--------|------|
| USER_ID | 系統自動產生流水號，系統內唯一 | 永久不變 | 內部使用者識別碼；`ET_USER_ROLE` / `ET_USER_TAG` / 各 Table 的 audit `CREATED_USER` 等 FK 一律指向此 ID（不存帳號字串） |
| 帳號 | 標準 Email 格式，UNIQUE | 使用者可變更 | 使用者登入用、寄信收件人；換 Email 時更新此欄，USER_ID 不動 |
| 密碼 | hashed（不可逆儲存） | 使用者可變更 | 系統登入用 |
| 姓名 | 中文姓名 | 使用者可變更 | UI 顯示 |

> **USER_ID 為內部穩定識別碼**，帳號（Email）/ 密碼 / 姓名皆可由使用者於個人專區變更；歷史紀錄因 FK 指向 USER_ID 而不受影響。

#### 1.3.2 登入流程

ET 登入頁提供兩個動作：**登入** / **註冊**（與 DM 共用同一張登入頁設計樣式）。

**登入**：

1. 使用者於登入頁輸入帳號（Email）+ 密碼，點「登入」
2. 系統查共用 `DP_USER` 驗證帳密：
   - **帳號不存在**：顯示錯誤訊息「**查無此帳號，請先註冊**」
   - **帳號存在但密碼錯誤**：顯示錯誤訊息「**密碼錯誤**」
   - **驗證通過**：取得該使用者之 ET 角色清單；若尚無紀錄（已透過 DM 註冊、首次進 ET），自動授予「**學員**」角色一筆
3. 登入完成 → 進入 ET 首頁（依角色不同：管理者 → 權限管理（ET07）；教師 → 課程列表（ET01）；學員 → 我的課程（ET04）；多重角色者由 sidebar 自行選擇進入哪個功能）

**註冊**：

1. 使用者於登入頁點「註冊」，跳出註冊表單請使用者填寫帳號（Email）/ 密碼 / 確認密碼 / 姓名
2. 確認後寫入共用 `DP_USER`（DM 端同步可登入），**ET 端自動授予「學員」角色**
3. 顯示「註冊成功」提示，**跳回登入頁**請使用者以新建帳號重新登入

#### 1.3.3 忘記密碼

- 登入頁提供「忘記密碼」連結
- 使用者輸入帳號（Email）→ 系統寄出密碼重設連結至該 Email（30 分鐘內有效）
- 重設後即可使用新密碼登入 ET 與 DM

## 2. 功能需求

### 2.1 ET01 課程列表

**進入權限**：教師。

#### 2.1.1 用途

教師於此檢視所有教師建立之課程，提供以下作業：

- **搜尋 / 篩選**：以關鍵字、受訓單位標籤、建立者等條件過濾課程（詳見 §2.1.2）
- **新增課程**：右上「新增課程」按鈕進入 ET02 新增模式
- **編輯課程**：點擊自己建立（`CREATED_USER` = 當前 USER_ID）之課程卡進入 ET02 編輯模式；點擊他人建立之課程進入檢視模式（唯讀）

#### 2.1.2 列表呈現

- 課程以卡片網格呈現（**不再依模組分組**；2026-07-02 變更）
- 上方 tab 篩選：
  - **我建立的**：顯示 `CREATED_USER` = 當前 USER_ID 之課程，**含草稿 / 已發布 / 已關閉全部狀態**（供本人管理）
  - **全部課程**：顯示所有教師建立之課程，但**僅限「已發布」狀態**（草稿與已關閉不列入；2026-07-02 變更）
- 卡片資訊：課程名稱、**受訓單位標籤 badges（可多個）**、章節數、課程狀態（草稿 / 已發布 / **已關閉**）、**課程起訖時間**、學員人數、**建立者姓名**
- 搜尋條件：關鍵字（課程名稱）、**受訓單位標籤**；**「全部課程」分頁額外提供「建立者」篩選**（「我建立的」分頁無此欄位，因卡片均為當前使用者）
- 右上「新增課程」按鈕進入 ET02 新增模式

### 2.2 ET02 新增/編輯 課程

**進入權限**：教師。

#### 2.2.1 兩種模式

- **新增課程**：由 ET01「新增課程」進入，建立新課程
- **編輯課程**：由 ET01 點擊既有課程卡進入，調整既有課程

進入後可維護課程下之**章節**（chapter）、**教材**（material）、**測驗**（quiz）、**課後問卷**（survey）四類子物件。

> 教師只能編輯自己建立的課程；點擊他人建立之課程僅可檢視，詳見 §2.1.1 用途。

#### 2.2.2 課程基本欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 課程名稱 | Y | 顯示於列表卡片與學員端 |
| 受訓單位標籤 | Y（發布時） | **多選**；自 `ET_TAG` 啟用中標籤選取（全體 / 護理師 / 行政人員 / 軍人 / 醫檢師…）；發布前至少 1 個；已發布後**可新增、不可移除**（新增觸發補邀請） |
| 課程起訖時間 | Y（發布時） | **閱課期間**（起、訖兩個時間）；起始前學員看不到課程；到訖止時間系統**自動關閉**閱課；草稿可留空，發布時必填 |
| 描述 | N | 課程說明（純文字，至多 500 字） |

#### 2.2.3 章節編排

- 章節為課程下之**順序容器**，學員須依序解鎖
- 透過拖拉介面（drag-and-drop）調整章節、教材、測驗之先後順序
- 章節下可放置任意數量之教材與測驗
- 刪除章節時 modal 確認；章節本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員於本章節之學習紀錄與測驗成績連同移除（hard delete）；後續章節順序自動往前遞補

#### 2.2.4 教材管理

教材項目由三類媒材自由組合而成（同一項目可同時包含影片、多份 DM 文件、與說明文字）：

| 媒材類別 | 來源 |
|---------|------|
| 影片 | ET 本地上傳 | 
| 文件 | DM 模組引用 | 
| 說明文字 | ET 自有 WYSIWYG 編輯器 | 

教材項目須填寫**標題**（必填）；標題顯示於 ET05 學員端章節導覽列。

**影片（本地上傳）**：

- 單檔大小上限：500 MB（避免影響系統效能）
- 播放進度追蹤：透過 HTML5 video API（`timeupdate` 事件）追蹤**實際播放區段**；觀看判定採**累計覆蓋率**（詳見 §2.5.3）；跳轉播放允許，但**跳過未播放之區段不計入**完成判定

**文件（DM 模組引用）**：

ET 不自行儲存文件檔案；訓練文件**統一由 DM 模組提供**（PDF / PPT / 圖片 / Word 等）。

- 新增方式：教材編輯介面以**下拉選單**從 DM「訓練教材」分類選取既有文件
- 自動帶最新版：DM 端發布新版本時，ET 教材自動取得最新版（FK 永遠對應 DM `CURRENT_VERSION_ID`），無需 ET 手動更新
- 新增文件需至 DM：若 DM「訓練教材」分類無所需文件，請至 DM 模組建立新文件並發布為「訓練教材」分類後，再回 ET 引用
- DM 文件廢止之處理：DM 廢止某文件後，ET 教材於 ET02 顯示「**該文件已廢止**」警告，學員端 ET05 仍可閱讀廢止前最後版本（避免突然斷檔），由教師決定是否移除引用

#### 2.2.5 測驗建立與題庫管理

- 測驗設定欄位：

| 欄位 | 必填 | 說明 |
|------|------|------|
| 測驗名稱 | Y | 顯示於 ET05 章節導覽與 ET06 線上測驗頁面 |
| 及格分數 | Y | 0–100 之整數；預設 80 |
| 作答時間限制 | N | 分鐘；**空白 = 不限時**、**≥ 1 = 限時分鐘**；停用測驗請刪除該測驗項目（不以時間限制表達） |
| 重考次數上限 | Y | 0–999；0 = 不允許重考；預設 3 |

> **題目順序皆為隨機**：系統內建機制，學員每次作答時題目順序自動洗牌，無需教師設定。

- 題型支援：**單選題**、**多選題**(下拉選單)
- 每題設定：
  - **題幹**：純文字（至多 500 字）
  - **選項**：至少 2 個、至多 6 個；每個選項至多 200 字
  - **配分**：同一測驗各題配分總和須等於 100
- 題目順序可拖拉調整
- 刪除題目時題目本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員於該題之作答紀錄與得分連同移除（hard delete）

#### 2.2.6 課後回饋問卷（2026-07-02 新增）

教師可為課程建立**課後滿意度 / 回饋問卷**：

| 規則項目 | 規則 |
|---------|------|
| 數量 | 一門課程 **0～1 份**（選配）；掛課程層級，不掛章節 |
| 題型 | 每題**單選**（如：滿意 / 普通 / 不滿意）；選項由教師自行新增 / 編輯 / 刪除 |
| 填寫時機 | 學員**完課後**開放填寫；課程**關閉後不可填**（與「不能再作答」一致） |
| 與完課關係 | 填寫問卷**不是**完課條件；不計入學習進度 % |
| 填寫次數 | 一人**一次**，送出後不可修改；可查看自己填過的內容 |
| 具名 | 填答**具名**；教師可看到每位學員之選項 |
| 題目凍結 | 問卷**已有任何學員填答後，題目與選項不可再修改**（僅可停用問卷）；無人填答前可自由編修 |
| 教師檢視 | 於 ET03「問卷結果」：各題**選項分布統計** ＋ **逐學員填答明細（具名）** |

#### 2.2.7 三種行為

| 按鈕 | 行為 |
|------|------|
| **取消編輯** | 放棄變更，回到 ET01 課程列表，不寫入任何資料 |
| **儲存草稿** | 建立課程並標示為 `drafting` 狀態，**草稿狀態下不對學員開放**；儲存後跳轉至建立完成提示頁，可繼續新增章節 / 教材 / 測驗 / 問卷 |
| **發布** | 系統檢核「至少 1 個章節、至少 1 項教材」＋「**至少 1 個受訓單位標籤**」＋「**課程起訖時間已填**」，不通過則阻擋並提示；通過則標示為 `published`、寫入 `PUBLISHED_AT` 時間戳，並**自動邀請標籤對應人員＋寄發通知信**（詳見 §2.2.9） |

> 發布後、**起始時間前**：課程對學員**不可見**（我的課程清單不顯示、無法進入）；到起始時間才開放學習。

#### 2.2.8 關閉與再開課（原「停課」，2026-07-02 變更：改為可逆）

課程關閉有**兩種觸發**，效果相同：

| 觸發 | 說明 |
|------|------|
| **到期自動關閉** | 課程到達**訖止時間**，系統自動將狀態 `published` → `closed` |
| **手動關閉** | ET02 頁面右上「**關閉課程**」按鈕（僅已發布狀態顯示）；點擊跳 confirm modal 確認 |

**關閉後影響（唯讀模式）**：

| 範圍 | 行為 |
|------|------|
| ET01 課程列表 | 卡片顯示「**已關閉**」狀態 pill；教師可進入編輯，並可執行「**再開課**」 |
| ET02 編輯頁 | **教師（owner）仍可編輯課程內容**（非唯讀，供準備下次開課）；右上顯示「**再開課**」按鈕（隱藏「關閉課程」；邀請碼關閉期間失效，補件邀請待再開課）|
| ET04 / ET05 學員端 | 課程**仍可見、可唯讀回看**已學過的內容；**不能**再累積進度、不能作答測驗、不能解鎖新章節、不能填寫問卷 |
| ET03 學員（教師端） | 仍可查看學員歷史完課狀態 / 成績 / 明細；不可再邀請新學員、不可重置重考次數 |
| 邀請碼 | 關閉期間**失效**（無法以邀請碼加入）；再開課後恢復有效 |
| 排程 | 關閉期間**停止**每週統計與提醒（§2.9）；再開課後恢復 |

**關閉當下若有學員正在作答測驗**：允許該次作答**完成並計分**；但立即禁止開始新的作答。

**再開課**：

- 教師於已關閉課程執行「**再開課**」，須**重新設定一組新的起訖時間**
- 確認後狀態 `closed` → `published`，學員可繼續學習（進度接續保留）
- 關閉 / 再開課可重複多次

#### 2.2.9 學員邀請（2026-07-02 變更：以標籤自動邀請為主）

| 方式 | 觸發 | 說明 |
|------|------|------|
| **標籤自動邀請**（主要） | 課程**發布時**系統自動執行 | 取課程所掛標籤對應人員（限具「學員」角色者）之**聯集、去重**，自動加入課程（寫入選課關聯，來源標記「標籤帶入」），**同時每人寄一封通知信**（統一範本）；「全體」標籤 = 全部具學員角色者 |
| **事後貼標追溯** | 管理者於 ET07 將使用者貼上標籤時 | 自動補加入該標籤所有「已發布且未關閉」課程；寄**彙整一封**通知信（列出所有新加入課程） |
| **Email 邀請**（補件用） | 教師於 ET02 手動輸入 Email 清單 | 供臨時追加不在標籤內的人；信件採**統一範本**（教師不可編輯主旨內文）；寄送失敗保留「待加入」狀態可再寄 |
| **邀請碼** | 課程發布時系統自動產生 | 8 碼，學員於 ET04 自行輸入加入；關閉期間失效 |

> 通知信範本由**管理者**統一維護（§2.10），寄出內容一致；不提供教師逐課編輯。
> 通知對象一律為**學員個人** Email；不另設「單位信箱」收件（受訓單位概念已由標籤吸收）。

### 2.5 ET05 課程學習

**進入權限**：學員。

#### 2.5.1 用途

學員觀看課程內容、追蹤章節進度；完課後填寫課後問卷。

#### 2.5.2 畫面結構

- 左側：**章節導覽列**，列出所有章節與其下之教材 / 測驗；完課後（若課程有問卷）顯示「**填寫課後問卷**」入口
- 中間：**內容區**，依當前選取項目顯示影片播放器 / DM 文件 / 說明文字
  - **影片**：HTML5 video player；含播放 / 暫停、**調速（倍速上限 2x：0.75 / 1 / 1.25 / 1.5 / 2）**、音量、全螢幕控制
  - **DM 文件**：PDF 於頁內嵌入預覽；非 PDF 格式顯示「**下載原檔以本機應用程式開啟**」連結
  - **說明文字**：直接渲染為 HTML 顯示
  - 教材內容跨章節保持上次觀看位置，學員返回時自動恢復
- 課程已關閉時：內容**唯讀回看**（可重看已看過的教材），不再累積進度、不可作答、不可填問卷

#### 2.5.3 章節解鎖邏輯

依系統內建之進度條件規則判定：

- 章節「**強制觀看完成**」 → 上一章節所有影片皆須**觀看超過 80%**
- 章節「**測驗通過**」 → 上一章節之測驗成績達及格分數
- 內容可以重複觀看

**影片「觀看超過 80%」判定規則（2026-07-02 明文化）**：

- 採**累計覆蓋率**：學員**實際播放過之區段聯集** ÷ 影片總長 ≥ 80% 即達成
- **可拉進度條**（允許跳轉），但**只有實際播放過的區段計入**；**直接拉到結尾不產生觀看區段、不算看過**
- **倍速播放照算**：以 2 倍速實際看完全片 = 覆蓋率 100%（倍速上限 2x）
- 同一區段**重複觀看不加成**（區段聯集去重）
- 不採「結束時間 − 開始時間」之牆鐘時間計算（該算法會誤傷倍速觀看者、且掛機暫停可作弊）

#### 2.5.4 章節內測驗入口

- 章節中之測驗項目以「**📝 測驗：xxx**」呈現；前置條件滿足後解鎖
- 點擊測驗 → 進入 ET06 線上測驗（詳見 §2.6）
- 測驗作答完成返回 ET05 後，章節導覽列即時更新該測驗之完成 / 成績狀態

#### 2.5.5 課後問卷填寫（2026-07-02 新增）

- 學員**完課後**，章節導覽列顯示「填寫課後問卷」入口（課程無問卷時不顯示）
- 逐題單選作答，送出後不可修改；可回看自己填過的內容
- 課程關閉後不可填寫

### 2.6 ET06 線上測驗

**進入權限**：學員。

#### 2.6.1 用途

學員作答課程章節內之線上測驗，系統自動閱卷並回傳成績；提供複習檢討與重考機制。測驗之建立與題庫管理屬 ET02，詳見 §2.2.5。

#### 2.6.2 測驗開始引導

- 學員自 ET05 點擊章節內測驗項目時，先進入引導頁
- 顯示資訊：測驗名稱、題目數量、及格分數、作答時間限制、剩餘可重考次數、上次成績（若有）
- 「**開始作答**」按鈕 → 進入答題介面；按下後系統開始計時（若有限時）
- 「**取消**」回到 ET05 課程學習頁

#### 2.6.3 答題介面

- 上方狀態列：題號進度（如 3 / 10）、倒數計時（若有限時）、「**提交**」按鈕
- 左側題目導覽列：列出所有題號，未答 / 已答以不同顏色標示，可任意跳題
- 答題操作：單選用 radio、多選用 checkbox；切換題目自動暫存當前作答
- **時間到自動提交**：若啟用時間限制，倒數歸零時系統自動以當前作答狀態提交
- **離開頁面警示**：作答中切換頁面或關閉視窗時，瀏覽器提示確認

#### 2.6.4 自動閱卷與成績顯示

- 學員按「提交」或時間到自動提交後，系統**即時閱卷**並回傳結果
- 顯示資訊：
  - 總分（0–100）
  - **是否及格**
  - **答題明細**：題號、題型、自己選的答案、正確答案、結果、得分；點擊該題答題明細，可看到題目更多資訊，讓學員複習內容
- **歷次成績與明細**（2026-07-02 明確化）：學員可回看自己**每一次**作答之成績與逐題明細（不限最近一次）；教師端亦可於 ET03 查看（§2.3.3）
- **多選題判定（部分計分）**：每答對一個正確選項加分、每選錯一個錯誤選項扣分，最低為 0 分。
  - 計算公式：`得分 = max(0, (答對正確選項數 − 誤選數) ÷ 應選正確數 × 該題配分)`

#### 2.6.5 重考機制

- 學員未達及格分數時，系統顯示「**重新作答**」按鈕（前提：剩餘重考次數 > 0）
- 點擊後返回引導頁，剩餘次數扣 1，重新計時、重新發題（題目順序重新洗牌）
- **成績紀錄**：每次重考之成績皆獨立記錄；於各處顯示成績時以**最高分**為準（如 ET03 學員清單之「平均成績」欄、學員端結業成績）
- 達及格分數後不再允許作答（避免重考刷高分）；唯讀模式查看成績明細
- 達重考次數上限仍未及格者：不再允許作答；教師可於 ET03 學員手動「**重置重考次數**」（詳見 §2.3.3）

### 2.7 ET07 權限管理（系統設定 -「權限管理」/「參數設定」分頁）

> 2026-07-02：管理者「**系統設定**」畫面分 3 分頁（比照 DM09）：**參數設定**（受訓單位標籤庫維護）、**權限管理**（角色與標籤指派）、**通知範本**（§2.10）。ET07 涵蓋「參數設定」與「權限管理」兩分頁；功能作業代碼 ET07 / ET09 沿用。

**進入權限**：僅管理者。

#### 2.7.1 用途

管理者於此設定每個帳號在 ET 中之**角色**與**受訓單位標籤**，並維護**標籤庫**（詳見 §1.2）。

#### 2.7.2 使用者清單

- 列出**所有共用 `DP_USER` 中之使用者**（ET 與 DM 共用，故包含 DM 註冊但未進過 ET 之使用者）
- 顯示欄位：姓名、帳號（Email）、角色(管理者 / 教師 / 學員)、**受訓單位標籤**（護理師 / 行政人員 / 軍人 / 醫檢師…）、最後修改（時間 + 修改者）
- 搜尋與篩選：角色篩選、**標籤篩選**、關鍵字搜尋（姓名 / 帳號）

#### 2.7.3 角色多重指派

- 三種角色（管理者 / 教師 / 學員）以**核取方塊**呈現，可任意組合（聯集生效）
- 角色職責定義詳見 §1.1

#### 2.7.4 受訓單位標籤指派（原「模組對應設定」，2026-07-02 變更）

- 點欄內「**編輯**」按鈕開啟 modal，以核取方塊勾選標籤（可複選、可全不勾即「未指派」）
- 標籤與角色為**獨立兩個維度**：角色決定能做什麼、標籤決定隸屬哪個受訓單位
- 用途為「課程標籤自動邀請」（§2.2.9）
- **貼標追溯**：新增某人 × 標籤時，系統自動將其補加入該標籤所有「已發布且未關閉」課程並寄彙整通知信；**移除**某人 × 標籤時，既有課程之學員名單**不變動**（已加入者可繼續學習），之後新發布之該標籤課程不再自動邀請

#### 2.7.5 標籤庫維護（2026-07-02 新增）

- 管理者可**新增 / 修改 / 停用 / 啟用**受訓單位標籤（參照 DM「可見對象/單位」之維護方式，但 ET 自持標籤庫）
- 內建種子：**全體 / 護理師 / 行政人員 / 軍人 / 醫檢師**；「全體」為系統內建特殊標籤，不可停用、不可刪除
- 停用之標籤不可掛新課程；既有課程不受影響

#### 2.7.6 預設行為（新使用者首次登入）

- 新使用者於 ET 首次登入時：
  - **角色**：自動授予「**學員**」一筆（寫入 `ET_USER_ROLE`）
  - **標籤**：預設「**未指派**」，由管理者後續指派

#### 2.7.7 稽核紀錄

- 每筆角色 / 標籤變更記錄修改者與時間於「最後修改」欄
- 完整變更歷史寫入系統稽核 log（含舊值 / 新值 / 修改者 / 修改時間 / IP），不於 UI 顯示，可由 DBA 查詢

### 2.9 排程統計與提醒（2026-07-02 新增）

系統以兩支排程自動統計看課狀況並寄發提醒；僅針對**開放中**（已發布且在起訖期間內）之課程執行，已關閉 / 未到起始之課程不納入。

#### 2.9.1 每週統計與週報（SCHET001，每週一 10:00，時間可由系統參數調整）

**統計快照**：每次執行將各開放中課程之統計結果留存（`ET_WEEKLY_STAT`，課程 × 週次），供週報之「與上週比較」與歷史回查。

**週報寄送**：

| 收件者 | 範圍 |
|--------|------|
| 教師 | 自己建立之開放中課程 |
| 管理者 | 全域所有開放中課程 |

週報內容（email 內文摘要 + CSV 附件）：每門課程之——

- **平均看課進度 %**（全班學員進度平均；與上週比較之增減）
- **人數分布**：未開始 / 進行中 / 已完課
- **完課率**（已完課 ÷ 已加入，不含已移除）
- **距訖止天數**
- **未開始名單**（進度 0% 者）
- CSV 明細：逐學員之姓名、Email、進度 %、完課狀態、最後活動時間

> 學員「看課進度 %」定義：**已完成章節項目數 ÷ 總章節項目數**（影片依覆蓋率 ≥ 80% 判完成、測驗依及格判完成、文件 / 說明文字依開啟判完成）。

**每週未看提醒**（與週報同批執行）：

- 對象：各開放中課程中**進度 0%（完全未開始）**之學員
- 一人**一信彙整**（同一學員多門課未開始時，一封信列出所有課程）
- 已開始（進度 > 0%）之學員不寄週提醒（避免長期課程之提醒疲勞）

#### 2.9.2 截止前加急提醒（SCHET002，每日執行）

- 每日檢查各開放中課程是否進入「**訖止時間前 3 天**」（天數可由系統參數調整）
- 進入者：對該課程**所有未完課**學員寄送加急提醒信（標明課程名稱與截止時間）
- 每門課程之加急提醒**只寄一次**（不每日重複）

### 2.10 通知信範本維護（系統設定 -「通知範本」分頁，管理者，2026-07-02 新增）

管理者維護課程 / 學習相關通知信之範本（主旨 + 內文，支援變數如課程名稱、起訖時間、學習連結），並可**啟用 / 停用**各範本（比照 DM；**停用則該類信件不寄送**，觸發事件照常運作）；教師不可逐課修改，寄出內容統一。可維護範本共 **6 類**：

| 範本 | 用途 | 觸發 |
|------|------|------|
