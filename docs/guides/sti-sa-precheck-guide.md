# `/sti-sa-precheck` 使用指南

## 使用時機

當你**已經寫好某個 US 的規格**（`docs/specs/` 下的 spec、data-model、contracts，以及 `docs/wireframes/` 的畫面），要**把對應 issue 交給 SD 開發之前**使用。

指令會自動幫你檢查規格是否齊備、四份文件彼此是否一致，並輸出一份缺口報告。

> **為什麼要做**：規格沒對齊，SD 開發到一半就得回頭問你。先自檢一次，把該補的補齊，開發才不會卡住。

指令會逐項檢查 **5 個面向**：

| 檢查 | 看什麼 |
|------|--------|
| A-1 spec 完整性 | FR/AC 有沒有「待補 / TBD」、訊息表完不完整、有沒有只寫成功流程 |
| A-2 data-model | 欄位的型別 / 可空 / 預設值齊不齊、業務規則寫了沒 |
| A-3 contracts | 跨模組契約定義完不完整（不跨模組會自動標 N/A） |
| A-4 wireframe | 對應畫面區段在不在、異常 / 空資料狀態畫了沒 |
| **A-5 一致性交叉** | **spec、data-model、wireframe、contracts 四者對不對得上**（最容易漏，也最會擋住 SD）|

> 只需要按幾次數字確認，不需要懂指令。

---

## 使用流程示範

以下是一個完整範例（SA 檢查供應模組 BS 的 US4 規格是否可交付）。

---

**在 Claude Code 輸入：**

```
/sti-sa-precheck bs us4
```

> 也可以只給模組（`/sti-sa-precheck bs`）或什麼都不給（`/sti-sa-precheck`），指令會反問你模組與 US 編號。

---

**步驟 1：定位規格檔**

```
已載入以下規格檔：
  docs/specs/bs/spec_us4.md
  docs/specs/bs/spec.md
  docs/specs/bs/data-model.md
  docs/specs/bs/contracts/SRVBS003.md
  docs/wireframes/bs/index.html
✅ 已載入 5 個規格檔，開始自檢。
```

> 如果你的 repo 結構不同、找不到檔案，指令會顯示偵測到的目錄並問你正確路徑——照著回答即可，不會出錯。

---

**步驟 2~6：逐項檢查**

指令會自動讀檔，依序跑 A-1～A-5。過程不需操作，等它跑完即可。

---

**步驟 7：輸出缺口報告**

```
# BS US4 規格交付自檢報告

## 逐項結果
| 項目 | 結果 | 缺口 |
|------|------|------|
| A-1 spec 完整性 | ⚠ | AC-3「適當時間內回應」未量化 |
| A-2 data-model | ❌ | spec 提到「退回原因」，Table 無對應欄位 |
| A-3 contracts | ✅ | — |
| A-4 wireframe | ✅ | — |
| A-5 一致性交叉 | ❌ | spec_us4 FR-3 的退回流程，wireframe 無對應畫面 |

## 必補項（擋交付）
1. data-model 的領血 Table 補「退回原因」欄位（對應 FR-3）
2. wireframe 補退回流程的畫面狀態

## 建議補強（不擋交付，但 SD 可能會問）
1. AC-3 把「適當時間」改成具體秒數

## 結論
⚠ 有 2 項必補，建議先補規格再交付（比事後回 SD 問題省時）。
```

---

## 讀懂報告的三種標記

| 標記 | 意思 | 你要做的事 |
|------|------|-----------|
| ✅ 齊備 | 這項沒問題 | 不用動 |
| ⚠ 建議 | 屬 SD 自己能決定的範圍（UX、欄位形式、檔案位置），但寫清楚更好 | 可補可不補 |
| ❌ 必補 | spec 沒寫、又會影響業務行為的缺口，SD 一定會回頭問 | **交付前補齊** |

> 結論只會是兩種：**「可交付」** 或 **「先補規格再交付」**。看到後者，先把必補項補完再開 issue 給 SD。

---

## 常見情況

| 情況 | 指令的處理方式 |
|------|---------------|
| 這個 US 不涉及跨模組 | A-3 contracts 自動標 N/A，不檢查 |
| repo 規格目錄結構不一樣 | 步驟 1 找不到檔案時，顯示目錄並問你正確路徑 |
| 某份文件本 US 不適用 | 步驟 1 會問，選「跳過該項檢查」即可 |
| 只想快速看缺哪些 | 看報告的「必補項」段落就夠 |

---

## 注意事項

- 執行前請先**確認規格文件都已儲存**
- 這支指令**只讀檔、不改檔**，不會動到你的規格
- 報告是輔助判斷，最終是否交付仍由你決定；`⚠ 建議` 項可自行斟酌
- 命令檔放在 `.claude/commands/sti-sa-precheck.md`，跟著本專案一起版本控管
