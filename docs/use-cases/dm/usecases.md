# 文件管理模組（DM）使用案例（UCDM）

> 本文件依據 [../../requirements/RQDM.md](../../requirements/RQDM.md) 整理，每條 UC 標註關聯的 RQ 編號。
> DM 與教育訓練模組（ET）共用 `DP_USER`；跨模組介接詳見 RQDM.md「跨模組介接」章節與 UCDM12。

---

## 整體作業流程（角色視角）

```
編輯者作業
──────────
1. UCDM03 於文件庫檢索既有文件
2. UCDM06 新增文件或上傳新版本，送交簽核
3. UCDM05 自詳細頁發起整份文件廢止申請
4. UCDM09 於個人專區管理草稿與撤回送審

審核者作業
──────────
1. UCDM07 於簽核中心處理指派給自己之新增 / 新版本 / 廢止申請
2. UCDM09 於個人專區回顧自身審核歷史

管理者作業
──────────
1. UCDM11 於平台 DP 系統管理後台維護分類、標籤、關聯作業項目、角色指派、可見對象授權、通知範本、催辦門檻（按模組過濾；維護介面在 DP 後台）
2. UCDM08 查詢已廢止文件供稽核
3. UCDM10 跨文件查詢公開變更紀錄

閱覽者作業
──────────
1. UCDM03 於文件庫瀏覽 / 搜尋已發布文件
2. UCDM04 進入詳細頁閱讀、下載目前版本、預覽舊版本

共用作業
────────
1. UCDM01 登入 / 註冊 / 忘記密碼（由平台 DP 提供，UCDP001–003；DM 不自設登入畫面）
2. UCDM02 登入後於系統儀表板掌握各類型文件總數與最新更新公告
3. UCDM09 個人資料維護（所有角色；由平台 DP 提供，UCDP004）
```

## 共通規則

以下規則適用於所有使用案例，各 UC 描述中不再重複說明：

- **帳號共用**：DM 與 ET 共用 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名），使用者註冊一次即可登入兩系統；DM 各角色（含閱覽者）由管理者於 UCDM11（平台 DP 後台權限管理）開通、無自動授予（新帳號預設僅 ET 學員，2026-07-08）
- **多重角色**：角色（管理者 / 編輯者 / 審核者 / 閱覽者）可重複指派，互不互斥，權限取聯集；管理者於 UCDM11 維護
- **稽核欄位**：所有資料異動自動記錄建立者 / 異動者、建立時間 / 異動時間
- **文件編號**：每份文件具備唯一識別碼 DOC_ID，作為對外引用與檢索之基準
- **受控標籤庫**：標籤由管理者於 UCDM11 維護（4 組內建：適用模組 / 適用角色 / 文件性質 / 法規關聯），撰寫者僅能挑選不可自由輸入
- **單層簽核**：撰寫者送審時必選一名指定審核者（具備審核者角色），系統僅通知該指定審核者；不廣播。指定審核者名單**排除撰寫者本人**（即使兼具審核者角色亦不可選自己），確保職責分離
- **狀態流轉**：新增 / 新版本（草稿 → 送審中 → 已發布 / 已退回）；廢止（已發布 → 廢止待簽核 → 已廢止 / 回已發布）；同一文件不可同時進行兩種送審
- **歷史保留**：所有版本永久保留供稽核，不開放實體刪除，僅允許廢止整份文件
- **跨模組介接**：教材文件由 DM 提供給 ET，詳見 UCDM12

## 使用案例描述

### UCDM01 登入 / 註冊 / 忘記密碼

> **由平台 DP 提供（UCDP001–003，單一登入；DM 不自設登入畫面）**：以下流程為平台 DP 之登入 / 註冊 / 忘記密碼行為，DM 端引用；原 DM 專屬規則「註冊 / 首次進入 DM 自動授予閱覽者」**已作廢**（2026-07-08：新帳號預設僅 ET 學員，DM 角色一律由管理者開通，無任何 DM 角色者無法進入 DM）。

- **主要角色**：全體使用者
- **關聯需求**：—（分析資料）
- **前置條件**：—
- **主流程 — 登入**：
  1. 使用者於登入頁輸入帳號（Email）與密碼
  2. 系統驗證後寫入登入 session 並導向系統儀表板（UCDM02）
- **主流程 — 註冊**：
  1. 使用者於登入頁切換至「註冊」頁籤
  2. 填寫帳號（Email）、密碼、確認密碼、姓名
  3. 系統檢核 Email 未被註冊、密碼兩次一致後建立共用 `DP_USER` 紀錄
  4. 系統跳回登入頁並預填 Email，由使用者重新登入（不自動授予 DM 角色；預設僅 ET 學員，DM 存取由管理者開通）
- **主流程 — 忘記密碼**：
  1. 使用者於登入頁點擊「忘記密碼」並輸入註冊 Email
  2. 系統寄發密碼重設連結至該 Email（30 分鐘內有效）
  3. 使用者點擊連結進入重設密碼頁，輸入新密碼後儲存
- **替代流程**：
  - 查無此帳號 → 顯示「查無此帳號，請先註冊」
  - 密碼錯誤 → 顯示「密碼錯誤」
  - 重設連結逾時 → 顯示「連結已失效，請重新申請」
- **後置條件**：使用者成功登入並進入系統儀表板

### UCDM02 系統儀表板

- **主要角色**：全體使用者
- **關聯需求**：—（分析資料）
- **前置條件**：使用者已登入
- **主流程**：
  1. 系統於登入後自動導向儀表板（無導覽選單入口；自系統名稱或返回首頁按鈕亦可回到本頁）
  2. 上方「各類型文件總數」呈現四類分類（標準作業程序 / 系統操作手冊 / 訓練教材 / 其他）之目前已發布文件總數
  3. 下方「最新更新公告」列出近 30 天內已發布文件，含新增首版與新版本兩類事件
  4. 使用者點任一公告進入該文件之詳細頁（UCDM04）；點「查看全部文件」進入文件庫（UCDM03）
- **替代流程**：
  - 無近 30 天內發布事件 → 公告區顯示「近期無新發布文件」
- **後置條件**：使用者掌握各分類文件數量與近期動態，可進一步檢視單一文件

---

### UCDM03 文件庫與檢索

- **主要角色**：全體使用者
- **關聯需求**：RQDM001, RQDM004
- **前置條件**：使用者已登入
- **主流程 — 搜尋**：
  1. 使用者於文件庫頁面以下列條件搜尋：關鍵字（文件名 / 變更摘要）、分類、作者、標籤（多選 AND）、發布日期區間（分類選「系統操作手冊」時，額外顯示「關聯作業項目」下拉選單）
  2. 系統列出符合條件之文件清單，依發布時間由新到舊排序分頁顯示
  3. 列表欄位：文件名稱、分類、發布日期、作者、標籤；系統操作手冊類額外顯示關聯作業項目
- **主流程 — 線上操作手冊檢索**：
  1. 使用者選擇分類「系統操作手冊」後，於額外出現之「關聯作業項目」下拉選單選取欲查詢之作業項目（func_name；來自受控清單，不可自由輸入）
  2. 系統列出對應之系統操作手冊文件，使用者可即時取得指導說明
- **主流程 — 進入詳細頁**：
  1. 使用者點任一文件進入 UCDM04 文件詳細頁瀏覽
  2. 具編輯者角色者另可從清單右側「新增文件」進入 UCDM06 新增模式
- **替代流程**：
  - 無條件輸入即查詢 → 列出全部已發布文件
  - 已廢止文件 → 不出現於本清單（管理者可至 UCDM08 查詢）
  - 廢止待簽核期間文件 → 因仍對外有效，持續顯示於清單，直到核准廢止後才下架
- **後置條件**：使用者取得目標文件清單，可進入詳細頁或新增文件

### UCDM04 文件詳細頁瀏覽

- **主要角色**：全體使用者
- **關聯需求**：RQDM001, RQDM002
- **前置條件**：使用者於 UCDM02 或 UCDM03 點選文件
- **主流程**：
  1. 上方標題列僅顯示識別與狀態資訊：文件名稱、文件編號（DOC_ID）、目前版本、狀態（已發布 / 已廢止）；描述性資訊（分類、作者、核准者、發布日期等）統一由右側「文件資訊」面板呈現，不於上方重複
  2. 中央「文件檔案」區呈現目前發布版本之檔案，提供下載；PDF 與圖片可線上預覽，Office 檔（Word / Excel / PPT）顯示「下載原檔以本機應用程式開啟」
  3. 右側面板顯示文件資訊（分類、作者、發布日期、核准者、核准時間、標籤）
  4. 使用者按「版本歷程」展開抽屜，以時間軸呈現所有版本（含已被取代之舊版本）之版號、撰寫者、發布時間、核准者、變更摘要
  5. 目前發布版本可下載；舊版本僅提供預覽，不開放下載（避免使用者誤用過時規範）
- **替代流程**：
  - 具編輯者角色 → 額外顯示「編輯新版本」與「廢止此文件」兩個動作入口（→ UCDM06 / UCDM05）
  - 文件處於新版本送審中或廢止待簽核狀態 → 編輯與廢止入口暫時失效，避免兩個送審週期並存
  - 自 UCDM08（已廢止文件查詢）進入者 → 以唯讀模式呈現，文件檔案區隱藏並自動展開版本歷程；所有版本僅可預覽
- **後置條件**：使用者完成閱覽；必要時下載原檔或進入編輯 / 廢止流程

### UCDM05 文件廢止申請

- **主要角色**：編輯者
- **關聯需求**：RQDM001
- **前置條件**：文件處於已發布狀態，且當前使用者具備編輯者角色
- **主流程**：
  1. 編輯者於 UCDM04 文件詳細頁點擊「廢止此文件」
  2. 系統開啟廢止申請對話框，必填廢止原因
  3. 編輯者選擇指定審核者（從具備審核者角色之名單）後送出
  4. 系統將文件狀態變更為「廢止待簽核」並通知指定審核者；該文件仍持續對外提供閱讀與下載
  5. 後續由 UCDM07 簽核處理：核准 → 已廢止；退回 → 回到已發布
- **替代流程**：
  - 文件正在進行新版本送審 → 不可同時發起廢止；系統阻擋並提示
  - 撰寫者欲撤回廢止申請 → 至 UCDM09 個人專區執行撤回，文件回到已發布
- **後置條件**：廢止申請進入簽核流程；待審核者於 UCDM07 處理

---

### UCDM06 文件新增與編輯

- **主要角色**：編輯者
- **關聯需求**：RQDM001, RQDM002
- **前置條件**：當前使用者具備編輯者角色
- **主流程 — 新增**：
  1. 編輯者於 UCDM03 點「新增文件」進入新增模式
  2. 填寫基本資料：文件名稱、分類、標籤（多選，從受控標籤庫挑選）；若分類為「系統操作手冊」需額外指定關聯作業項目（單選）
  3. 指定首版版本號（系統建議 v1.0，可調整）；填寫首版摘要
  4. 上傳檔案：拖曳或點選；支援 PDF / Word / Excel / PowerPoint / 圖片，單檔上限 50 MB
  5. 選擇指定審核者後送出簽核
- **主流程 — 編輯（上傳新版本）**：
  1. 編輯者於 UCDM04 點「編輯新版本」進入編輯模式，文件名稱 / 分類 / 關聯作業項目為唯讀
  2. 系統建議下一版號（+0.1 / Major / Minor），編輯者可調整
  3. 填寫變更摘要（必填）並上傳新版本檔案
  4. 選擇指定審核者後送出簽核
- **替代流程**：
  - 上傳 Office 檔（Word / Excel / PPT）→ 系統提示閱讀者於詳細頁無法線上預覽；編輯者可改傳 PDF 或選「仍使用此檔案」繼續（PDF 與圖片可預覽，不提示）
  - 點「儲存為草稿」 → 文件存為草稿狀態；可於 UCDM09 個人專區之文件草稿區繼續編輯
  - 點「取消新增 / 取消編輯」 → 若表單已有未儲存變更則跳出確認對話框
  - 文件處於廢止待簽核狀態 → 不可同時上傳新版本；系統阻擋並提示
  - 系統操作手冊指定之 func_name 已被其他「已發布」手冊使用 → 系統於送審 / 發布時阻擋並提示「此關聯作業項目已有對應手冊」（同一 func_name 至多一份已發布手冊）
- **後置條件**：文件送審進入簽核流程（→ UCDM07）；或存為草稿待後續編輯

### UCDM07 簽核處理

- **主要角色**：審核者
- **關聯需求**：RQDM001
- **前置條件**：當前使用者具備審核者角色，且有送審項目指派給自己
- **主流程 — 待簽核**：
  1. 審核者進入簽核中心，預設「待簽核」頁籤
  2. 系統列出指派給自己之送審項目（新增 / 新版本 / 廢止三類），含文件名稱、分類、版本、撰寫者、送審時間、停留天數
  3. 審核者點任一筆展開明細，檢視變更摘要（或廢止原因）與送審檔案資訊；可下載檔案查看（新版本申請另提供新舊版並列下載比對）
- **主流程 — 核准**：
  1. 審核者按「核准並發布」（新增 / 新版本）或「核准並廢止」（廢止）
  2. 系統二次確認後執行：發布類即建立新版本並轉為已發布；廢止類即轉為已廢止
  3. 系統通知撰寫者結果並寫入文件變更歷程（UCDM10）
- **主流程 — 退回**：
  1. 審核者按「退回」開啟對話框，必填退回原因
  2. 系統將文件轉回對應狀態（草稿 / 已發布）並通知撰寫者；文件草稿出現於撰寫者之 UCDM09 個人專區供繼續編輯
- **主流程 — 已完成**：
  1. 審核者切至「已完成」頁籤，呈現自己過往處理過之項目（已核准 / 已退回）；提供搜尋與分頁瀏覽
  2. 已完成項目不可再操作
- **替代流程**：
  - 送審項目停留 ≥ 催辦門檻天數 → 系統每日自動發送催辦訊息給審核者；清單同時標示已停留天數
  - 撰寫者撤回送審 → 該項目自待簽核清單消失；審核者於 UCDM09 個人專區之「我的文件動態」可見「已被撰寫者撤回」事件
- **後置條件**：送審項目完成核准 / 退回；文件狀態相應更新並通知撰寫者

### UCDM08 已廢止文件查詢

- **主要角色**：管理者
- **關聯需求**：RQDM001
- **前置條件**：當前使用者具備管理者角色
- **主流程**：
  1. 管理者自導覽選單進入已廢止文件查詢
  2. 以條件搜尋：關鍵字（文件名 / 廢止原因）、分類、廢止日期區間
  3. 系統列出已廢止文件，欄位包含：文件名稱（含末版版號）、分類、原撰寫者、廢止時間、廢止申請人、核准者、廢止原因
  4. 管理者點任一筆進入文件詳細頁唯讀模式（→ UCDM04 替代流程）；版本歷程自動展開
  5. 必要時按「匯出 CSV」匯出當前查詢結果供稽核資料封存
- **替代流程**：
  - 一般使用者於導覽選單看不到此項目；無法直接以 URL 進入（系統阻擋）
- **後置條件**：管理者取得已廢止文件清單；可進一步閱覽單一文件之歷史脈絡

### UCDM09 個人專區（資料 / 草稿 / 動態）

- **主要角色**：所有角色（依分區）
- **關聯需求**：—（分析資料）
- **前置條件**：使用者已登入
- **主流程 — 個人資料維護**（所有 4 角色；**由平台 DP 提供，UCDP004，經右上使用者選單進入，DM 不自設此畫面**——以下為 DP 個資行為之引用）：
  1. 使用者於個人專區「個人資料維護」區檢視自己之帳號（Email）、姓名、密碼變更入口
  2. 編輯姓名後直接儲存
  3. 編輯帳號（Email）按「變更」後系統寄發驗證信至新 Email；點驗證連結後新帳號才生效
  4. 變更密碼按「變更密碼」開啟對話框，填寫舊密碼 + 新密碼 + 確認新密碼後儲存；同步更新共用 `DP_USER`，ET 端亦生效
- **主流程 — 文件草稿**（編輯者）：
  1. 個人專區「文件草稿」區呈現未送審草稿 / 被退回待修改 / 已撤回送審三類
  2. 按「繼續編輯」或「修改」進入 UCDM06 編輯畫面；按「刪除」可刪除該草稿（不可復原）
- **主流程 — 撤回送審**（編輯者）：
  1. 對於已送出但卡在審核者端之送審項目（例：審核者請假 / 長期未處理），編輯者於本區按「撤回送審」
  2. 系統將文件狀態回復：新增 / 新版本送審 → 草稿；廢止申請 → 已發布
  3. 系統通知原指派之審核者（僅系統內訊息，不發 Email）
  4. 編輯者可重新編輯並改選新審核者再送（→ UCDM06）
- **主流程 — 我的文件動態**（編輯者 / 審核者）：
  1. 「我的文件動態」區依角色分頁籤呈現近 30 天事件
  2. 撰寫者視角：列出我撰寫之文件事件（送審中 / 核准發布 / 退回 / 廢止待簽核 / 已廢止 / 已撤回）
  3. 審核者視角：列出我作為審核者之事件（待處理 / 催辦中 / 已被撰寫者撤回 / 已處理歷史）
  4. 點任一事件進入對應之詳細頁或編輯畫面
- **替代流程**：
  - 帳號（Email）變更於驗證信有效期內未點擊 → 新帳號不生效，舊帳號維持
  - 舊密碼錯誤 → 提示「舊密碼不正確」
  - 閱覽者與管理者（未兼具編輯者 / 審核者角色時）僅看得到個人資料維護分區；文件草稿與我的文件動態不顯示
- **後置條件**：個人資料 / 草稿 / 送審狀態已更新

### UCDM10 文件變更歷程查詢

- **主要角色**：管理者
- **關聯需求**：RQDM001
- **前置條件**：當前使用者具備管理者角色
- **主流程**：
  1. 管理者自導覽選單進入文件變更歷程查詢
  2. 以條件搜尋：日期區間、申請人 / 核准人（帳號或姓名）、操作類型（全部 / 發布 / 廢止）
  3. 系統列出變更紀錄，依時間由新到舊呈現，欄位包含：時間、申請人、核准人、操作（發布 / 廢止）、文件名稱、版本號、備註（廢止原因 / 變更摘要）
  4. 必要時按「匯出 CSV」匯出查詢結果供資安稽核
- **替代流程**：
  - 撰寫過程動作（上傳 / 編輯 / 送審 / 退回 / 撤回）為撰寫者個人視角，不寫入此公開歷程；改由 UCDM09 個人專區之我的文件動態呈現
  - 閱讀類動作（下載 / 預覽）不記錄
- **後置條件**：管理者取得跨文件變更紀錄；可進一步匯出供稽核

### UCDM11 系統設定（維護介面於平台 DP 後台）

> **維護介面移至平台 DP 系統管理後台（2026-07-08 集中化）**：權限管理、系統參數與清單、通知範本統一於平台 DP 後台維護，DM 管理者按模組過濾操作（只見 / 只改 `MODULE=DM` / `DM_` 前綴），**DM 不自設系統設定畫面（原 DM09）**。以下流程之業務規則不變，僅畫面落點改為 DP 後台；「定義」存平台 `DP_PARAM` / `DP_NOTIFY_TEMPLATE`，「指派 / 關聯」（`DM_USER_ROLE` / `DM_USER_TAG`）與角色判定留 DM。

- **主要角色**：管理者
- **關聯需求**：RQDM001
- **前置條件**：當前使用者具備管理者角色，於平台 DP 系統管理後台選擇 DM 模組
- **主流程 — 系統參數與清單**：
  1. 管理者進入平台 DP 後台「系統參數與清單」（DM 模組）
  2. 維護文件分類（4 內建分類 + 自訂）：可新增 / 改名 / 啟用 / 停用，不開放實體刪除
  3. 維護關聯作業項目清單（func_name）：可新增 / 改名 / 啟用 / 停用
  4. 維護標籤（4 內建組 + 群組內標籤）：可新增 / 改名 / 啟用 / 停用
  5. 設定自動催辦門檻（預設 7 天，可調 1–30 天）
- **主流程 — 權限管理**：
  1. 管理者進入平台 DP 後台「權限管理」（DM 模組），系統列出共用 `DP_USER` 之使用者
  2. 對任一使用者勾選 / 取消勾選 DM 角色（管理者 / 編輯者 / 審核者 / 閱覽者），可複選；並維護「閱覽者可見對象授權」（使用者 × 可見對象標籤）
  3. 系統即時生效並寫入異動紀錄（角色寫 `DM_USER_ROLE`、可見對象授權寫 `DM_USER_TAG`）
- **主流程 — 通知範本**：
  1. 管理者進入平台 DP 後台「通知範本」（DM 模組），系統列出 DM 內建事件
  2. 對任一範本可調整主旨 / 內文、啟用 / 停用；自動催辦另含催辦門檻天數設定
- **替代流程**：
  - 管理者嘗試停用自己之管理者角色 → 系統阻擋並提示
  - 停用之分類 / 標籤 / 關聯作業項目 → 既有文件之標記不受影響，但撰寫者新建時不顯示
  - 離職 / 調職 → 由共用 `DP_USER` 之管理介面停用該帳號，DM 不另外處理
- **後置條件**：系統參數 / 權限 / 通知範本已更新；下次使用者操作時依新設定生效
- **平台對齊（2026-07-08 集中化）**：本 UC 之**維護 / 編輯介面統一於平台 DP 系統管理後台**（權限管理、系統參數與清單、通知範本；DM 管理者按模組過濾操作、只編輯 `MODULE=DM` / `DM_` 前綴的項，DM 不自設系統設定畫面）；儲存亦集中於平台 DP——參數存 `DP_PARAM`（前綴 `DM_`）、通知範本存 `DP_NOTIFY_TEMPLATE`（`MODULE=DM`）；「指派 / 關聯」（使用者×角色 `DM_USER_ROLE`、使用者×可見對象授權 `DM_USER_TAG`、文件×標籤）與角色判定、業務規則落地為 DM 自持不變（見 [`_refs/09-平台模組.md`](../../_refs/09-平台模組.md) 決策紀錄）

### UCDM12 跨模組教材引用（DM ↔ ET）

- **主要角色**：DM 編輯者 / ET 教師
- **關聯需求**：RQDM003
- **前置條件**：DM 端已建立並發布「訓練教材」分類之文件；ET 課程處於建立 / 編輯狀態
- **主流程 — ET 教材引用 DM 文件**：
  1. ET 教師於教材編輯介面從 DM「訓練教材」分類下拉選取既有文件，建立 DOC_ID 引用
  2. 學員於 ET 章節學習時，系統依引用之 DOC_ID 自動取得 DM 當前發布版本之檔案
- **主流程 — DM 發布新版本後自動帶最新版**：
  1. DM 編輯者於 UCDM06 上傳新版本並完成簽核後（→ UCDM07 核准）
  2. ET 端引用該 DOC_ID 之教材自動取得最新版（依 CURRENT_VERSION_ID），不需 ET 端手動更新
- **替代流程**：
  - DM 文件被廢止 → ET 教材引用之內容仍可呈現廢止前最後版本；DM 端通知 ET 教師檢視是否需移除引用
  - DM 文件處於廢止待簽核期間 → ET 端仍取得廢止前最後發布版本（因 DM 端文件仍對外有效）
- **後置條件**：ET 教材內容與 DM 文件保持同步；DM 端版本更新即時反映於 ET 端
