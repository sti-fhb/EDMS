# 教育訓練文件管理模組（ET）使用案例（UCET）

> 本文件依據 [../../requirements/RQET.md](../../requirements/RQET.md) 整理，每條 UC 標註關聯的 RQ 編號。
> 教材文件由文件管理模組（DM）提供；ET 與 DM 共用 user table，跨模組介接詳見 RQET.md 跨模組介接章節。

---

## 整體作業流程（角色視角）

```
教師作業
────────
1. UCET001 於課程列表瀏覽自己 / 全部課程
2. UCET002 建立新課程（基本資料 → 章節編排 → 教材 → 測驗 → 發布）
3. UCET004 邀請學員（Email / 邀請碼 / 模組預設帶入）
4. UCET005 追蹤學員學習狀況；必要時 UCET006 追蹤待加入邀請
5. UCET003 課程結束後停課

學員作業
────────
1. UCET012 登入系統（首次使用者於登入頁註冊）
2. UCET007 於我的課程瀏覽已加入課程 / 輸入邀請碼加入新課程
3. UCET008 依章節順序學習教材（影片 / 文件 / 說明文字）
4. UCET009 完成章節後作答線上測驗
5. 重複學習至全部章節通過

管理者作業
──────────
1. UCET010 設定其他帳號之角色（管理者 / 教師 / 學員）與業務模組對應

共用作業
────────
- UCET011 所有登入使用者皆可維護自己之個人資料
```

## 共通規則

以下規則適用於所有使用案例，各 UC 描述中不再重複說明：

- **帳號共用**：ET 與 DM 共用 user table（USER_ID / 帳號 / 密碼 / 姓名），使用者註冊一次即可登入兩系統；ET 端首次登入自動授予「學員」角色
- **多重角色**：角色（管理者 / 教師 / 學員）可重複指派，權限取聯集；登入後依角色預設首頁導向，多重角色由側邊選單切換
- **稽核欄位**：所有資料異動自動記錄建立者 / 異動者、建立時間 / 異動時間
- **教材文件來源**：所有教材引用之文件統一由 DM 模組之「訓練教材」分類提供；DM 端發布新版時 ET 自動取得最新版
- **擁有權判定**：課程之擁有者於建立當下記錄並永久不可變更；他人課程僅可閱覽（唯讀），編輯與其他操作按鈕皆不顯示
- **跨模組介接**：詳見 RQET.md 跨模組介接章節

## 使用案例描述

### UCET001 課程列表瀏覽

- **主要角色**：教師
- **關聯需求**：—（分析資料）
- **前置條件**：使用者具備教師角色並已登入
- **主流程**：
  1. 教師登入後進入課程列表（教師預設首頁）
  2. 選擇分頁：「我建立的」僅顯示當前使用者建立之課程；「全部課程」顯示所有教師建立之課程
  3. 透過搜尋區篩選：關鍵字（課程名稱）、關聯模組；「全部課程」分頁額外提供「建立者」篩選
  4. 系統依「關聯模組」分組呈現課程卡片（採血 / 成分 / 檢驗 / 供應 / 醫務 / 報表與標籤 / 其他）
  5. 教師點擊卡片進入課程編輯（自己建立）或檢視（他人建立、唯讀）頁面
- **替代流程**：
  - 點擊搜尋區右側「新增課程」→ 進入 UCET002 課程建立流程
  - 卡片右上顯示「檢視」標籤者為他人建立之課程，進入後為唯讀模式
  - 已停課之課程於列表顯示「已停課」狀態標示，點擊進入後同樣為唯讀模式
- **後置條件**：教師取得課程清單；可進一步進入編輯（UCET002）、停課（UCET003）、邀請（UCET004）或學員追蹤（UCET005）

---

### UCET002 課程建立與編輯

- **主要角色**：教師
- **關聯需求**：RQET001, RQET002, RQET003, RQET005
- **前置條件**：教師於 UCET001 點擊「新增課程」或既有課程卡片
- **主流程 — 基本資料**：
  1. 教師填寫課程名稱（必填）、關聯模組（必填）、課程描述（選填）
  2. 系統於課程建立後固定「關聯模組」欄位（避免影響已依模組預設帶入之學員名單）
  3. 開課日期由系統於首次發布時自動寫入，教師不需手動填寫
- **主流程 — 章節編排**：
  1. 教師於章節編排區新增章節，並以拖拉方式調整章節順序、章節下教材與測驗之順序
  2. 每個章節下可放置任意數量之教材與測驗項目
  3. 刪除章節時跳 confirm modal；確認後章節本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員之學習紀錄與測驗成績連同移除；後續章節順序自動往前遞補
- **主流程 — 教材管理**：
  1. 點擊章節中「新增項目 → 教材」開啟教材編輯視窗
  2. 教材項目由三類媒材自由組合：影片（本地上傳 mp4 / webm，單檔 ≤ 500 MB）、文件（DM 引用）、說明文字（WYSIWYG）
  3. 同一教材項目可同時包含多支影片、多份文件與說明文字
  4. 教師從 DM「訓練教材」分類下拉選取既有文件建立關聯；DM 端發布新版時自動取得最新版
- **主流程 — 測驗建立**：
  1. 點擊章節中「新增項目 → 測驗」開啟測驗編輯視窗
  2. 教師設定測驗名稱（必填）、及格分數（預設 80）、作答時間限制（必填）、重考次數上限（預設 3）
  3. 教師建立題目（單選題 / 多選題），設定題幹、選項（2–6 個）、正確答案、配分（同一測驗各題配分總和 = 100）
  4. 題目順序可拖拉調整；刪除題目時題目本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員之該題作答紀錄與得分連同移除
- **主流程 — 儲存與發布**：
  1. 教師於頁面底部選擇「取消編輯 / 儲存草稿 / 發布」其中之一
  2. 「儲存草稿」：寫入草稿狀態，學員端不顯示
  3. 「發布」：系統檢核「至少 1 章節 + 1 教材」；通過後寫入發布時間戳（即開課日期），開放學員加入
- **替代流程**：
  - DM 文件廢止 → 教材頁顯示警告，由教師決定是否移除引用；學員端仍可閱讀廢止前最後版本
  - 發布檢核未通過 → 阻擋並提示缺漏項目
  - 課程進入已發布狀態後再次編輯 → 儲存後生效，不需重新發布
- **後置條件**：課程已建立或更新；若已發布則學員可於 UCET007 加入並於 UCET008 學習

### UCET003 課程停課

- **主要角色**：教師
- **關聯需求**：—（分析資料）
- **前置條件**：課程處於已發布狀態，且當前使用者為課程擁有者
- **主流程**：
  1. 教師於課程編輯頁右上點擊「停課」按鈕（草稿與已停課狀態不顯示）
  2. 系統跳 confirm modal 提醒停課不可逆
  3. 教師確認後系統將課程狀態變更為「已停課」
  4. 系統於列表顯示「已停課」狀態標示；學員端不再顯示於進行中清單
  5. 學員嘗試開啟已停課課程時顯示「此課程已停止開放」訊息頁
- **替代流程**：
  - 已停課狀態為終態，不可再恢復為已發布
  - 已加入學員之歷史紀錄完整保留，僅停止新進與學習動作
- **後置條件**：課程已停課；教師可閱覽歷史紀錄但無法再行編輯或邀請

### UCET004 邀請學員

- **主要角色**：教師
- **關聯需求**：RQET006
- **前置條件**：課程處於已發布狀態
- **主流程 — 模組預設帶入**：
  1. 教師於 UCET002 建立課程時指定「關聯模組」
  2. 系統依「使用者×業務模組」對應（由 UCET010 維護）自動加入該模組所屬人員
- **主流程 — Email 邀請**：
  1. 教師於課程編輯頁點擊「邀請學員 → Email 邀請」
  2. 教師輸入學員 Email 清單，系統產生邀請信預覽（含課程名稱、邀請連結、邀請碼）
  3. 教師可手動編輯主旨與內文後寄出；系統呼叫外部 Email Server 寄送
  4. 學員點擊邀請連結後自動加入課程
- **主流程 — 邀請碼**：
  1. 課程發布時系統自動產生唯一邀請碼（8 碼）
  2. 教師點擊「邀請學員 → 邀請碼」複製邀請碼提供學員自行加入
  3. 學員於 UCET007 輸入邀請碼加入
- **替代流程**：
  - 已加入之學員再次收到邀請 → 點擊後直接進入 UCET008 學習頁
  - 邀請寄送失敗 → 寄出狀態保留為「待加入」，教師可於 UCET006 再次寄送
- **後置條件**：學員加入課程（已加入分頁）或進入待加入清單（UCET006）

---

### UCET005 學員學習狀況追蹤

- **主要角色**：教師
- **關聯需求**：RQET008
- **前置條件**：教師具備課程之擁有權，課程已有學員加入
- **主流程 — 查詢**：
  1. 教師於頂部下拉選單切換要檢視之課程
  2. 系統於「已加入」分頁列出學員清單：學員、加入日期、完課狀態（已完成 / 進行中 / 未開始）、學習進度、平均成績、最後活動
  3. 教師依條件篩選或排序檢視
- **主流程 — 操作個別學員**：
  1. **重置重考次數**：當學員測驗達上限仍未及格，教師可重置次數讓學員恢復可用次數
  2. **移除學員**：學員從前台不再顯示，但學習歷史紀錄完整保留；移除後不計入完課率統計
- **主流程 — 匯出**：
  1. 教師點擊「匯出 CSV」依當前篩選條件匯出清單
- **替代流程**：
  - 課程已停課 → 仍可閱覽學員清單，但不可再執行重置 / 移除
- **後置條件**：教師掌握學員學習狀況；必要時已執行重置或移除動作

### UCET006 待加入邀請追蹤

- **主要角色**：教師
- **關聯需求**：RQET006
- **前置條件**：教師於 UCET004 寄出 Email 邀請後存在尚未加入之學員
- **主流程**：
  1. 教師於 UCET005 頁面切至「待加入」分頁
  2. 系統列出已寄出邀請但尚未加入之學員清單（Email、寄送時間、邀請狀態）
  3. 教師依清單對個別邀請執行「再次寄送」或「撤回邀請」
  4. 學員於收到邀請後加入課程，系統自動將其從「待加入」移至「已加入」分頁
- **替代流程**：
  - 撤回邀請後該 Email 對應之邀請連結失效，學員若點擊將顯示邀請已撤回
- **後置條件**：待加入清單已更新；學員加入後進入 UCET005 已加入清單

---

### UCET007 我的課程與加入新課程

- **主要角色**：學員
- **關聯需求**：RQET006
- **前置條件**：學員已登入系統
- **主流程 — 瀏覽**：
  1. 學員登入後進入我的課程頁（學員預設首頁）
  2. 頁面上方依學習狀態分區呈現總數：進行中 / 未開始 / 已完成
  3. 下方依關聯模組分組呈現各課程卡片（課程名稱、當前進度、章節等資訊）
  4. 學員點擊課程卡片進入 UCET008 學習頁面
- **主流程 — 加入新課程**：
  1. 學員點擊「+ 加入新課程」按鈕，輸入 8 碼邀請碼
  2. 系統驗證邀請碼有效後顯示課程資訊（課程名稱、教師、章節數等）
  3. 學員確認後加入課程，系統將該課程加入「未開始」清單
- **替代流程**：
  - 邀請碼無效（不存在或已重置）→ 顯示錯誤訊息
  - 該邀請碼對應之課程學員已加入 → 直接跳轉至 UCET008
  - 課程已停課 → 顯示「此課程已停止開放」訊息頁
- **後置條件**：學員取得課程清單；新加入課程出現於「未開始」分區

### UCET008 章節學習

- **主要角色**：學員
- **關聯需求**：RQET001, RQET003, RQET004
- **前置條件**：學員已加入課程
- **主流程**：
  1. 學員於 UCET007 點擊課程進入學習頁，系統載入該課程之章節導覽列
  2. 系統依「上次觀看位置」自動定位至學員上次學習處
  3. 學員於中間內容區依章節順序學習：
     - 影片：以 HTML5 video player 播放（含播放 / 暫停 / 調速 / 音量 / 全螢幕）
     - DM 文件：PDF 於頁內嵌入預覽；非 PDF 顯示「下載原檔以本機應用程式開啟」連結
     - 說明文字：直接顯示
  4. 系統記錄學習進度（影片播放進度、文件閱讀狀態）
  5. 完成當前章節後系統依解鎖規則開放下一章節
- **解鎖規則**：
  - 上一章節之所有影片需播放超過 80%
  - 若該章節有測驗，學員須達到及格分數方可解鎖（→ UCET009）
- **替代流程**：
  - 教材內容可重複觀看，不影響進度
  - 課程於學習中被停課 → 系統顯示「此課程已停止開放」，學員無法繼續學習
- **後置條件**：學員完成當前章節之學習；可繼續下一章節或執行 UCET009 測驗

### UCET009 線上測驗作答

- **主要角色**：學員
- **關聯需求**：RQET005
- **前置條件**：學員已通過當前章節學習，章節中設有測驗
- **主流程 — 引導頁**：
  1. 系統顯示測驗引導頁：測驗名稱、題數、及格分數、作答時間限制、剩餘重考次數、上次成績
  2. 學員點擊「開始測驗」進入答題介面
- **主流程 — 答題**：
  1. 系統依內建機制將題目順序自動洗牌
  2. 學員於答題介面操作：上方狀態列含題號進度、倒數計時、提交按鈕；左側題目導覽列可任意跳題
  3. 單選題以 radio 選擇、多選題以 checkbox 選擇；切換題目時系統自動暫存當前作答
- **主流程 — 提交與閱卷**：
  1. 學員點擊「提交」或時間到自動以當前作答狀態提交
  2. 系統即時自動閱卷並顯示總分、是否及格、答題明細
  3. 學員點擊明細可看題目原文與正確答案複習
- **多選題評分規則**：採部分計分公式：`max(0, (答對正確選項數 − 誤選數) ÷ 應選正確數 × 該題配分)`；每答對一個正確選項加分、每選錯一個錯誤選項扣分，最低為 0 分
- **替代流程**：
  - 未達及格分數且剩餘重考次數 > 0 → 學員可選擇「重新作答」
  - 未達及格且無剩餘重考次數 → 等待教師於 UCET005 重置次數後方可重考
  - 作答中切換瀏覽器頁面 → 系統提示確認避免誤離
  - 每次作答之成績皆獨立記錄；以最高分為結業成績
- **後置條件**：學員取得測驗成績；通過則解鎖下一章節（→ UCET008）

### UCET010 權限管理

- **主要角色**：管理者
- **關聯需求**：RQET007
- **前置條件**：當前使用者具備管理者角色
- **主流程 — 角色指派**：
  1. 管理者進入權限管理頁，依條件查詢目標使用者
  2. 系統列出使用者清單，每筆顯示三角色（管理者 / 教師 / 學員）核取方塊與業務模組對應
  3. 管理者勾選 / 取消勾選對應角色（聯集生效）後儲存
  4. 系統記錄修改者與時間於「最後修改」欄；完整變更歷史寫入稽核 log
- **主流程 — 業務模組對應**：
  1. 管理者於使用者列展開「業務模組」設定
  2. 勾選使用者所屬之業務模組（多對多）
  3. 系統將該對應於教師建立課程指定「關聯模組」時用於自動加入該模組人員
- **替代流程**：
  - 當前登入之管理者嘗試停用自己之管理者角色 → 系統阻擋並提示「無法停用自己之管理者角色」
  - 新使用者首次登入後自動授予「學員」角色，業務模組對應預設「未指派」
- **後置條件**：使用者之角色與業務模組對應已更新；下次登入時依角色導向預設首頁

### UCET011 個人資料維護

- **主要角色**：使用者本人（所有登入使用者）
- **關聯需求**：—（分析資料）
- **前置條件**：使用者已登入
- **主流程 — 編輯姓名**：
  1. 使用者於個人資料維護頁編輯姓名後儲存
  2. 系統同步寫入共用 user table，DM 端同步生效
- **主流程 — 變更帳號（Email）**：
  1. 使用者輸入新 Email 並儲存
  2. 系統寄發驗證信至新 Email
  3. 使用者點擊驗證連結後新帳號才生效
- **主流程 — 變更密碼**：
  1. 使用者點擊「變更密碼」開啟 modal
  2. 填寫舊密碼 + 新密碼 + 確認新密碼
  3. 系統檢核舊密碼正確、新密碼兩次一致後儲存
- **替代流程**：
  - 驗證信未於有效時間內點擊 → 新 Email 不生效，舊帳號維持
  - 舊密碼錯誤 → 提示「舊密碼不正確」
  - 忘記密碼改走 UCET012「忘記密碼」流程
- **後置條件**：個人資料已更新並同步寫入共用 user table

### UCET012 登入 / 註冊 / 忘記密碼

- **主要角色**：全體使用者
- **關聯需求**：—（分析資料）
- **前置條件**：—
- **主流程 — 登入**：
  1. 使用者於登入頁輸入帳號（Email）與密碼
  2. 系統驗證帳號 / 密碼正確後寫入登入 session
  3. 系統依使用者角色導向預設首頁：管理者 → 權限管理；教師 → 課程列表；學員 → 我的課程
  4. 若使用者具備多重角色，登入後由側邊選單自行切換
- **主流程 — 註冊**：
  1. 使用者於登入頁點擊「註冊」進入註冊頁
  2. 填寫帳號（Email）、姓名、密碼、確認密碼
  3. 系統檢核 Email 未被註冊、密碼兩次一致後建立共用 user table 紀錄
  4. ET 端自動授予「學員」角色；系統跳回登入頁，使用者以新帳號重新登入
- **主流程 — 忘記密碼**：
  1. 使用者於登入頁點擊「忘記密碼」並輸入註冊之 Email
  2. 系統寄發密碼重設連結至 Email（30 分鐘內有效）
  3. 使用者點擊連結進入重設密碼頁，輸入新密碼後儲存
  4. 使用者以新密碼重新登入
- **替代流程**：
  - 查無此帳號 → 提示「查無此帳號，請先註冊」
  - 密碼錯誤 → 提示「密碼錯誤」
  - 重設連結逾 30 分鐘 → 提示「連結已失效，請重新申請」
- **後置條件**：使用者已成功登入並進入對應預設首頁
