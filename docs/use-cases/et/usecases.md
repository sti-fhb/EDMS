# 教育訓練文件管理模組（ET）使用案例（UCET）

> 本文件依據 [../../requirements/RQET.md](../../requirements/RQET.md) 整理，每條 UC 標註關聯的 RQ 編號。
> 教材文件由文件管理模組（DM）提供；ET 與 DM 共用 `DP_USER`，跨模組介接詳見 RQET.md 跨模組介接章節。
> **2026-07-02 更新（客戶 6 項變更）**：關聯模組改受訓單位標籤（UCET002/004/010）、發布標籤自動邀請＋寄信（UCET004）、課程起訖時間與關閉/再開課（UCET002/003）、每週統計與提醒（UCET014 新增）、課後問卷（UCET013 新增）與每次作答明細（UCET005/009）、影片覆蓋率規則明文（UCET008）、通知範本維護（UCET015 新增）。

## 整體作業流程（角色視角）

```
教師作業
────────
1. UCET001 於課程列表瀏覽自己 / 全部課程
2. UCET002 建立新課程（基本資料（標籤、起訖時間）→ 章節編排 → 教材 → 測驗 → 問卷 → 發布）
3. UCET004 邀請學員（發布時標籤自動邀請＋寄信；Email / 邀請碼補件）
4. UCET005 追蹤學員學習狀況（含每次作答明細、問卷結果）；必要時 UCET006 追蹤待加入邀請
5. UCET003 課程到期自動關閉或手動關閉；必要時再開課（重設起訖時間）

學員作業
────────
1. UCET012 登入系統（首次使用者於登入頁註冊）
2. UCET007 於我的課程瀏覽已加入課程 / 輸入邀請碼加入新課程
3. UCET008 依章節順序學習教材（影片 / 文件 / 說明文字）
4. UCET009 完成章節後作答線上測驗
5. 重複學習至全部章節通過；完課後 UCET013 填寫課後問卷

管理者作業（於平台 DP 統一後台操作，按模組過濾只見 ET 相關項）
──────────
1. UCET010 於 DP 後台「權限管理」設定其他帳號之角色（管理者 / 教師 / 學員）與受訓單位標籤指派；於「系統參數與清單」維護標籤庫
2. UCET015 於 DP 後台「通知範本」（`MODULE=ET`）維護通知信範本；於「系統參數與清單」維護排程參數

系統排程
────────
1. UCET014 每週統計看課狀況寄週報；未開始學員寄提醒；截止前 3 天寄加急提醒

共用作業
────────
- UCET011 所有登入使用者皆可維護自己之個人資料
```

## 共通規則

以下規則適用於所有使用案例，各 UC 描述中不再重複說明：

- **帳號共用**：ET 與 DM 共用 `DP_USER`（USER_ID / 帳號 / 密碼 / 姓名），使用者註冊一次即可登入兩系統；ET 端首次登入自動授予「學員」角色
- **多重角色**：角色（管理者 / 教師 / 學員）可重複指派，權限取聯集；登入後依角色預設首頁導向，多重角色由側邊選單切換
- **稽核欄位**：所有資料異動自動記錄建立者 / 異動者、建立時間 / 異動時間
- **教材文件來源**：所有教材引用之文件統一由 DM 模組之「訓練教材」分類提供；DM 端發布新版時 ET 自動取得最新版
- **擁有權判定**：課程之擁有者於建立當下記錄並永久不可變更；他人課程僅可閱覽（唯讀），編輯與其他操作按鈕皆不顯示
- **受訓單位標籤**：ET 自持標籤庫（不與 DM 共用）；「全體」為特殊標籤代表所有具學員角色者
- **通知信**：所有系統通知信採管理者維護之統一範本（UCET015），收件對象一律為學員個人 Email
- **跨模組介接**：詳見 RQET.md 跨模組介接章節

## 使用案例描述

### UCET001 課程列表瀏覽

- **主要角色**：教師
- **關聯需求**：—（分析資料）
- **前置條件**：使用者具備教師角色並已登入
- **主流程**：
  1. 教師登入後進入課程列表（教師預設首頁）
  2. 選擇分頁：「我建立的」顯示當前使用者建立之課程（含草稿 / 已發布 / 已關閉全部狀態）；「全部課程」僅顯示所有教師之「已發布」課程（草稿與已關閉不列入）
  3. 透過搜尋區篩選：關鍵字（課程名稱）、受訓單位標籤；「全部課程」分頁額外提供「建立者」篩選
  4. 系統以卡片網格呈現課程（課程名稱、受訓單位標籤 badges、章節數、狀態、起訖時間、學員人數、建立者姓名）
  5. 教師點擊卡片進入課程編輯（自己建立）或檢視（他人建立、唯讀）頁面
- **替代流程**：
  - 點擊搜尋區右側「新增課程」→ 進入 UCET002 課程建立流程
  - 卡片右上顯示「檢視」標籤者為他人建立之課程，進入後為唯讀模式
  - 已關閉之課程於列表顯示「已關閉」狀態標示，教師點擊進入後可執行「再開課」（→ UCET003）
- **後置條件**：教師取得課程清單；可進一步進入編輯（UCET002）、關閉 / 再開課（UCET003）、邀請（UCET004）或學員追蹤（UCET005）

---

### UCET002 課程建立與編輯

- **主要角色**：教師
- **關聯需求**：RQET001, RQET002, RQET003, RQET005
- **前置條件**：教師於 UCET001 點擊「新增課程」或既有課程卡片
- **主流程 — 基本資料**：
  1. 教師填寫課程名稱（必填）、受訓單位標籤（多選；發布前至少 1 個）、課程起訖時間（發布時必填）、課程描述（選填）
  2. 標籤自 `ET_TAG` 啟用中標籤選取（全體 / 護理師 / 行政人員 / 軍人 / 醫檢師…）；已發布課程可**新增**標籤（觸發補邀請 → UCET004）、**不可移除**既有標籤；草稿可自由增刪
  3. 起訖時間為閱課期間：起始前學員看不到課程；到訖止時間系統自動關閉（→ UCET003）
- **主流程 — 章節編排**：
  1. 教師於章節編排區新增章節，並以拖拉方式調整章節順序、章節下教材與測驗之順序
  2. 每個章節下可放置任意數量之教材與測驗項目
  3. 刪除章節時跳 confirm modal；確認後章節本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員之學習紀錄與測驗成績連同移除；後續章節順序自動往前遞補
- **主流程 — 教材管理**：
  1. 點擊章節中「新增項目 → 教材」開啟教材編輯視窗
  2. 教材項目由三類媒材自由組合：影片（本地上傳 mp4 / webm，單檔 ≤ 500 MB）、文件（DM 引用）、說明文字（WYSIWYG）
  3. 同一教材項目可同時包含多支影片、多份文件與說明文字
  4. 教師從 DM「訓練教材」分類下拉選取既有文件建立關聯；DM 端發布新版時自動取得最新版
- **主流程 — 測驗建立**：
  1. 點擊章節中「新增項目 → 測驗」開啟測驗編輯視窗
  2. 教師設定測驗名稱（必填）、及格分數（預設 80）、作答時間限制（選填；空白 = 不限時、≥ 1 = 限時分鐘）、重考次數上限（0–999，預設 3）
  3. 教師建立題目（單選題 / 多選題），設定題幹、選項（2–6 個）、正確答案、配分（同一測驗各題配分總和 = 100）
  4. 題目順序可拖拉調整；刪除題目時題目本體軟刪除（DELETED=1，前台不再顯示但 DB 保留供稽核），學員之該題作答紀錄與得分連同移除
- **主流程 — 課後問卷建立（2026-07-02 新增）**：
  1. 教師於課程層級「課後問卷」區塊建立問卷（一課程 0～1 份，選配）
  2. 逐題建立單選題：題幹＋選項（如：滿意 / 普通 / 不滿意）；選項可自行新增 / 編輯 / 刪除
  3. 問卷已有任何學員填答後，題目與選項凍結不可修改（僅可停用問卷）；無人填答前可自由編修
- **主流程 — 儲存與發布**：
  1. 教師於頁面底部選擇「取消編輯 / 儲存草稿 / 發布」其中之一
  2. 「儲存草稿」：寫入草稿狀態，學員端不顯示
  3. 「發布」：系統檢核「至少 1 章節 + 1 教材」＋「至少 1 個受訓單位標籤」＋「起訖時間已填」；通過後寫入發布時間戳，並觸發標籤自動邀請＋寄信（→ UCET004）
- **替代流程**：
  - DM 文件廢止 → 教材頁顯示警告，由教師決定是否移除引用；學員端仍可閱讀廢止前最後版本
  - 發布檢核未通過 → 阻擋並提示缺漏項目
  - 課程進入已發布狀態後再次編輯 → 儲存後生效，不需重新發布
- **後置條件**：課程已建立或更新；若已發布且到起始時間，學員可於 UCET007 看到並於 UCET008 學習

### UCET003 課程關閉與再開課

- **主要角色**：教師 / 系統自動
- **關聯需求**：—（分析資料；2026-07-02 變更：關閉可逆）
- **前置條件**：課程處於已發布狀態（關閉）；課程處於已關閉狀態（再開課）
- **主流程 — 到期自動關閉**：
  1. 課程到達訖止時間，系統自動將課程狀態變更為「已關閉」
- **主流程 — 手動關閉**：
  1. 教師於課程編輯頁右上點擊「關閉課程」按鈕（僅已發布狀態顯示）
  2. 系統跳 confirm modal 確認
  3. 教師確認後課程狀態變更為「已關閉」
- **關閉後影響（唯讀）**：
  - 列表顯示「已關閉」狀態標示；學員端仍可見課程並**唯讀回看**已學過的內容
  - 學員不能再累積進度、不能作答測驗、不能解鎖新章節、不能填寫問卷
  - 關閉當下正在作答之測驗允許完成並計分；之後禁止開始新的作答
  - 邀請碼失效（再開課後恢復）；教師不可再邀請學員、不可重置重考次數
  - 每週統計與提醒（UCET014）停止，再開課後恢復
- **主流程 — 再開課**：
  1. 教師於已關閉課程點擊「再開課」
  2. 系統要求重新設定一組新的起訖時間
  3. 確認後課程狀態回到「已發布」；學員進度接續保留，可繼續學習
  4. 關閉 / 再開課可重複多次
- **後置條件**：課程已關閉（唯讀）或已再開放；歷史紀錄完整保留

### UCET004 邀請學員

- **主要角色**：教師 / 系統自動
- **關聯需求**：RQET006
- **前置條件**：課程處於已發布狀態
- **主流程 — 標籤自動邀請（主要，2026-07-02 變更）**：
  1. 教師於 UCET002 為課程掛受訓單位標籤並發布
  2. 系統取課程所掛標籤對應人員（依 UCET010 維護之使用者×標籤，限具「學員」角色者）之聯集、去重
  3. 系統將名單自動加入課程（加入來源標記「標籤帶入」），並每人寄一封通知信（統一範本，含課程名稱、起訖時間、學習連結）
  4. 「全體」標籤 = 全部具學員角色之使用者
- **主流程 — 事後貼標追溯**：
  1. 管理者於 UCET010 將某使用者貼上標籤
  2. 系統自動將其補加入該標籤所有「已發布且未關閉」課程
  3. 寄彙整一封通知信（列出所有新加入課程）
- **主流程 — Email 邀請（補件用）**：
  1. 教師於課程編輯頁點擊「邀請學員 → Email 邀請」，輸入學員 Email 清單
  2. 系統以統一範本寄出邀請信（教師不可編輯主旨與內文）；系統呼叫平台唯一發信服務寄送（經 `DP_EMAIL_LOG` outbox 至外部 Email Server）
  3. 學員點擊邀請連結後自動加入課程
- **主流程 — 邀請碼**：
  1. 課程發布時系統自動產生唯一邀請碼（8 碼）
  2. 教師點擊「邀請學員 → 邀請碼」複製邀請碼提供學員自行加入
  3. 學員於 UCET007 輸入邀請碼加入
- **替代流程**：
  - 已加入之學員再次收到邀請 → 點擊後直接進入 UCET008 學習頁
  - 已發布課程新增標籤 → 觸發該標籤人員之補邀請＋寄信
  - 邀請寄送失敗 → 寄出狀態保留為「待加入」，教師可於 UCET006 再次寄送
- **後置條件**：學員加入課程（已加入分頁）或進入待加入清單（UCET006）；被加入學員收到通知信

### UCET005 學員學習狀況追蹤

- **主要角色**：教師
- **關聯需求**：RQET008
- **前置條件**：教師具備課程之擁有權，課程已有學員加入

### UCET008 章節學習

- **主要角色**：學員
- **關聯需求**：RQET001, RQET003, RQET004
- **前置條件**：學員已加入課程，且課程於開放期間（起訖時間內）
- **主流程**：
  1. 學員於 UCET007 點擊課程進入學習頁，系統載入該課程之章節導覽列
  2. 系統依「上次觀看位置」自動定位至學員上次學習處
  3. 學員於中間內容區依章節順序學習：
     - 影片：以 HTML5 video player 播放（含播放 / 暫停 / 調速（倍速上限 2x）/ 音量 / 全螢幕）
     - DM 文件：PDF 於頁內嵌入預覽；非 PDF 顯示「下載原檔以本機應用程式開啟」連結
     - 說明文字：直接顯示
  4. 系統記錄學習進度（影片實際播放區段、文件閱讀狀態）
  5. 完成當前章節後系統依解鎖規則開放下一章節
- **解鎖規則**：
  - 上一章節之所有影片需觀看超過 80% —— 採**累計覆蓋率**：實際播放過之區段聯集 ÷ 影片總長 ≥ 80%；可拉進度條但只有實際播放過的區段計入，**直接拉到結尾不算看過**；倍速播放照算（2 倍速看完 = 覆蓋率 100%）；同一區段重複觀看不加成
  - 若該章節有測驗，學員須達到及格分數方可解鎖（→ UCET009）
- **替代流程**：
  - 教材內容可重複觀看，不影響進度
  - 完課後（若課程有問卷）章節導覽列顯示「填寫課後問卷」入口（→ UCET013）
  - 課程於學習中被關閉 → 內容轉唯讀回看，不再累積進度、不可作答
- **後置條件**：學員完成當前章節之學習；可繼續下一章節或執行 UCET009 測驗

### UCET009 線上測驗作答

- **主要角色**：學員
- **關聯需求**：RQET005
- **前置條件**：學員已通過當前章節學習，章節中設有測驗；課程於開放期間
- **主流程 — 引導頁**：
  1. 系統顯示測驗引導頁：測驗名稱、題數、及格分數、作答時間限制、剩餘重考次數、上次成績
  2. 學員點擊「開始測驗」進入答題介面
- **主流程 — 答題**：
  1. 系統依內建機制將題目順序自動洗牌
  2. 學員於答題介面操作：上方狀態列含題號進度、倒數計時、提交按鈕；左側題目導覽列可任意跳題
  3. 單選題以 radio 選擇、多選題以 checkbox 選擇；切換題目時系統自動暫存當前作答
- **主流程 — 提交與閱卷**：
  1. 學員點擊「提交」或時間到自動以當前作答狀態提交
  2. 系統即時自動閱卷並顯示總分、是否及格、答題明細
  3. 學員點擊明細可看題目原文與正確答案複習
  4. 學員可回看自己**每一次**作答之成績與逐題明細（不限最近一次）（2026-07-02 明確化）
- **多選題評分規則**：採部分計分公式：`max(0, (答對正確選項數 − 誤選數) ÷ 應選正確數 × 該題配分)`；每答對一個正確選項加分、每選錯一個錯誤選項扣分，最低為 0 分
- **替代流程**：
  - 未達及格分數且剩餘重考次數 > 0 → 學員可選擇「重新作答」
  - 未達及格且無剩餘重考次數 → 等待教師於 UCET005 重置次數後方可重考
  - 作答中切換瀏覽器頁面 → 系統提示確認避免誤離
  - 作答中課程被關閉 → 允許本次作答完成並計分；之後不可開始新的作答
  - 每次作答之成績皆獨立記錄；以最高分為結業成績
- **後置條件**：學員取得測驗成績；通過則解鎖下一章節（→ UCET008）

### UCET012 登入 / 註冊 / 忘記密碼

  4. 若使用者具備多重角色，登入後由側邊選單自行切換

  1. 使用者於登入頁點擊「忘記密碼」並輸入註冊之 Email
  2. 系統寄發密碼重設連結至 Email（30 分鐘內有效）
  3. 使用者點擊連結進入重設密碼頁，輸入新密碼後儲存
  4. 使用者以新密碼重新登入
- **替代流程**：
  - 查無此帳號 → 提示「查無此帳號，請先註冊」
  - 密碼錯誤 → 提示「密碼錯誤」
  - 重設連結逾 30 分鐘 → 提示「連結已失效，請重新申請」
- **後置條件**：使用者已成功登入並進入對應預設首頁

  2. 一人一信彙整（同一學員多門課未開始時，一封信列出所有課程）
  3. 已開始（進度 > 0%）之學員不寄週提醒
- **主流程 — 截止前加急提醒（每日執行）**：
  1. 系統每日檢查各開放中課程是否進入「訖止時間前 3 天」（天數可由系統參數調整）
  2. 進入者：對該課程所有未完課學員寄加急提醒信（標明課程名稱與截止時間）
  3. 每門課程之加急提醒只寄一次（不每日重複）
- **統計定義**：
  - 學員看課進度% = 已完成章節項目數 ÷ 總章節項目數（影片依覆蓋率 ≥ 80%、測驗依及格、文件 / 說明文字依開啟判完成）
  - 完課率 = 已完課 ÷ 已加入（不含已移除）
- **替代流程**：
  - 已關閉 / 未到起始時間之課程不納入統計與提醒；已完課學員不收提醒
- **後置條件**：週報已寄達教師與管理者；未看 / 未完課學員收到提醒

### UCET015 通知信範本維護（2026-07-02 新增）

- **主要角色**：管理者
- **關聯需求**：—（分析資料，2026-07-02 客戶變更）
- **前置條件**：當前使用者具備 ET 管理者角色；ET 通知範本集中存於平台 `DP_NOTIFY_TEMPLATE`（`MODULE=ET`），**編輯介面於平台 DP 後台「通知範本」畫面（按模組過濾）**（2026-07-08 集中化）
- **主流程**：
  1. 管理者於 DP 後台進入「通知範本」畫面（ET09；按模組過濾），讀寫平台 `DP_NOTIFY_TEMPLATE` 之 `MODULE=ET` 列
  2. 系統列出 ET 6 類範本：課程邀請通知、課程邀請彙整通知、課程內容更新通知、每週未看提醒、截止前加急提醒、週報（密碼重設 / 帳號變更驗證為平台系統信 `MODULE=DP`，不在 ET 清單、由平台維護）
  3. 管理者編輯範本之主旨與內文（支援變數：課程名稱、起訖時間、學習連結等）後儲存
  4. 管理者可調整排程參數（平台 `DP_PARAM.ET_WEEKLY_STAT_DAY_TIME` 週報執行時間、`DP_PARAM.ET_URGENT_REMIND_DAYS` 加急提醒天數）
- **替代流程**：
  - 教師不可逐課修改信件內容，寄出內容一律採統一範本（經平台唯一發信服務寄送）
- **後置條件**：範本與排程參數已更新；後續通知信以新範本經平台發信服務寄出
