執行以下步驟，互動式修改指定 GitHub Issue 的標題、body 或 label。

## 確認機制

所有需要使用者確認的環節，一律提供**數字選項**，使用者輸入數字即可：

| 使用者回覆 | 動作 |
|---------|------|
| 數字（如 `1`、`2`、`3`） | 執行對應選項 |
| 直接輸入文字 | 視為自訂內容 |
| `0` / `取消` | 停止並顯示「已取消。」 |

## 參數說明（$ARGUMENTS）

| 參數 | 說明 | 範例 |
|------|------|------|
| `#編號` 或 `編號` | 要修改的 Issue 編號（必填）| `/sti-issue-edit #22` |

---

## 執行步驟

### 步驟 1：解析參數

從 `$ARGUMENTS` 解析 Issue 編號（去除 `#` 符號）。
若未傳入編號，提示使用者：「請提供 Issue 編號，例如：/sti-issue-edit #22」，並停止執行。

---

### 步驟 2：取得目前 Issue 內容

使用 Bash 工具執行：

```bash
gh issue view {編號} --json number,title,labels,body
```

---

### 步驟 3：選擇修改欄位

顯示目前內容後，詢問使用者要修改哪些欄位（可多選）：

```
目前 Issue #{編號} 內容如上，請選擇要修改的欄位：
1. 標題
2. Body（內文）
3. Label（新增）
4. Label（移除）
請輸入編號（可多個，以逗號分隔，例如：1,3）：
```

---

### 步驟 4：填寫新值

依使用者選擇，逐一詢問新的值（每次一個，等待回答後再問下一個）。

---

### 步驟 5：顯示預覽並確認

顯示修改預覽，列出即將變更的欄位與新值，詢問使用者：
```
  1. 確認執行
  2. 需要修改（請說明）
```

---

### 步驟 6：執行修改

使用者選擇 `1` 後，組合 `gh issue edit` 指令並執行：

```bash
gh issue edit {編號} [--title "{新標題}"] [--body "{新body}"] [--add-label "{label}"] [--remove-label "{label}"]
```

---

### 步驟 7：顯示結果

輸出：

```
✅ Issue #{編號} 已更新。
```

並列出更新的欄位與新值。
